Create a Custom Notification

To create a Custom Notification, click the + Add Custom Notification row under the solution. The system will create a new empty row and enable the Notification Properties, on the right-side of the screen, to allow configuration.

Field

Description

Notification Solution (Required)

The system will default the Solution list based on where the Custom Notification was initiated.  Selecting a different Solution will update the location, on the left-side of the screen. The Solution selection will determine the icon used, in the Message sent to you, and can be used for sorting in your Notification Inbox.

Notification Name (Required)

The Notification Name is used to describe the purpose of the Message being sent to users. Enter the Notification Name following your corporate standard naming, if applicable. The Notification Name will not be seen in the Message sent but will be updated in the Notification subscription window, on the left-side.

Notification Type (Required)

The Notification Type provides those, receiving the Message, with an easy identifiable icon describing the purpose of the Message. The system will default all Custom Notifications as a ‘General’ Notification Type. Select the Notification type appropriate to purpose of the Message:

  • Assignment
  • Completed
  • Error
  • General
  • Information
  • New Task
  • Warning

Notification Message (Required)

The Notification Message is what is sent and can also be seen in your Notification Inbox. Enter the messaging to be sent.

UUID (Read Only)

The Universal Unique ID is a system generated identifier used when configuring the Notification to be initiated from either the JustEnough Job Manager or through customized stored procedure (i.e. CUSP hook)

Once the required information has been entered for the Notification, click Save. Please refer to the Schedule a Custom Notification and Configure a Custom Notification in a CUSP sub chapters to configure the Notification to be initiated.
