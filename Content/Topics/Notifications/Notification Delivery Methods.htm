Notification Delivery Methods

Below is a description of each Delivery Method. Selection at the Subscription level will set system defaults for all users. You may override these default by modifying the Delivery Method in the User Notification Preferences screen.

  • Hard Notifications – The system will display a persistent visual notification for manual interaction. When you manually interact with the notification, the system will mark the notification as read without updating the notification flag but the notification may be found in the notification history.

 

  • Soft Notifications – The system will display a visual notification to you for 5 seconds. If you decide to manually interact with the notification, to dismiss, the system will mark the notification as read and not update the notification flag but the notification can be found in the notification history.

    If you do not manual interact with the notification, the system will update the notification flag with the most current count of unread messages so you may access by clicking the Notification icon.

 

  • Silent Notifications – The system will not display a notification but will update the notification flag with the current count of unread messages. You may reference the notification in your 'inbox' where the system will display the unread messages in a highlighted format.

 

  • Ignore – The System will not display a notification to you nor will the system update the unread message count. The notification will be considered ‘dismissed’ and can be found in the notification history.
