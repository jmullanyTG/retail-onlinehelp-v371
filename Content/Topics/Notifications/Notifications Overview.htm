Notifications Overview

The purpose of Notifications is to communicate to you when submitted processes (e.g. calculations, forecasts, validations, etc.) have been submitted, completed and if any errors occurred. System Administrators can configure Notification delivery method defaults to facilitate your feedback or to allow you to continue work without interruption.

The Notification solution is made up of three (3) areas: Notification Subscriptions, User Notification Preferences and the User Notification Inbox.

The Notification Subscription screen provides the ability to subscribe to system owned notifications, by solution, and configure the default delivery method for each. Additionally, administrators have the ability to create Custom Notification, by solution, which can be sent on a scheduled basis by the JustEnough Job Manager or through customized stored procedures (i.e. CUSP hooks) within the appropriate solution.

The User Notification Preferences allows you to override any system configured delivery method defaults how the Notification will be received by the specific user.

The User Notification Inbox lists the Notifications as configured. Associated to the Notification Inbox is the Notification (i.e. flag) icon, located in the top right of the screen, which serves as an indicator as well as providing the number of ‘unread’ notifications. Clicking the Notification icon will display the Notification Inbox.

Notifications can be subscribed to and configured for the following solutions:

  • Allocation
  • Assortment Plan
  • Order Manager - Prepack
  • Assortment (Receipt) Plan
  • Cluster Site Manager (Custom Notifications Only)
  • Content Manager (Custom Notifications Only)
  • Curves
  • Events (Custom Notifications Only)
  • Inventory and Level Forecast
  • Group Forecast
  • Price Manager
  • Promotions (Custom Notifications Only)
  • System (Custom Notifications Only)
  • Workflow (Custom Notifications Only)
