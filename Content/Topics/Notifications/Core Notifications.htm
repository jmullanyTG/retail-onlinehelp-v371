Core Notifications

The Notification Solution Subscription list displays core notifications that a user/role might receive. Below is a description of each type and when it would be sent:

  • Queue Submission/Preparing/Coalescing/Ready – A notification is provided, once a process they submitted to the JustEnough Workload Queue, providing prioritization and the current number of processes ahead of it.
  • Queue Completion (w/o error) – A notification is provided when the JustEnough Workload Queue has completed the submitted process without error. This allows you to be alerted when a long processing job has been completed while working on another task within the software.
  • Queue Completion (w/ error) – A notification is provided when the JustEnough Workload Queue has completed the submitted process with error. This allows you to be alerted when items within the process could not be completed for further investigation.
  • Queue Removal – A notification is provided when your process is removed from JustEnough Workload Queue by a system Super User.
  • Queue Expired – A notification is provided when a submitted process has been on the JustEnough Workload Queue for a length of time that the system is configured to ‘expire’ the job.
