Report Using Attributes

Certain reports are designed to allow attribute values to be included to provide for the rollup of data sliced by the attribute values. One such report is the assortment choice count report (Plan > Assortment > Dashboard > Assortment Choice Count Report).

In the filter section of the report, users are able to select an attribute family to slice the data by, users will also be able to select the specific attribute values that are relevant to the report.

The returned report will then show the data sliced by the attribute values in question, in this case color group. The counts of variants will also be sliced across regular, promotional and markdown counts although only regular counts are shown above.

Using Attributes with Filters

After linking attribute families and values to entities such as variants or item masters users will have the ability to filter the entities using the attribute families in question. Because filters are commonly used throughout the system, this is an important concept.

In the filter maintenance (Maintain > Filters) users can select to create a new filter or edit an existing filter.

The system will request that an entity type is selected, this ensures that the appropriate attribute families can be returned and used in the filter.

All available attribute families associated with the selected entity will be available and multiple attribute families can be selected to further refine the attribute in line with filter creation criteria.

Users will now be able to select single or multiple values related to the attribute family selected
