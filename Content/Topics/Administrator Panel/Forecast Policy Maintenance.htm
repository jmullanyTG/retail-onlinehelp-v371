Forecast Policy Maintenance

Forecast policy maintenance views are used to manage settings related to forecasting which are specified by inventory. The functionality within these views will be incorporated into a Forecast Policy module in an upcoming release.

Forecast Effective Priority Maintenance

The forecast effective priority maintenance screen allows users to specify the priority and resolution when there are multiple potential answers for which demand signal source should be in effect.

Forecast View Config Details

  • Database View – The name of the database object which controls how forecast data is returned and consumed.
  • BOM – The Bill of Materials (BOM) flag should be checked if the BOM forecasts should be considered. If selected, BOM forecasts will be returned in the forecast type drop down within the forecast view config priorities widget.

Forecast View Config Priorites

Parameter driven widget based on the selected line within the forecast view config details widget. Users may create new entry lines for each forecast type to be considered.

  • Forecast Type – One of the available forecast types to be considered.
  • Priority – Priority ranking of the selected forecast type.

Any changes made within the view must be saved before updates are made.

Forecast Phase Time Frame Maintenance

Users have the ability to link an effective forecast series to a time frame in order to create a very specific phase period or periods. This allows for non-contiguous (Multi-Phasic) forecast series through configuration of schedule based time frames.

Users may select an inventory filter and apply time frames where applicable.

 

Results will be reflected in the effective forecast view.
