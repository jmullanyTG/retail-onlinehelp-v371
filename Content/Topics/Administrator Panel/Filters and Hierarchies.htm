Filters and Hierarchies

Hierarchy Creation

The JustEnough Hierarchy Filtering view, which is accessible through the views module and is used in the configuration process of an implementation.

Parameter

  • Entity Type: Hierarchy type

Hierarchy Level Configuration Editable Fields

  • Active: The Active flag is, by default, set to checked after creation of the hierarchy. When unchecked and applied the hierarchy, the level will no longer be accessible from the front end.
  • Available for AP: Available for AP need to be checked if the hierarchy must be available to select within Assortment Planning.
  • Is used in active AP Plan: This flag will become checked if the hierarchy is used in an active assortment plan.

Hierarchy Configuration Editable Fields

  • Order Number: The lowest number represents the highest level of the hierarchy.
  • Enabled: After the hierarchy is created, this flag will, by default, be unchecked. Setting this to checked a group makes it part of the current hierarchy being configured.
  • Important: Assortment Planning (AP) Specific Entity Types that must be configured are Variant and Site. Both of these must be created prior to creating any assortment plan. The combination of the lowest levels of both hierarchies will make the lowest level of the assortment plan and the level at which placeholders will be created.

Site Hierarchy

A full description of the location hierarchy must be constructed. Using the Site Groups available (Friendly Names are maintained through the Setup Table Definition Friendly Names view) as well as the explicit “channel” attribute, configure the Site hierarchy to have levels of commonality between Merchandise Financial Planning and Allocation Planning.

Item Master Hierarchy

Item master hierarchies are the hierarchies of the product dimension. They are used throughout the JustEnough modules to define hierarchal relationships and levels for both aggregation and dissemination of information.

Item master hierarchies are built out of item master groups, as well as the “variant” entity, and the explicit “style”,  and “size” attributes (where applicable).

Variant Hierarchy

Assortment Planning uses a Variant Entity hierarchy for its Product Hierarchy. This hierarchy will be used for all rollups and reporting in Assortment Planning. The Variant Hierarchy should be structured to extend to one level above Variant (equivalent to Placeholder in Assortment Planning; commonly referred to as style-color), unless a level for Style (referred to as a Collection within Assortment Planning; basically, a group of variants – often a single design in multiple colors) exists, then the Variant Hierarchy should extend to one level above that.

Note: When using the Segment By Hierarchy function of Target Creation within Assortment Planning, the lowest level of this hierarchy is the level at which target segments will be created.

Notes:

  • In order to use targets from a merchandise financial plan within a given assortment plan, the product and site hierarchies assigned to both plans must share common levels from the top down to the lowest level of the merchandise financial plan hierarchies (assortment plan hierarchies generally extend to at least one level lower). The plans must also share common time structures (calendar, period type).
  • Cluster Sets to be used within assortment plans must be contained within a single leaf level of the site hierarchy assigned to those plans – keep this in mind when structuring hierarchies.

Find and Replace Filter Criteria

The Find and Replace Filter Criteria view allows the user to find any filters created with a particular criteria, and to replace that criteria with a new one.  A common use of this view may be after a hierarchy reclass; for example, you may need to find any filters that include the condition “items in Company > Department Womens > Subdepartment Tops > Category Tees” and replace that condition with “Company > Department Womens > Subdepartment Shirts > Category Short Sleeve.”

Inputs

In order to use the view, select:

  • An entity type of filters to search for
  • A “Find Filter”
  • A “Replace Filter”

Usage

The view process will find any filters who contain the Find Filter criteria and value.  It will find filters with additional criteria, besides the one specified in the Find Filter.

You can select the Update Criteria? Checkbox, and click Save, and it will replace the criteria and value with the Replace Filter criteria.
