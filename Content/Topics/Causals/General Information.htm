General Information

The general information will display header level information related to the causal, including the name, and type.

  • Name – The user created name for the causal.
  • Type – The type of causal. Understanding the distinctions between the causal types and their correct applications is a key to properly using the causals manager.
  • (normal) Causal – An additive causal, it will add or subtract based on a non-explicit scoring factor (the default value).
  • Manual Causal – A multiplicative causal, it will adjust up or down based on an explicit factor, such as a 10% increase or a 20% decrease.
  • Event – Useful if there is an obvious jump in the data series, but the cause of which is unknown. The forecasting engine will match it to the best fit.
  • Informational Causal – For informational purposes only. This type of causal is rarely used within the JustEnough system.
  • Bucket Type – The type of container period for the series values of the associated causal. This must match the bucket type(s) of the eventual linked items/inventories.
  • Default Value – Defines the nominal value for the causal set. In almost all cases, this should be set to 0. Applying a non-zero to the default value will elevate (or decrease) the nominal value to the entire history of the linked items.
