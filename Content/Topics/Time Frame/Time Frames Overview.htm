Time Frame Overview

Time frames simply define the window(s) of time to use for a JustEnough operation, such as creating an Allocations product model or generating a demand forecast. They are reusable across JustEnough modules and shareable with other users.

Static and dynamic options

You can create time frames based on static calendar dates (January 1, 2017 - December 31, 2017) or on date ranges that dynamically update on a rolling basis (last 180 days, next 90 days, etc.)

Support for multiple time periods

You can also create time frames that span multiple time ranges. For example, you may want a time frame that covers each January for the years 2015-2017. That time frame would include the following distinct date ranges: January 1-31, 2015; January 1-31, 2016; January 1-31, 2017.

Schedules support

For convenience, you can base a time frame on schedules created in the Schedules module. Simply select the schedule you want to use and its effective dates are used for the time range.

Controlled Access

You can decide whether to keep a time frame private (for your use only), public (for use by any JustEnough role or user), or shared only with selected users or roles.

Central maintenance and local availability

The Time Frame module provides a centralized maintenance window where you can create and edit time frames for use in other JustEnough modules. You can also access this maintenance area when selecting a time frame in a supported JustEnough module, making it easy to view, edit, and create time frames without having to switch away from your current work area.
