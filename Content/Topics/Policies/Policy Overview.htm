Policy Overview

Inventory policies determine the rules by which JustEnough's replenishment engine operates. They are defined by a set of parameters that can be managed at the inventory level within the Item Manager. You can use policy tasks to assign default settings for these parameters to specific groups of items, and then refine these settings as needed on an item-by-item basis in the Item Manager.

Policy parameters

Policy parameters greatly influence the replenishment strategy for the item or items contained within the filter. Replenishment is done at review period intervals. By considering whether inventory levels are expected to drop below the safety stock level within the next lead-time, replenishment values are recommended to restore inventory to its maximum level.

Policy parameters define:

  • What type of order policy to apply to an item (normal stocked, non-stocked, bin, redundant). This determines whether or not an item should be replenished and, if so, what ordering methodology to use.
  • The minimum amount of safety stock for an item at a location.
  • The expected rates of returns and reuse.
  • How frequently inventory levels are reviewed for re-ordering.
  • How often orders can be placed for the item.
  • What stores can stock an item (store eligibility).
  • And more.

Policy tasks

A policy task lets you define the default values for a set of policy-related parameters and apply those values to the items in a specific filter. Once created, tasks can be executed manually in the Policy module and automatically through the Job Manager. You can set start and end dates for a policy task, giving you the flexibility to create tasks that should only apply for a specific period of time, such as a holiday season.

You can create as many policy tasks as you need. Note, however, that if multiple tasks are applied to the same item, each successive task may "overwrite" the previous task's policy. You can control the order which tasks are executed from within the Policy module.
