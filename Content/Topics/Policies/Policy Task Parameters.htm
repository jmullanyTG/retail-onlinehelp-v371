Policy Task Parameters

Policy task parameters appear on two tabs: General and Policy.

General tab parameters

Parameter Description
Name The display name of the policy task.
Description An optional description field.
Filter The filter containing the items that the settings in this policy task will apply to.
Default Task Whether or not this is the default policy task for all items.
Task Enabled Whether or not this task is enabled. If the task is not enabled, it will not be applied to items.
Task Execution Order The order in which this task will be executed among all other active tasks. Execution order is important because tasks executed "later" in the order supersede those executed earlier.
Task Available From

Select these options if you want to set start and/or end dates for this task. The task will automatically be enabled on the Task Start Date and disabled on the Task End Date.
Task Available To

Task Available From Date

 Select Task Enabled From to enable this property.
Task Available To Date Select Task Enabled To to enable this property.

 

Policy tab parameters

Policy Task Actions box

Parameter Description
Manual Order Policy Indicator

Sets whether or not the Order Policy Type of items included this task can be automatically changed by this or other policy tasks.

  • Set All Items As Auto - An item's Order Policy Type can be automatically changed by a policy task.
  • Set All Items As Manual - An item's Order Policy Type cannot be automatically changed by a policy task. It must be changed manually.
  • Do Not Update - Preserves the current setting of this property in each item.
Manual Policy Setting

Sets whether or not the policy settings of items included this task can be automatically changed by this or other policy tasks.

  • Set All Items As Auto - An item's policy settings can be automatically changed by a policy task.
  • Set All Items As Manual - An item's policy settings cannot be automatically changed by a policy task. It must be changed manually.
  • Do Not Update - Preserves the current setting of this property in each item.

 

Policy box

Parameter Description
Order Policy Type

Sets whether and how order recommendations are generated for items in this task.

  • [Do Not Set] - Preserves an item's current Order Policy Type.
  • Bin- Applies a fixed-quantity order methodology to items (for example, if one is sold another one is ordered). Best suited for items with intermittent, low demand.
  • Non-Stocked - Prevents JustEnough from making replenishment recommendations for these items, except in case of customer orders that cannot be filled with current stock on hand.
  • Normal Stocked- Items that are everyday-selling items which should be continually stocked and replenished. JustEnough generates order recommendations for these items based on inventory policy and item-specific ordering criteria.
  • Redundant- Items that will not be sold. JustEnough will not re-order these items.

 

Replenishment Strategy box

Note: This table describes all possible replacement strategy parameters. Not all of these parameters appear in a given policy task. The parameters that appear in a specific policy task are determined by the task's Order Policy Type.

Parameter Description
Back Order Ratio The percentage of lost sales which areexpected to result in a customer returning to purchase the item.
Bin Min Units The minimum quantity of units required in the bin.
Bin Max Units The maximum quantity of units allowed in the bin.
Review Period Days The number of days between inventory reviews for this item. Inventory reviews determine whether or not the item needs to be re-ordered to comply with stocking parameters.
Replenishment Cycle Days The number of days that must pass between orders for this item. Must be greater than or equal to Review Period Days.

 

Safety Stock box

Notes:

  • This table describes all possible safety stock parameters. Not all of these parameters appear in a given policy task. The parameters that appear in a specific policy task are determined by the task's Order Policy Type.
  • The system policy Max Safety Stock as Percent of Max Stock lets you cap safety stock quantities at a specified percentage of an item's Maximum Stock Units value. Unit-based minimums (for example, Minimum Safety Stock Units) that exceed the Maximum Stock Units value will still be respected.
Parameter Description
Minimum Safety Stock Units The minimum planned inventory level (in units) for this item at your locations.
Safety Stock Days

The number of days during which safety stock units may be sold to meet demand for the item.
Target Service Level The target service level (as a percentage) for the item.

 

Returns box

Parameter Description
Return Rate The percentage of items which are expected to be returned.
Days Before Return

The expected average number of days between the sale of an item and its return.

 
Reuse Rate

The percentage of returned items that are expected to be salable stock again.
Days Before Reuse

The expected average number of days between an item being returned and it being considered salable stock again.
