Workflow Manager Overview

Workflows define processes related to specific JustEnough objects. At each step in a workflow, you control which users or roles have access to what actions, and, optionally, what actions the system performs. You can use workflows to:

  • Set up approval processes for quality control.
  • Promote "safe" cross-team collaboration.
  • Ensure required steps and operations are always executed.
  • Automatically enforce corporate standards and policies.

The Workflow Manager lets you create workflows for the following JustEnough objects:

  • Events
  • Promotions
  • Price Actions
  • Order Headers

Workflows

JustEnough workflows have two main components: statuses and transitions. A status is a specific "step" in the workflow. A transition moves the object from its current status to its next status. Statuses and transitions are illustrated in the following workflow example:

This workflow has 4 statuses: Planning, Ready for Approval, Changes Needed, and Approved/Confirmed for Export. The transition arrows connecting the statuses indicate what status can come "next". For example, the Ready for Approval status can transition to either "Changes Needed" OR "Approved/Confirmed for Export".

When designing a workflow, you can define the conditions in which a status can move from one status to the next. In some cases, you may want a specific person or role to decide (for example, a reviewer should determine whether an object in the "Ready for Approval" status moves to "Approved" or "Changes Needed"). In other cases, you way want to the transition to occur automatically on a specific date.
