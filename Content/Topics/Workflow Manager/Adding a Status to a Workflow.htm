Adding a Status to a Workflow

  1. From the Maintain tab, click Workflow Manager.
  2. In the Workflows list. select the workflow you want to add a new status to.

  3. Click New in the Status toolbar.

  4. A new status box appears in the workflow diagram.

    TIP: If you select an existing status box in the workflow diagram before clicking New, a transition will automatically be created from the selected status box to the new status box.

  5. In the Information pane, edit the following status information properties:

    Property

    Description
    Name The status name appears in the status box in the Design pane.
    Completion Days Out

    The number of days before or after the workflow object's goal date that the status is due. See Trigger Method below. (Optional)

    A positive number indicates the status is due before the goal date. A negative number indicates the status is due after the goal date.

    For example, a workflow's goal date is typically the object's start date (promotion goal date = the sale/promotion start date; event goal date = the ad start date; price action oal date = the price action start date, etc.). So if you want this status to be due 30 days before the start date, enter the value "30". If you want this status to be due 14 days after the start date, enter the value "-14".

    Prefix Method

    The method JustEnough should execute before applying this status. (Optional)

    A method is an advanced process applied at the database level (such as the execution of a SQL script). This list is empty unless methods are configured to appear in it.
    Post Fix Method

    The method JustEnough should execute after applying this status. (Optional)

    A method is an advanced process applied at the database level (such as the execution of a SQL script). This list is empty unless methods are configured to appear in it.
    Trigger Method

    The method JustEnough should execute once the completion day specified in the Completion Days Out property arrives.(Optional)

    This list includes the method "Auto Advance Node Status by Completion Days Out for Workflow Triggers", which automatically advances the workflow to its "next" status once the completion day is reached. If there is more than one possible "next" status (for example, "Approve" or "Change Needed"), the workflow will advance to the status with the highest transition weight.

    Note:

    • Completion Days Out are tracked using the Workflow Heartbeat task, which is in the daily job run by the JustEnough Job Manager. This task stops tracking days out in a workflow on the furthest "negative" day out specified in the workflow. If the heartbeat task does not run on that specific day, the trigger method will never automatically execute.
    Status Permission Blacklist

    The state-based permission sets to enforce in JustEnough while the workflow is in this status. The list includes a row for each user right controlled by the workflow and a column for each possible object state. A black user icon indicates the user right is granted to all users when an object governed by the workflow is in that state. A blue user icon indicates the user right is denied to one or more users when an object governed by the workflow is in that state.

  6. Click Save on the Workflow toolbar to save the change to the workflow.

Next steps

Once a status is created, you can start making the following changes:

  • Edit its status information and permission blacklist.
  • Add or edit transitions to the new status.
