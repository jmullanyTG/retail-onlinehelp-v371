Essential Business Concepts

Demand plans are generated from one of four potential sources:

  • Financial Plan – A strategic merchandise financial plan. The output of this planning process will be a by period target value for a segment of the product hierarchy. This may be further segmented with a location dimension, but the specificity of a by store plan is usually reserved for location planning.
  • Location Plan – A by store merchandise financial plan.
  • Rolling WOC – A calculated rolling weeks of cover value, based on sampled sales history.
  • User Target Value – A manual entry for a need target.

When creating a demand plan, users will be prompted to select one of these possible demand sources; the configuration screen will then update with the relevant input parameters for the selected demand source.

  • Demand Plan Name – The unique name for the demand plan. Will be used to look-up demand plans within the scenario plan configuration screen.
  • Demand Plan Description – Optional description for the demand plan.
  • Demand Source – Displayed value for the demand source selected.
  • Product Hierarchy – The system policy selected product hierarchy for use within the module. This product hierarchy should be configured to cover the product dimension from top to bottom. It should be consistent with hierarchies in use within any financial planning functions.
  • Product Hierarchy Selection – The sampled level within the product hierarchy.
  • Time Frame – Defines the desired periods of history (Rolling WOC only) to be used when creating the demand plan target values.
  • Access Level – Used to define user access to the demand plan. This will control its visibility in this screen as well as restrict its selection for use within a scenario plan.
  • Public – Available to all users within the application.
  • Private – Available only to the creator.
  • Shared – Available to only specific users.
  • User Select – If set to a shared access level, the list of shared users.
