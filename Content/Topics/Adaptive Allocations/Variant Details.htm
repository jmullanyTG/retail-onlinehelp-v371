Variant Details

The variant details section will be populated for a product set once configuration has completed. It is updated based on the highlighted product set in the top grid.

  • Selected – Will be checked if the variant is selected to be included in the product set. The default behavior of the “selected” box is determined by the “opt-in” box in the product set configuration grid. If an update is made to the product set while the “opt-in” box is checked, upon save, all variants will become selected.
  • Variant Code – The unique code describing the variant.
  • Variant Description – The optional description for the variant.
  • Total On Hand Units – Total in-network on-hand for the variant. Includes on-order, pending, and supersession stock.
  • Total On Hand Value – The value, in base currency, of the total on-hand units.
  • Store On Hand Units – Total on-hand for stores.
  • Store On Hand Value - The value, in base currency, of the store on-hand units.
  • DC On Hand – Total on-hand for all DCs.
  • DC On Hand Values - The value, in base currency, of the DC on-hand units.
  • On Order Quantity Units – Total quantity on order.
  • On Order Quantity Value - The value, in base currency, of the on-order units.
  • Size Fulfilment – A percentage representation of ownership across the size range. While not representative of specific size quantities, it will provide a quick reference for variants with exceptionally broken size runs.
