Essential Business Concepts

Choice count models are built to sample sales history based on a filter of items, a time frame, sales types, and then removing disqualified sales periods.

  • Choice Count Model Name – The unique name for the choice count model. Will be used to look-up choice count models within the scenario plan configuration screen.
  • Choice Count Model Description – An optional description for the choice count model.
  • Product Filter – The filter of items which will be used to calculate the choice count. Though not required, a more consistent result is expected if this filter has some relevance to the product selection used within demand plan generation.
  • Time Frame – The time frame used to sample sales history.
  • Sales Volume % of Max Threshold – The percentage of the maximum sales volume of the sample which must be met for a selling period to be included.
  • Active Variant Threshold Factor – The percent of each store-variant’s average sales which will set the minimum threshold for de-seasonalized values. This value is available for use when calculating longer sales periods with highly seasonal products within the sample, but is otherwise not applicable to many common Adaptive Allocations scenarios. System default value for this threshold is “0”. This field is hidden by default, but may be returned to the configuration gird through the “choose columns” functionality.
  • Minimum % of Selling Periods Threshold – Defines the percentage of periods within the sample to have recorded a sale in order for the variant to be considered a part of the choice count model. By setting this to a high value, this will eliminate low selling periods for variants which may have set late or had stock-out related issues.
  • Sales Types – The sales types included in the sampling. Options include regular, promotion, markdown, and all supersession sales types (should be used for modeled stores and new products).
  • Access Level – Used to define user access to the choice count model. This will control its visibility in this screen as well as restrict its selection for use within a scenario plan.
  • Public – Available to all users within the application.
  • Private – Available only to the creator.
  • Shared – Available to only specific users.
  • User Select – If set to a shared access level, the list of shared users.
