Rolling WOC

If “rolling WOC” is selected as the demand source, the following fields will appear within the detail configuration window:

  • Product Hierarchy – This is a read-only field which will display the system policy defined product hierarchy. It is purely informational.
  • Product Selection – A mapping of the applicable levels within the product hierarchy which will be sampled to create the demand plan.
  • Time Frame – Defines the periods within the plan which will be sampled to create the demand plan.
  • Configure – Users are able to create a new time frame for use if an applicable selection is not already available.
  • Sales Types – The included sales types for sampling. Users may choose to include or exclude markdown or promotional sales types, or any type of supersession (modeled) sales types.
  • Product Model – The product model is a calculated index of sites which will function to appropriately dis-aggregate need target values across stores. They may be setup and configured within JustEnough Allocations.
  • Adjustment % - An overall adjustment factor for the sampled period. May be a positive or negative value. If utilized, will adjust the overall by store need target values.
