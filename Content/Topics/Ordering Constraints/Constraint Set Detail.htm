Constraint Set Detail

Values will change as users highlight different constraint sets within the header. When building a new constraint it is easy to think of the constraint as the following template sentence:

Constrain so that “Constraint Type” is “Operator” the value “Constraint Value”.

  • Priority – Constraints are applied in order of priority within context of the constraint set. Priority 1 will applied last and may override lower prioritized constraints.
  • Constraint Type – The inventory property which is to be constrained. All monetary constraint values apply to the system default currency. One of the following attributes:
  • Default Cost Value – A standard cost value.
  • Default Sell Value – A standard value based on retail price.
  • Quantity – The quantity of units.
  • Volume – A volumetric measurement. Volumetric data must be present.
  • Weight – A weight measurement. Weight data must be present.
  • Secondary Cost Price – A specific cost type labeled as “secondary cost price”.
  • Secondary Sell Price – A specific cost type labeled as “secondary sell price”.
  • Freight Cost – Associated shipping cost for the item or items.
  • Cost – Overall cost of the order.
  • Operator – The mathematical operator for the constraint. May be one of the following:
  • Greater Than
  • Less Than
  • Equal To
  • Equal to Multiple of – Used for basic case pack logic.
  • Constraint Value – Numerical value of the constraint.
  • Tolerance Above – Should the system be unable to constrain the value exactly to the correct amount, the tolerance above the value which is considered acceptable. This may be expressed as a percentage, or quantity of units.
  • Tolerance Below – Should the system be unable to constrain the value exactly to the correct amount, the tolerance below the value which is considered acceptable. This may be expressed as a percentage, or quantity of units.
  • Rounding – For instances when the operator is “equal to multiple of”, acceptable rounding rules must be selected.
  • Round Up
  • Round Down
  • Round to Nearest

Additional fields may be added by including groups of items within the selection.
