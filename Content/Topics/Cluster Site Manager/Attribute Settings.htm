Attribute Settings

The Attribute Settings cluster set property lets you add, edit, delete, and rank the attributes used to calculate cluster sets.

Attribute toolbar

Action Description
Add Launches the Add Attribute dialog box.
Edit Edits the selected attribute.
Delete Deletes the selected attribute.
Adjustment Factors Launches the Adjustment Factor dialog box, where users can specify ratio modifiers for each attribute.

Attribute properties

Property Description
Attribute Name The name of the attribute.
Attribute Description The description of the attribute.
Attribute Type

Options include:

  • Site Attribute: Select if you want to calculate cluster sets based on numeric site attributes such as selling square footage or average temperature.
  • Sale Attribute: Select if you want to calculate cluster sets based on historical sales data.
  • Forecast Attribute: Select if you want to calculate cluster sets based on forecasted sales data.
Priority The priority ranking of each attribute. Adjust ranking up and down using the arrows on the right-hand side of the attribute list. The attribute with the lowest number has the highest ranking and will contain the stores with the highest relative value.

Adding an attribute

  1. In the Attribute Settings property toolbar, click Add.
  2. In the Add Attribute dialog box, enter a name an description for the attribute.

  3. Select the Attribute Type (see Attribute properties above for a description of each type).

    • Sale Attribute and Forecast Attribute types have additional properties. See the following tables for descriptions.
  4. Click OK to create the attribute.

Sale attribute properties

Property Description
Sale Metric Type

The sale metric to use for calculating cluster sets. Options are:

  • Unit: uses sales unit volume to cluster stores.
  • Revenue: uses sales revenue volume to cluster stores.
  • Cost: uses cost of sales (COGS = Sales units * average cost per unit) to cluster stores.
  • Gross Margin: uses gross margin revenue (Sales Revenue – COGS) to cluster stores.
  • Gross Margin %: uses gross margin percentage ((Sales Revenue – COGS) / Sales Revenue) to cluster stores.
  • GMROII: uses gross margin return on inventory investment (Margin Revenue / COGS).
Time Frame The time frame to use for gathering historical sales metrics ("last 90 days", etc.). Options include all available time frames. Click Maintain to create a new time frame or edit one in the list.
Use Item Filter

Whether or not to use a product filter to select the product data to use in cluster calculation.

Note: Item Filters, if used, supersede the selected product hierarchy level for each cluster set.

Example: I create a cluster set template of type K-Means Collection Item Partition that uses the Subclass level of the selected product hierarchy to create different cluster sets; I choose to use an Item Filter with the criteria of Color Group = Black; for each cluster set, the data used in calculation will all product for which the user has access based on their assigned permiassions that have the Black Color Group attribute assigned to them.

Item Filter The item filter to use for gathering sales metrics.
Sales Types The type of sales to use in cluster calculation. (Sales types are defined during software implementation.)
Include Returns Whether or not returns (negative sales) are included in the sales data used in cluster set calculation.
