Editing a Filter

You can edit a filter’s details, criteria, and conditions by selecting Update the Following Filter from the Maintain Filters dialog box.

Filter details

Detail Description
Filter Name The name given to the filter.
Filter Note An optional notes section.
Filter Group Optional. Filter groups function in a similar fashion to a folder structure and are used to organize filters for ease of navigation.
Share This Filter Select to share this filter with all or some users. (If sharing with some users, click ‘…’ to choose the set of users.)
Filter is Static Select this option to prevent the data returned by this filter from updating once it has been created.
  • Select Rebuild Data on Save to update the data in this filter when you save changes to this filter.

About filter groups

Filter groups make it easier to organize and find filters. You can create a new filter group by clicking “…”next to the Filter Group list.

Filter criteria and conditions

Filter criteria and conditions work hand-in-hand. Filter conditions determine the field values that a filter searches for, and filter criteria determine whether an entity that meets those conditions is included or excluded as a result.   When managing filters with multiple conditions, it is important to give care to the order in which the conditions are listed.

Filter criteria

Filter criteria may be one of the following:

  • Criteria where all of the following must be met
  • Criteria where any of the following must be met
  • Criteria where none of the following must be met

Filter conditions

Adding or editing filter conditions is done through the filter condition window.

Filter conditions involve the following settings:

Setting Description
Field The filed to use for the condition.
Condition How the value of the field in the entity data must compare to the value specified in the condition.
Equals Matches the exact value
Contains the Phrase Searches the field for the exact value.
Contains the Phrase at the Beginning Searches the field for the exact value at the beginning of the phrase.
Contains the Phrase at the End Searches the field for the exact value at the end of the phrase.
More Than Greater than the value.
Is After Greater than the value for date-times.
More or Equal to Greater than or equal to the value.
Is After or On Greater than or equal to the value for date-times.
Smaller Than Less than the value.
Is Before Less than the value for date-times.
Smaller or Equal to Less than or equal to the value.
Is Before or Equal to Less than or equal to the value for date-times.
In Between Set between two values. Selecting this option will require more than one value, identified as the “start value” and “end value”.
Is One of the Following Will include/exclude the entities based on several supplied values.
Sounds Like Searches the field for similar sounding values (%Value%).
Value(s) The value or values to search for in the field data. Varchar, char, and nvarchar fields will present drop down lists. Decimal, float, and int (integer) fields will present users and opportunity to enter a numerical value. Datetime fields will present the calendar and clock view to create a value. Bit fields must have a value of 1 or 0.

Allow dynamic condition

Select Allow dynamic condition to let users choose the condition value when they use the filter. For example, a site filter with a dynamic Site Region value condition lets users choose which regions they want to work on at the time they select the filter. This can be much more convenient than creating separate filters for each region.

Filter conditions and data implications

When designing a filter, you have several decisions to make that determine how (and what) data is returned when users select the filter.

  • Should the whole filter be static?

    • A static filter always returns the same data set.

    • To make a filter static, select Filter is Static in the filter details.
    • Example: A filter is created in January 2018 at the end of the planning period” Block 1” for the class “Ladies Shirts”. The data set returned by this filter will contain all variants in Ladies Shirts at the end of Block 1. This data set will not change once the filter is saved.

  • What conditions should be static?

    • Static filter conditions do not change when the filter is used, however the actual data returned by the condition may change as time goes on.

    • To make a static condition, add the condition and do not select Allow dynamic condition.
    • Example: A filter is created to return all variants for the class, “Ladies Shirts”. A condition is added (class value = “Ladies Shirts”). As new styles are created for the class “Ladies Shirts”, the data set returned by the filter is updated to include the new styles.

  • What conditions should be dynamic?

  • Dynamic conditions may be changed when a user selects the filter. The data returned by the filter will be determined by the user’s selections.

    Example: a filter is created with a dynamic condition for Class. When a user selects this filter, they can choose which class to filter on. When Ann selects this filter, she selects the class value, “Ladies Shirts” to see all styles of ladies shirts. When Zoe selects this filter, she selects the class value, “Men’s Shirts” to see all styles of men’s shirts.

    To make a dynamic condition, add the condition and select Allow dynamic condition.

Filters are color-coded so that when selecting a filter, users know which filters are static and which have dynamic conditions.
