Technical Notes

  • When a new filter is created, its cache is built immediately.
  • Rebuilding filter caches is an extremely important task and should, in most instances, be run on a daily basis. This may take a considerable amount of time during the overnight run and will depend on the volume of data and the number of filters which need to be rebuilt.
  • Custom code should no longer reference tbl_FilterCache, because population of tbl_FilterCahce is optional when filters are rebuilt.
  • Udf_ApplyFilter has been officially scheduled for deprecation and should not be called.
