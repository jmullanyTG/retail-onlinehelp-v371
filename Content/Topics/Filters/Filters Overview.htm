Filters Overview

Filters are an essential tool for working in JustEnough, as they let you see only the subsets of data that you want to work with in a given screen (or that you want JustEnough to work with in a given process). You can make filters as simple or as complex as you need, and then save them for future use. Once created, you can share filters with your entire organization, with select others, or keep them for your own private use.  

Flexible designs

Filters are extremely flexible and are able to support:

  • Static data: once the filter is created, the data it returns never changes. This assures users that they will see the same data whenever they use the filter.
  • Static conditions: the conditions used to filter data cannot change, but the data returned can change. This assures users that the data they see when using the filter is current. 
  • Dynamic conditions: users can choose the condition values used to filter data. This assures users that the data they see when using this filter is relevant to their current need.

Entity-specific filtering

Because different areas of JustEnough involve different data “entities” (suppliers, inventory, sites, etc.), JustEnough automatically shows users only the filters that apply to the relevant entity. For example, in the Suppliers screen, users will see a list of available supplier filters as opposed to a mix of supplier, inventory, and site filters.
