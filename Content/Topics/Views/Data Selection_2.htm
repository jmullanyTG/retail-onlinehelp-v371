Data Selection

The data selection tab will determine how data is brought into the widget.

  • SQL Connection Type – Choosing a SQL connection type will tell the widget from which database to return the data.
  • Operations Database – The normal database containing most of the current and operational data for JustEnough. This will be the selection for most applications.
  • Warehouse Database – If configured, the warehouse database may function as a container for archived data. Selecting the warehouse database to return data to a widget
  • Method – The method drop down will return every SQL stored procedure listed in tbl_Common_StoredProcedureLookup, which should be representative of every stored procedure in the database. This is the procedure which will execute when the “execute” button is pressed on the widget (when parameters are present) or which will return the data to the widget (i.e. a SELECT statement).
  • Method Parameters – Upon selection of the stored procedure, the “Method Parameters” window will populate with the required parameters for the procedure. These parameters will vary greatly depending on the requirements of the stored procedure.
  • Method Parameter Name – The name of the required parameter as specified in the stored procedure.
  • Parameter – The selected parameter entry to be passed. These may be selected from the list of parameters detailed in the “maintain parameters” section.
  • Maintain Parameters – Will bring up the “select parameter” window, displaying all available parameters. More detail may be found in the “maintain parameters” section.
  • Default Value – The default value, if any, passed to the parameter.
