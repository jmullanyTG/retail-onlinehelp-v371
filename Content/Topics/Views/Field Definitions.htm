Field Definitions

  • View Name – The name for the selected view. View names do not need to be unique within the view group, but it is highly suggested to add some manner of unique identifier to the view name for ease of navigation in the future. View groups may be created and maintained in the view groups pane to the left of the view creation/edit window.
  • View Description – A simple description of the view. May include author or date if desired.
  • View Type – Will determine the layout possibilities for the widgets contained within the view. Requirements will vary depending on the number of widgets.
  • Vertical Widget Layout – Will stack the widgets vertically based on the order provided in the widgets pane below. If the view contains only one widget, it is common to see this view type selected.
  • Tabbed Widget Layout – Will create a tab style layout similar to those provided by the view collections.
  • Layout Controlled Widget Layout – The most customizable layout, allowing for much greater customization and configuration of the view. If selected, the widget order will become irrelevant and the user will be able to “drag and drop” elements within the views to manipulate the widgets, scroll bars and separators in order to best frame the workflow process.
  • Hide Parameters – If there are required parameters for the view to pass to the database in order to retrieve data, they will be displayed near the top of the view. If it is preferred to hide the parameter section in order to accommodate more screen space, this box may be checked.
  • Auto Refresh on Parameter Change – In cases where it is intended for the users to toggle between several parameters within the same view, the “auto refresh on parameter change” may be selected to make the entire view refresh without the need to click the “execute” button.
  • Auto Refresh on Widget Parameter Change – Will refresh data when a parameter within a widget changes.
  • Allow Update Method Parameter Value – In situations where there are more than one parameter and they are dependent upon one another, selection of this box will allow the parameters to update if there is a change to one or the other.
  • Allow Use and Save User View Parameter Values – If selected, will save the parameters when the view is closed and reopened.
  • Widgets – The widget pane will display the widgets selected for inclusion in the view.
  • Widget Name – The assigned name as defined in the “maintain widgets” configuration sub-menu.
  • Widget Description – The assigned description as defined in the “maintain widgets” configuration sub-menu.
  • Order – For vertical widget layouts and tabbed widget layouts, this will determine the order in which the widgets appear in the view. The lowest number will be stacked near the top in a vertical layout and near the left in a tabbed layout.
  • Assign Access Rights – This button will bring up the included and excluded roles and users wizard commonly used throughout JustEnough. Roles or specific users may be granted access to this view or restricted as required. If users are having trouble finding a needed view, this is a common troubleshooting spot to check.
  • Parameter Sequence – The parameter sequence pane will display the parameters required for each of the widgets listed in the above widgets pane.
  • Parameter Name – The assigned name as defined in the “maintain parameters” configuration sub-menu.
  • Widget Name – The name of the widget for which the parameter is a requirement.
  • Sequence – The order in which the parameters are displayed (left to right) across the top of the screen.
  • Refresh – Will refresh the changes made to reflect the new preferences for the view.
