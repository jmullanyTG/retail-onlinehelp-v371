Site Product Exclusion

The hierarchy used to populate the product dimension within this widget is determined by the following system policy: Site.ItemMaster.ExcludeFilterItemMasterHierarchyID

Site product exclusion is designed to help users manage inventory restrictions at a site level. Users are able to create restrictions by adding item master filters to the exclusion list or by selecting segments of the product hierarchy. The product hierarchy used by the site product exclusion is determined by a system policy settings.

  • Site Code – The site code of the selected site in the grid view.
  • Site Description – Description of the selected site.
  • Exclude Filter Type – The exclude filter type will determine the method for product exclusion. The selected filter type will “gray-out” the ineligible option; the eligibility will not overlap.
  • ItemMaster Hierarchy Level – If selected, the “Exclude Levels” field will become active. Users may then select segments of the product hierarchy for exclusion.
  • ItemMaster Filter Master – If selected, the “Filter Master ID” field will become active. Users may then select products for exclusion based on an item master filter.
  • Filter Master ID – If selected, users may select products for exclusion based on an item master filter.
  • Exclude Levels – If selected, users may select segments of the product hierarchy for exclusion

----

Only one product exclusion may be set per site at a time. As such, users must either create an appropriate item master filter to fulfil their requirements, or set the exclusion at an applicable level of the hierarchy.

ItemMaster Hierarchy Level

When ItemMaster hierarchy level is selected, users will be presented with a tree wizard which represents the selected hierarchy as set by the system policy.

Levels which should be excluded should be selected with a check mark.

ItemMaster FilterMaster

When ItemMaster filter master is selected, users will choose a filter using the filter chooser, which will then be applied.

Users should apply any desired changes by selecting “Apply” before exiting. If desired, the exclusions may be removed by deleting the entry using the “Delete” button.

------

Product Exclusion Sets

The Site Product Exclusion feature has been redesigned to provide greater flexibility and convenience. In previous versions, product exclusions had to be defined on a site-by-site basis and could be based on either a filter or hierarchy level. Now, you can create reusable product exclusion sets based on combinations of item filters and product hierarchy levels. Once created, a product exclusion set can be quickly assigned to any site in the Site Manager Grid.

New items

The following items have been added to support this feature:

  • Metadata: Two new fields are now available:
    • Product Exclusion Exists - Indicates whether or not a product exclusion set has been applied to the selected site.
    • Product Exclusion Set - Specifies the product exclusion set applied to the site.
  • Site Manager Detail tab:The Site Product Exclusion Set tab replaces the previous Site Product Exclusion tab.

Migration of existing site product exclusions

If upgrading from a previous version of Demand Management, any existing site product exclusions will be migrated to new product exclusion sets during the upgrade process. For more information, see the Demand Management Installation Guide.
