Site Placeholder Tab

You can create placeholder sites for use with Assortment Planning, Merchandise Financial Planning, Allocations, and Adaptive Allocations. These virtual sites can facilitate planning for stores that do not yet have mater data in your ERP system.

For example, in Allocations, you can use placeholder stores to create initial orders for a soon-to-open store and then reserve stock for the new store at the DC. In Assortment Planning and Merchandise Financial Planning, placeholder stores can be used to represent stores which will open in the future.

Note: Once master data for a store is imported, the placeholder store will map itself to the “real” store’s site code and all data related to the placeholder site will reference the new, permanent site.

Prerequisites

The following user rights are required to add, modify, and delete placeholder sites.

Right Group: Site Manager

  • Allow Deletion of Placeholder Sites
  • Allow Updating of Placeholder Sites

Site Placeholder Tab

Placeholder sites are created and managed in the Site Placeholder tab in the Site Manager’s Details pane. Once created there, they are accessible through site filters in supported modules. Placeholder sites can only be created for stores, not DCs or DC/Stores.

To create a placeholder site:

1. In the Site Placeholder tab, click New to add a new row to the Site Placeholder grid.

2. In the new row, enter details for the placeholder site.

3. Click Execute to save the new placeholder site in the system.

Placeholders and Order Management

JustEnough does not distinguish between orders for "real"and placeholder sites. The workflows behave the same and authorized orders to placeholder sites will be sent to your ERP system. If you do not want orders to placheolder sites to be sent to your ERP, set up a validation in your export process to block such orders.

Filtering Placeholder Sites

The Is Placeholder Site? field can be used in site filters to identify placeholder sites.
