Site Modeling

Users are able to create site model headers by using the site model options widget, then determine the effect of the models through the site model details widget.

Site Model Options

Site model options allows users to create site level supersessions by determining a start date, an end date, and an active status. Any supersession without a start date is assumed to go to the beginning of history within the system; any supersession without an end date is assumed to continue in perpetuity.

New entries may be made by selecting the “New” button. Existing entries may be deleted by selecting the “Deleted” button. Any changes made will require users to select “Apply” before exiting in order to take effect.

Site Model Details

Once the product level hierarchy as ben set for the system (through use of a system policy), the applicable product levels will appear as rows within the site model details grid.

For each line, users must select a model site, which will determine the set of historical sales to be copied, as well as a factor. The factor will determine the percentage of sales to be applied in the model. A factor of “1” will copy 100% of the sales quantity; a factor of “0.5” will copy 50% of the sales quantity.
