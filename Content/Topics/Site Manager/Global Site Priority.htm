Global Site Priority

Global site priority and priority group settings will alter the way in which the replenishment and allocations ordering engines prioritize stock distribution. The Replenish Engine ostensibly operates in two separate modes based on whether or not any values are present within priority group fields for any site.

Site Prioritization Logic

In a constrained DC stock situation, with more than one store requesting stock:

  • If priority groups are not present (meaning that the values are NULL, not set to 0) the stores are prioritized by their global site priority on a day-by-day basis when their stock requests are fulfilled.
  • If priority groups are present, stores are grouped by their respective priority group value, then ordered by the requested demand quantity on a day-by-day basis when their stock requests are fulfilled.
  • If all stores are within a single priority group, they are ordered by the requested demand quantity on a day-by-day basis when their stock requests are fulfilled. This can be used to create a “demand prioritization” version of the Replenish Engine.

G & Group Priority

Global and group priority is a globally applicable view and is not driven by the selected site in the top grid. Within this view, users will rank sites based on their priority as it relates to receiving stock in potential shortfall situations.

Priority group will allow users to “protect” a specific set of sites before referring to global priority to resolve shortfall situations.

  • Site Code – Site code for the store or DC.
  • Site Type – The site type (DC, Store, and DC/Store).
  • Site Description– The name of the site.
  • Global Priority – Unique, globally applied priority. Used to resolve shortfall situations. 
  • Priority Group – Associated priority group. Used to resolve shortfall situations.  Takes precedence over global priority.
