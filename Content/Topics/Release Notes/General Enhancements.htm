General Enhancements

The following enhancements affect multiple JustEnough modules.

Schedule-based order processing lead times

The inventory policy field, Order Processing Lead Time, specifies the number of days after a delivery is received to allow for processing before the ordered inventory can be considered "in stock" and available for sale/distribution. You can now choose whether that lead time is counted using calendar days (disregarding whether or not the business is open or closed on those days) or business days (using schedules to define days such as weekends and holidays that should not count as lead time).

If the lead time is counted using calendar days (existing behavior), the number of lead time days allotted to an order will always match the Order Policy Lead Time value set in the governing inventory policy. If the lead time is counted using business days (new behavior), the number of lead time days allotted to an order may exceed this value depending on the associated schedule (for example, closures for weekends or holidays). This enables better accuracy in determining the dates on which inventory will be available at a given location.

A number of Demand Management modules have been enhanced to support this feature. These enhancements are described below.

System policy

A new system policy, Order Processing Lead Time Type, has been added to the Engine:Replenish group. This new policy lets you define globally whether order processing lead time is based on calendar days (default behavior) or business days.

Engine: Replenish

Display Name System Name Description
Order Processing Lead Time Type Replenish.OrderProcessingLeadTimeType

Sets whether the Order Processing Lead Time value specified in an inventory policy is applied according to calendar days or business days.

  • 0 = Calendar days.
  • 1 = Business days. Business days are counted according to the Order Processing Lead Time Schedule set for a site or item. if schedules are defined for both a site and an item, the item schedule is used.

Default value = 0.

Metadata

New metadata has been added to support schedule-based order processing lead times.

Group Name Description
Inventory Schedules Order Processing Schedule

Sets the schedule to use when calculating the number of lead time days needed to process orders of specific items.

Site Schedules Order Processing Schedule Sets the schedule to use when calculating the number of lead time days needed to process orders at specific sites.

Item Manager

You can use the Order Processing Schedule field to specify the schedule to use for calculating the order processing lead time of specific items.

Note: Item-specific schedules override site-specific schedules.

  1. Add the Order Processing Schedule field to an inventory document.

  2. Select the schedule to use for an item in its Order Processing Schedule field.

Site Manager

You can use the Order Processing Schedule field to specify the schedule to use for calculating order processing lead times at specific sites.

Note: Item-specific schedules override site-specific schedules.

  1. Add the Order Processing Schedule field to a site document.

  2. Select the schedule to use for a site in its Order Processing Schedule field.

Order Manager

The order processing lead time for a site or item is used to calculate the order's "available date" at a DC or store. The Order Manager can now show the available dates calculated for an order. See Order Manager enhancements for more information and examples.

Dashboard widget configuration

You can now configure dashboard widgets using the DevExpresss Dashboard Designer. To get started, sImply right-click in the widget and select Setup.

New user right

The ability to configure dashboard widgets is controlled by a new user right, Configure Dashboard Widget.

Right Group Right Name Description
Widgets Configure Dashboard Widget Enables the Setup option when a user right-clicks in a dashboard widget.

Dashboard Designer

DevExpress Dashboard Designer provides an intuitive UI that facilitates data binding and shaping and layout design. Many of these normally complex tasks can be accomplished with a simple drag-and-drop operation, allowing you to start designing your dashboard immediately.
