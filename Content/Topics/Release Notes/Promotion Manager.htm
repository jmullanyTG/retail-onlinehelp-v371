Promotion Manager

The following promotion-related enhancements are now available.

Validation Failure Override

You can now override promotion validation failures in order to make those promotions acceptable and available for downstream processes and systems. The ability to override validation failures is controlled by a new user right and configurable at the validation set level.

This enhancement involves changes to a number of areas within Demand Management. These changes are described below.

New user right

A new right controls a user's ability to override validation failures.

Right Group Right Name Description
Promotions Allow Override of Validation Failures Enables the user to edit the Validation Failure Override checkbox in the Promotion Validation Log.

Workflow Manager integration

The new Allow Override of Validation Failures user right appears in the Status Permission Blacklists of all new promotion workflows, letting you control which users are able to override validation failures and when.

To add the new user right to the Status Permission Blacklists in an existing promotion workflow, edit the workflow and add the new user right to the list of workflow-controlled user rights.

Promotion Validation screen

The Promotion Validation screen in the Promotion Configuration window now includes an Allow Validation Override checkbox. To enable validation failure overrides for a validation set, mark the Allow Validation Override checkbox for that validation set.

Promotion Validation Log

You can view and override validation failures from within the Promotion Validation Log, which can be opened by double-clicking on a promotion's Validation Severity icon.

The Promotion Validation Log includes the following new columns.

Column Description
Validation Severity Displays an icon representing the promotion's current validation state (see Validation Severity Column section below).
Override This checkbox indicates whether the validation failure has been overridden. Users with the Allow Override of Validation Failures right can edit this checkbox.
  • Checked - This validation failure has been overridden.
  • Unchecked - This validation failure has not been overridden.
Override By The user who overrode the validation failure.
Override Date The timestamp of when the validation failure was overridden.

Validation Severity columns

Validation Severity columns display an icon representing the worst validation result found in an affected object (promotion header, price cluster, etc.). This enables users to understand what the "worst case scenario" may be before using that object. For example, the screenshot below shows the Available Promotions grid from the Campaigns and Events module. Ad coordinators can reference the Validation Severity column in this grid to understand at a glance what the validation status is for a promotion before deciding to place it on a page.

All Validation Severity columns across Demand Management now include "Reviewed" severity levels to indicate when validation failures have been reviewed and overridden, thereby giving users greater confidence knowing that those failures have been reviewed and accepted.

The following table shows the complete set of icons and severity levels sorted in order of severity, from least severe (on top) to most severe (at bottom).

Icon Severity Level Description
Validated This validation passed with no failures.
Information Reviewed The validation information provided has been reviewed and the failure is overridden.
Warning Reviewed This validation warning has been reviewed and the failure is overridden.
Error Reviewed This validation error has been reviewed and the failure is overridden.
Information Provided This validation failed with severity level, "Information Provided".
Warning Encountered This validation failed with severity level, "Warning".
Error Found This validation failed with severity level, "Error".

Validation Severity columns now appear in the following locations:

  • Promotions module
    • Promotions Home screen. Shows the worst validation severity found in each promotion listed.
    • Price cluster grids. Shows the worst validation severity found in each price cluster listed.
  • Campaigns and Events module
    • Available Promotions grids. Shows the worst validation severity found in each promotion listed.
    • Event Pricing Report. The Validation Status column displays the worst validation severity level found in each promotion listed.

Effects of overriding validation failures

When you click Save on the Promotion Validation Log after overriding any validation failures:

  • The Override By and Override Date fields are populated in the Promotion Validation Log.
  • Affected Validation Severity columns across Demand Management are updated to reflect the new "worst" validation severity. For example, if a price cluster contains a promotion with 4 "validated" validations and 1 error, and you just overrode the error, the validation severity for that promotion will change from "Error" to "Error Reviewed".

When a promotion with a validation failure override is recalculated, the override will persist if the validation fails again with the same severity as the previously overriden failure. For example, if you override a validation warning and, after recalculating, the same warning is generated, that warning will still be overriden. However, the override will be automatically removed if instead a different failure severity is generated (or if the validation succeeds).

Promotion exports

You can now export promotions with "overriden" validation errors, provided that all errors in the promotion have been overriden.

A new exportable field, PPH.[ValidationStatusIconValue], has been added with the following values:

  • NULL = Validated
  • 1 = Information Provided
  • 2 = Warning Encountered
  • 3 = Error Found
  • 9 = Information Reviewed
  • 10 = Warning Reviewed
  • 11 = Error Reviewed

Configuration Migration

When promoting validation set configurations to different environments, the new Allow Validation Override settings are included.
