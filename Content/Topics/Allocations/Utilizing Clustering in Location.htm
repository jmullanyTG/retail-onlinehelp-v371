Utilizing Clustering in Location Strategies

In order to utilize clusters in allocation strategies, users will need to create cluster sets and provide an effective date for the cluster set snapshot that applies to the period in which the allocation strategy will occur.

In the Cluster site manager, create a snapshot of the cluster set and give it an effective date.

Once cluster sets have been created and snapshotted, users can now select from those cluster sets from within the allocation strategy screen. Users may have to add the “Cluster Sets” column to the allocation strategy view if it is not immediately visible (choose columns).

Users have access to select specific clusters from the cluster set. Selecting the Cluster Sets column will now provide you an ellipses which will pop out the cluster selection screen

 

The pop-out cluster sets screen will now allow you to select specific clusters within each cluster set.

You will be able to select clusters from different cluster sets but note that there can be no duplication of sites. The system will check for duplicate sites selected and notify the user if duplicates are detected. This would only happen if two different cluster sets were formed on the same subset of stores.

Intersection of Location Strategy and Clusters

It is very important to note that cluster sets will intersect with the selected location strategy if any demand strategy but the “Rate of Sale by Grade” is selected. This is important to note because users could mistakenly exclude store that they intended to. There are two ways users should consider using this functionality

  1. When using clusters to drive location eligibility in an allocation strategy, ensure the location strategy is set to all locations to ensure that the intersection of clusters and location strategy covers all locations.
  2. The alternative is to drive a meaningful intersect for example; I would like A, B and C clusters that are also cold stores. In this case you would select A, B and C clusters as well as a filter that represents all stores attributed as cold weather stores. The intersection of these lists would provide the desired result.

Note that the overrides in the location strategy column still work and return with the information of which cluster set and classification (cluster) a store belongs to for the effective life of the allocation.

Cluster Selection for Rate of Sale by Grade

In the case when clusters are selected to filter eligible stores for an allocation strategy and the rate of sale by cluster is selected as a demand type, the system will automatically set the location strategy for the user to “Cluster Based All Sites”. The reason for this is that all sites must be considered to accurately spread a cluster rate of sale to the appropriate locations even if those locations.

Demand Signal

A demand signal must be present to complete the strategy configuration. This may come from many possible sources and will vary from item to item. Examples of sources may include total sales demand, rate of sale, temporal plan values, and others.
