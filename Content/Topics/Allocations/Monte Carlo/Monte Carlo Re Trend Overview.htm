Monte Carlo Re-Trend Overview

The re-trending of forecasts is an important step to correctly managing the continued allocation of products as they transition from a pre-season plan to in-season. As products begin to sell, their sales patterns will inevitably trend away from the initial plan (sometimes in the positive, sometimes in the negative) and the included re-trend mechanisms within JustEnough allow users to account for the trends in their sales and form an amended go-forward sales plan against which continued allocations may be structured.

With the initial launch of the JustEnough Allocations solution, a PID mechanism was included in order to provide a reactive result set based on potentially limited data. With the introduction of the Monte Carlo simulation re-trend mechanism, organizations now have additional tools at their disposal in order to provide flexibility when dealing with irregular sales patterns.

Note: This is currently a beta feature.

Essential Business Concepts

Using historical demand for an item, we try to calculate a distribution for the observed demand values using Monte Carlo simulation.  From there we calculate a dynamic confidence level based on the variability of the item’s demand and the number of observations available.

Using the confidence level and the demand distribution we calculate a total demand value and compare that to the total forecast over the same time frame.

A re-trend factor is then calculated based on a desired demand satisfaction percentage.

There are six inputs into the model:

Input

Description

Min multiplication factor (min mf)

Lower bound for re-trend factor

Max multiplication factor (max mf)

Upper bound for re-trend factor

Evaluation window

The max number of historic demand periods that will be used to calculate historic demand statistics

Forecast Period

The number of forecast periods that will be used to calculate total forecast value, if value<0 past forecast periods will be used, if value>0 future forecast periods will be used (relative to the current date). It’s recommended to keep value greater than 0 with a suggested default of 12 weeks/periods

Min observations

Items where total observations < min observations will be penalized when calculating a final confidence level.

Needs to be greater than Evaluation window.

Values between 4 and 12 weeks are preferable, recommended default is 6 weeks

Number of Simulations

The number of Monte Carlo simulations that will be run to generate a demand distribution. More simulations will give a more continuous distribution but will increase calculation time.

Recommended default value is 100 simulations (in testing 100 simulations have been sufficient for producing consistent results)

Desired demand satisfaction percentage

The percentage of total demand that needs to be satisfied i.e. if total demand = 40, and Desired demand satisfaction percentage=95% total forecast needs to be equal to at least 40*0.95=38.

Higher values (90% and upwards) are recommended for more consistent results.

Recommended default value is 100%
