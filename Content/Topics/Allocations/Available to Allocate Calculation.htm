Available to Allocate Calculation

The available to allocate calculation is used in many places throughout the module. It is a dynamic calculation with several parameters, the inclusion of which will affect how the calculation is accomplished.

  • @bIncludeStockOnHand BIT = 1 --Should the StockOnHand column be included? Read from tbl_InventoryLevel
  • @bIncludeOnOrder BIT = 1 --Should entries from SupplierOrderLine be included?
  • @bIncludePrepacks BIT = 1 --Should stock in prepacks be included? If there are prepacks defined, it will go to the prepacks, then go to inventory line of the prepacks
  • @bIncludeReserved BIT = 0 --Should the Reserved column be accounted for? Stock reserved is currently subtracted from StockOnHand in tbl_InventoryLevel; if selected, it will be added back in.
  • @bIncludePending BIT = 1 --Should the Pending column be accounted for?--subtracted if included
  • @bIncludeSuperceded BIT = 1 --Should the Superseded column be accounted for? â€“ added if included
  • @iTodayDays INT = NULL --Number of days from "Today" to be considered as available (uses sys policy allocate.availableorderswindow if null)
  • @dToday DATETIME = NULL --defines "Today", uses actual today if null
  • @bPrepackOnly BIT = 0 --Return only prepack stock ignoring all others
  • @iLateDays INT = NULL --How many days back from today to look at late orders and include them (uses sys policy allocate.lateorderswindow
  • if null
  • @bOnOrderOnly BIT = 0 --Return only on order stock ignoring all others
  • @cDestinationTableName NVARCHAR(128) = NULL -- This will Change the Stored Procedure Data exchange mechanism. Not relevant to the calculation.
