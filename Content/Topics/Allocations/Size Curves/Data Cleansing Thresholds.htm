Data Cleansing Thresholds

  • Percent Covered: this threshold allows the user to specify a percentage of sizes that must have been in-stock at a site on a given period for a variant for the corresponding sales from that period to be included in the size curve calculation for that site; the threshold is applied as follows:

  • First, because not all sizes carry equal importance at a given site, the system determines the relative importance of each size present in the selected Size Range for each site by summing the uncleansed sales by size for all items in the selected Filter and date range then dividing by the sum of sales for all sizes.

  • Next, each period in the selected date range is evaluated against the set Percent Covered threshold. The % contribution of all sizes in stock for each variant at that site are summed and compared to the Percent Covered threshold – if the sum value is equal or greater, the sales for that variant/period are included in the size curve calculation for that site; if not, they are removed.

  • Weeks Covered: this threshold allows the user to specify a minimum number of periods that a variant must have been in-stock at a given site for its sales to be included in the size curve calculation for that site; when paired with a Percent Covered threshold, this threshold is applied after any periods that did not pass the Percent Covered threshold are removed; the threshold is applied as follows:

  • The system evaluates each period for each variant to see if it was acceptably in-stock at the site. If no Percent Covered threshold is specified, the system simply checks if there was stock on-hand in any size for each period .

  • In-stock periods are then counted for each variant and compared to the Weeks Covered threshold – variants with counts equal to or greater than the threshold are included in the size curve calculation for that site; variants with lower counts are removed.
