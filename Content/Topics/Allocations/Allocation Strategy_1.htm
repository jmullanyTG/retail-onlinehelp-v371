Allocation Strategy

The allocation strategy screen is the main workflow of the allocations module. Once a set of strategies has been returned to the screen, users will select drivers from the product model, location strategy, and size curve sections in order to determine the appropriate method for distributing stock. Several fields exist within the allocation strategy screen which will only be applicable to certain strategy types; as such, it is encouraged to use saved layouts in order to streamline the workflow.

  • Calculate Allocation – When checked, upon action of the “Save” button, the allocation strategy will be recalculated based on any changes made.
  • Rollup – The rollup view button embedded within the allocation strategy screen will launch a context sensitive rollup view containing information on the highlighted strategy. This rollup view will present a more narrow focus when compared to the allocation rollup screen.
  • Active Flag – Checked if the strategy is currently active.
  • Allocate Strategy – A unique identifier for the strategy line. Any active product groups will also be assigned a unique allocate strategy ID.
  • Code – The identifying code for the level ID associated with the strategy. This code will be the item master group code of the hierarchal level defined in the allocation hierarchy.
  • Description – A description accompanying the code. This description will be the item master group description of the hierarchal level defined in the allocation hierarchy.
  • Allocation Strategy Selection – This will define the strategy bucket used to classify the strategy. Typically, these strategy selections will be determined by product lifecycle status. This field will drive which exceptions are triggered, how some of the settings are applied, and which process (Allocations or Replenishment) manages the item.
  • New Item – An item without sales history, or with a minimal volume of sales history data which has been deemed insufficient to drive allocation measures. The “new item” strategy selection may be used to drive initial load supply chains (e.g. circumventing regional distribution centers). The individual supply chain mapping configuration for the different strategy selection values is maintained in the Supply Chain Map Configuration maintenance view). New Items are managed by the Allocations engine.
  • Ongoing Item – An item which has multiple points of sales history data. This item may have a re-trending method applied to the sales projection. Ongoing items are managed by the Allocations engine.
  • Replenishment – An item which no longer needs to be managed at the strategy level and may be put on an automated replenishment strategy. Replenishment items are managed by the Replenishment engine, which takes over responsibility from the Allocations engine for executing processes and providing item-related information. When this option is selected and saved, the strategy line becomes read-only.
  • Item Exceptions – Any and all item level exceptions which have been triggered will appear in this field. Multiple exceptions may be viewed in the exception detail view.
  • Product Models – The product models field will display the selected model or models. These may be adjusted by clicking on the ellipses in order to launch the product model selection wizard.
  • Size Curve Assignment – The size curve assignment filed will display the selected size curve for the strategy. A listing of available size curves is presented as part of a drop down menu.
  • Presentation Priority – Presentation priority allows users to define how presentation quantities are treated within an allocation strategy.
  • Demand / Presentation – Will prioritize the planned demand for all locations over presentation minimums.
  • Presentation / Demand – Will fulfil minimum presentation requirements for all locations before addressing demand.
  • Order Multiple / Presentation / Demand – Will prioritize coverage to ensure all locations receive units, then will fulfill minimum presentation requirements for all locations, then will address demand. If there is not enough stock available to cover all stores, then global site priority is used to determine which stores get coverage.
  • Demand Type – Demand type will determine the driver for the demand signal. This setting will also determine which fields within the allocation strategy are and are not relevant to shaping the demand signal.
  • Rate of Sale – For use with rate of sale type plans.
  • Total Sales Quantity – Will set a total quantity of projected sales based on the sales plan.
  • Total Push – Will allocate all available inventory in the selected inventory sources at one time.  A specific quantity to push can be designated by entering a value in the “Push Qty” field. Total push demand type will prioritize the movement of stock from the DC and may violate store level constraints or ideal stock quantities.
  • Rate of Sale by Grade – For users with access to the JustEnough Rate of Sale Calculator module, a rate of sale value by cluster grade may be applied to individual locations. A location strategy of “cluster based” must be specified, and site clusters must be selected to define the location strategy for this demand type. Included grades by cluster set must be configured within the allocate cluster rate of sale view.
  • Re-Trend Method - The method to use for obtaining a retrend forecast for the allocation strategy.
    • Default - Uses the option set in the Default Forecast Retrend Method To Use system policy.
    • PID - Use the standard retrend calculation to create a retrend forecast.
    • Monte Carlo - Use the Monte Carlo retrend method to create a retrend forecast.
    • Demand Forecast - Use a generated inventory-level demand forecast as the retrend forecast. Requires the Demand Forecasting module and an existing demand forecast. (Demand forecasts are not generated as part of Allocation execution.)
  • Use Re-Trend – If selected, the allocation strategy will recalculate the allocation strategy quantities using the JustEnough re-trend calculator.
  • Rate of Sale Override – A user override value for the strategy rate of sale.
  • Demand Rate of Sale – The rate of sale currently being applied to the phase period (defined as the period between the phase-in date or current date, and the phase-out date).
  • Total Sales Demand – Quantity of sales units defined as demand for strategies utilizing total sales quantity as the demand type.
  • Actual Rate of Sale – Actualized rate of sale based on sales history.
  • Rate of Sale Variance – Percentage difference between the demand rate of sale and the actual rate of sale.
  • Push Qty – A specific quantity of stock with which to execute a “total push” demand type allocation.
  • Location Strategy – The location strategy will contain a group of sites which are the intended destination for the allocated stock. This field will display the selected location strategy for the allocation.
  • Cluster Sets – For users with the JustEnough Cluster Site Manager module, location strategies may be defined by selecting specific site grades within cluster sets (cluster sets must be configured for use with Allocations). A location strategy value of “Cluster based all sites” must be set in order to use cluster sets to define the effective location strategy. Location exceptions are still applicable.
  • Eligible Channel – A configurable named site group, or the explicit channel attribute. Additional site groups may be returned as required. Informational only.
  • Site Count – Site count lists the number of sites (stores) contained within the location strategy.
  • Safety Stock – The minimum required quantity of stock to allocate. Sometimes referred to as presentation stock.
  • Max Stock – A maximum stock constraint for each site contained within the location strategy. Set for the allocation strategy level (typically style-color).
  • Min Stock per Size – Item master level safety stock.
  • Phase In Date – The date on which the items will be set or placed on sale within the stores. The phase-in date is the starting anchor point for the demand signal.
  • Markdown Date – The planned date on which the product is to be marked down.
  • Phase Out Date – The end-of-life or clearance date of the product.
  • Weeks of Cover – The number of weeks of demand for which stock will be allocated.
  • Weeks of Supply– The effective replenishment cycle for this product. Overrides the Allocation:Replenish Cycle system policy value.
  • Inventory Source Rule – Defines the priority for the sourcing of stock.
  • No Priority – No source priority should be applied. This applies to selected DC and PO stock restrictions as well (i.e. absolutely no restrictions are placed).
  • PO Only – Only use purchase orders.
  • PO then WH – Source from purchase orders first, then warehouse stock.
  • Warehouse Only – Only use warehouse stock.
  • WH then PO – Source from warehouse stock first, then purchase orders.
  • PH Only – Use only placeholder stock.
  • PH + Available – Use placeholder stock, then any available stock.
  • Primary WH Only – The primary warehouse only flag will restrict the sourcing to the only the primary warehouse as defined by the supply chain.
  • Strategy Stock – Will display the available stock count given the strategy parameters. Uses the available to allocate calculation and is inclusive of any stock availability restrictions which have been applied and saved.
  • Placeholder Stock – Placeholder stock is an editable field that allows for allocators to specify a stock quantity that is expected to be ordered for allocation. This will allow allocators to determine the result of an allocation based on a mock quantity to ensure the sizing and quantity is sufficient to meet the allocation constraints of a given strategy.
  • Direct Ship Functionality – Users wishing to execute orders based on opportunity buys may create direct supplier orders by using placeholder stock. When placeholder stock is selected as an inventory source and users have keyed in a placeholder quantity, orders will be generated as direct supplier orders to the sites contained in the location strategy. These order recommendations may be approved and actioned upon within Order Manager when selecting a “from strategy” allocation order.
  • Sellthrough Target – Sellthrough target is an input which designates a planned sellthrough percentage.
  • Holdback % - Holdback percentage will define a quantity of stock to reserve in the warehouse or distribution center.
  • Holdback Quantity – Users may specify a holdback quantity instead of a holdback percentage if desired. The holdback quantity is applied at a variant level allowing allocators to reserve a fixed quantity of stock in reserve for new store openings or other related purposes. Holdback quantity is not applicable to product groups.
  • Strategy State – Determines how the strategy is managed and if it is visible to users.
  • Active – Default state. The strategy may be actively managed and allocations may be created using the strategy. Type 1.
  • Hidden – Strategies with no actionable stock may be set to hidden to ease navigation. Type 2.
  • Inactive – User driven status to deactivate a strategy. Type 3.
  • Deleted – System setting for inactive strategies which may be removed from JustEnough. Type 4.
  • Exception Status – Lists any strategy level exceptions which may prevent calculation or execution. If there are no blocking exceptions, then Ready to Allocate displays. To view exceptions, click within the Exception Status field and then click View to launch the Allocations Strategy Exceptions dialog.
  • Strategy State Management – Determines if users are able to action on the strategy state field.
  • Open – No restrictions (may be updated based on system policies and execution).
  • Manual – Only users are able to change the field.
  • Sales Curve Basis – Determines the type of sales curve which will be used to shape the demand signal of the allocation strategy.
  • Last Year – Curve based on last year’s historical sales. This information will come from the JustEnough curves module.
  • Imported – Use an imported sales curve.
  • Imported Plan – Use an imported temporal plan.
  • Manual Selection – Allows for the manual selection of a sales curve as generated in the JustEnough Curves Module. If selected, the curve must be specified within the sales curve selection field.
  • Sales Curve Selection – Displays the curves available for manual selection. The selected curve will only be applied to the demand forecast if the sales plan basis is set to “manual selection”.
  • Temporal Plan – A plan with a time period based strategy which is used to define the sales curve. Effective if “Imported Plan” is selected in in the sales curve type field.
  • Allow Reservation – Selecting Allow Reservation ensures that all stores that were set up to have reservation quantities will receive their reserved stock quantities.
