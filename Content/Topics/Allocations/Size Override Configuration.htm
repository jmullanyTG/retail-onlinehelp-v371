Size Override Configuration

Users will create a new size curve with name, size range, etc. and choose if the size curve is calculated from historical data, or allows overrides.

  1. If size range selected has default contribution specified in tbl_Size_SizeRange_Link, the DefaultContribution value will populate the Contribution field on tbl_Size_SizeCurve_Quantity field.
  1. If bad default values are provided, (rebalance proc) is called to distribute the values properly with respective weights.
  • Ex: if only one default value is provided of 0.75, that value becomes 1 and the rest of the sizes get 0.
  1. If good default values are provided, Contribution field is populated.
  1. If size range selected does not have default contribution values provided, the Contribution field will be split evenly between the count of sizes in the size range.
  2. All sizes at all sites will be marked as IsEligible = 1 on creation.
  3. Calculate option calls (calculate proc) and Contribution value is updated based on history, given dates or a Last X Days.
  4. Override option allows bottom size curve detail grid to be editable by user input.

tbl_Size_SizeCurve_Quantity is populated for all sizes and sites; entries are only destroyed if the size range is changed during configuration to accommodate for new sizes selected in the size curve.

Calculated checkbox updates all default related fields; IsEligible is set back to 1 as a default and the newly calculated value is updated in the Contribution field on tbl_Size_SizeCurve_Quantity.
