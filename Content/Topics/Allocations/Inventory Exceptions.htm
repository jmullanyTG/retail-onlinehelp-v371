Calculation

New Item

Item Recording Selling

Item is defined as "New Item" but has actualized selling of more than 3 periods in a system policy set number of stores

If an Inventory has 3 or more sales history by type sales quantity that are > 0 . Will only be displayed if that item is selling at more than the set number of stores

New Item

Purchase Order, No Allocation

Item is defined as "New Item" and has Purchase Orders and no SOH in stores, and no orders have been generated against the variant

If an inventory has an order placed/pending with an estimated receipt date between phase in and phase out but no allocations engine orders and no inventory SOH

New Item

Stock on Hand, No Sales

Item is defined as "New Item" and has Warehouse stock and no SOH in stores, and no units in transit.

If a DC/Warehouse SOH > 0, but inventory SOH <= 0 and there is no supplier order in placed or pending for inventory that have order quantity > 0

New Item

NDC Missing Cutoff

Item is defined as "New Item", with no SOH in warehouse or stores, with Purchase Order that has an NDC date greater than current set week minus lead time for any stores using destination DC as Primary DC.

If a DC/Warehouse SOH <= 0, and Inventory SOH <= 0, then see if DC Default Lead Time + DC's Supplier Order estimate date [range of stale date to phase in date] will exceed Phase In Date (with exceptions taken into account). Also flag if there are no orders

Ongoing Allocations

Out of Stock

Item is defined as "Ongoing Allocation" where there is not enough inventory to fill projected weeks of cover, where any stores will be at zero units.

Allocate Engine Projection Lost Sales between the projection dates that fall between today and the weeks of cover we should look ahead (taking into account exceptions)

Ongoing Allocations

Below Min SOH

Item is defined as "Ongoing Allocation" where there is not enough inventory to fill projected weeks of cover, where any stores will be below item defined minimum SOH.

Allocate Engine Projection EOPSafetyStock > EOPSOH between the projection dates that fall between today and the weeks of cover we should look ahead (taking into account exceptions)

Ongoing Allocations

Markdown Upcoming

Item is defined as "Ongoing Allocation" and Markdown Date is within the user defined weeks of cover, or is the week after current week.

There is a markdown date that comes before/in our range of inspection (getdate() + 1 week OR getdate() + weeks of coverage [if defined])

Ongoing Allocations

Phase-out Upcoming

Item is defined as "Ongoing Allocation" and Phase Out Date is within the user defined weeks of cover, or is the week after current week.

There is a phase out date that comes before/in our range of inspection (getdate() + 1 week OR getdate() + weeks of coverage [if defined])

Ongoing Allocations

Incoming MD, Below Sellthrough %

Item is defined as "Ongoing Allocation" where the projected ST% at the item defined MD date is less than either the item defined ST% or a user configured default ST target %.

Markdown Date and Sellthrough % defined, and sales (from sales history)/received (from supplier receipts) is less than sell thru% before markdown date (range is phase in date to markdown date taking exceptions into account)

Ongoing Allocations

Purchase Order, No Allocation

Item is defined as "Ongoing Allocation" and Purchase Order has not been allocated, and Purchase Order receipt date is within WOC.

Inventory PO within the range of days of cover and placed/pending, but no Source Site Allocation Engine Order's (quantified as quantities <= 0)

Ongoing Allocations

Aged DC Inventory

Item is defined as "Ongoing Allocation" and Warehouse SOH is greater than zero, and no reduction in DC stock for user configured number of periods (default 13 weeks)

Warehouse SOH is > 0 and there are no sales quantities in Sales history > 0 for the last 13 weeks history on Inventory for 13 weeks

Ongoing Allocations

Flow Expected, No Receipt

Item is defined as "Ongoing Allocation" and Purchase Order exists with receipt date before current week, no actualized receipt.

Supplier order (placed/pending/delivered) estimated receipt is < GETDATE() but no receive quantity > 0 in supplier receipt

Replenishment

PO Not Approved

Item is defined as "Replenishment" and PO submitted to vendor has not been approved (Exception starts day after PO transmitted)

For now a PO Not approved = Supplier Order is placed and the purchase order date from Supplier Header falls between the begin and end date for calculations [i.e. 52 weeks of coverage]

Replenishment

Order Not Released From Vendor

Item is defined as "Replenishment" and PO has not been released from vendor, and PO Ship date is less than current date.

Supplier order that is still in placed but has an estimated date before current date

Replenishment

NDC Missing Cutoff

Item is defined as "Replenishment" with Purchase Order that has an NDC date greater than current projected receipt week minus lead time for any stores using destination DC as Primary DC.

DC Default Lead Time + DC's Supplier Order estimate date [range of stale date to phase in date] will exceed Phase In Date (with exceptions taken into account)

Replenishment

Inventory Unavailable to Allocate

Item is defined as "Replenishment" where total Warehouse + PO SOH is greater than projected demand, but Actionable Inventory is less than projected demand for WOC.

Projected date from Allocate Engine Projection falls within GETDATE() + Exception Weeks Of Cover & Action Inventory (from proc/ SOH + Inbound - Stock Reserved - Stock Pending) < Projected Demand in same date range

Engine Projection Lost Sales between the projection dates that fall between today and the weeks we should look ahead (taking into account exceptions)

Replenishment

Below Min SOH

Item is defined as "Replenishment" where there is not enough inventory to fill projected weeks of cover, where any stores will be below item defined minimum SOH.

Allocate Engine Projection EOPSafetyStock > EOPSOH between the projection dates that fall between today and the weeks of cover we should look ahead (taking into account exceptions)

Replenishment

Flow Expected, No Receipt

Item is defined as "Replenishment" and Purchase Order exists with receipt date before current week, no actualized receipt.

Supplier order (placed/pending/delivered) estimated receipt is < GETDATE() but no receive quantity > 0 in supplier receipt
