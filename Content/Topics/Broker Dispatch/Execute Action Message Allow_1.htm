Execute Action Message Allow Singleton Execution On Distinct Action

The Execute Action Messages Allow Singleton Execution On Distinct Action is a strategy that allow only allows one singleton instance of an Action Message to be executed if Action Messages have identical Actions specified. This is the strictest of all the schemes in terms of granular control, ensuring that only the minimal amount of Action Messages are being processed. Note if there is an identical Action for an Action Message currently executed and a new Action Message with identical Action arrives for processing the new Action Message will be discarded and this discarding of the Action Message will be reported.

Consider the following scenario: Action Message A which is not identical to Action Message B but do have the same Action while Action Message C is not identical to A and B and does not have the same Action. All Action Messages A, B and C arrive at the Dispatch Broker for processing. In addition assume that they messages do arrive in order A, B and C. What would happen under this scenario for Execute Action Messages Allow Singleton Execution On Distinct Action strategy is that A and C would be processed immediately would be processed immediately while B would be discarded or thrown away.

In order to use this execution scheme one needs to compose an Action Message where the Action is the following constant string “DISPATCH_SERVICE_SEND_EXECUTE_ACTION_SINGLETON_DISTINCT_ACTION”. The example provided below is a simple action message to Ping the JEBDS to determine is the service is currently running using the Execute Action Messages Queue On Distinct Action strategy:

<ActionMessage>

<Sender>

<string>DATABASE</string>

</Sender>

<Recipients>

<ArrayOfString xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">

<string>DISPATCH_SERVICE</string>

</ArrayOfString>

</Recipients>

<Action>

<string>DISPATCH_SERVICE_SEND_EXECUTE_ACTION_SINGLETON_DISTINCT_ACTION</string>

</Action>

<ActionParameters.Count>

<int>2</int>

</ActionParameters.Count>

<ActionParameters>

<ActionParameters.Item>

<ActionParameters.Item.Key>

<string>DISPATCH_SERVICE_EXECUTE_ACTION_ACTION_MESSAGE</string>

</ActionParameters.Item.Key>

<ActionParameters.Item.ValueType>

<string>JustEnough.Common.Action.ActionMessage, JustEnough.Common.Action</string>

</ActionParameters.Item.ValueType>

<ActionParameters.Item.Value>

<ActionMessage>

<Sender>

<string>DATABASE</string>

</Sender>

<Recipients>

<ArrayOfString xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" />

</Recipients>

<Action>

<string>COMMON_SEND_PING</string>

</Action>

<ActionParameters.Count>

<int>0</int>

</ActionParameters.Count>

<ActionParameters />

</ActionMessage>

</ActionParameters.Item.Value>

</ActionParameters.Item>

<ActionParameters.Item>

<ActionParameters.Item.Key>

<string>DISPATCH_SERVICE_EXECUTE_ACTION_ACTION_MESSAGE_CALL_IDENTIFICATION_TOKEN</string>

</ActionParameters.Item.Key>

<ActionParameters.Item.ValueType>

<string>System.String, mscorlib</string>

</ActionParameters.Item.ValueType>

<ActionParameters.Item.Value>

<string>33CAECFF-2419-4E78-A4BD-9612953EC2D0</string>

</ActionParameters.Item.Value>

</ActionParameters.Item>

</ActionParameters>

</ActionMessage>
