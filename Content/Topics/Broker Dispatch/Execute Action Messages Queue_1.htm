Execute Action Messages Queue On Distinct Action

The Execute Action Messages Queue On Distinct Action, is a strategy that queues Action Messages if they have identical Actions. This strategy optimizes resource utilization by queuing Action Messages that have the same Action defined. One thing to consider is that the queue will preserve the execution order for all action messages for each queue bucket.

Consider the following scenario: Action Message A, which is not identical to Action Message B, but do have the same Action, while Action Message C is not identical to A and B and does not have the same Action. All Action Messages A, B and C arrive at the Dispatch Broker for processing. In addition assume that the messages do arrive in order A, B and C. What would happen under this scenario for Execute Action Messages Queue On Distinct Action strategy is that A and C would be processed immediately while B would be queued to be processed after A has completed.

In order to use this execution scheme, one needs to compose an Action Message where the Action is the following constant string “DISPATCH_SERVICE_SEND_EXECUTE_ACTION_QUEUE_DISTINCT_ACTION”. The example provided below is a simple action message to ping the JEBDS to determine is the service is currently running using the Execute Action Messages Queue On Distinct Action strategy:

<ActionMessage>

<Sender>

<string>DATABASE</string>

</Sender>

<Recipients>

<ArrayOfString xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">

<string>DISPATCH_SERVICE</string>

</ArrayOfString>

</Recipients>

<Action>

<string>DISPATCH_SERVICE_SEND_EXECUTE_ACTION_QUEUE_DISTINCT_ACTION </string>

</Action>

<ActionParameters.Count>

<int>2</int>

</ActionParameters.Count>

<ActionParameters>

<ActionParameters.Item>

<ActionParameters.Item.Key>

<string>DISPATCH_SERVICE_EXECUTE_ACTION_ACTION_MESSAGE</string>

</ActionParameters.Item.Key>

<ActionParameters.Item.ValueType>

<string>JustEnough.Common.Action.ActionMessage, JustEnough.Common.Action</string>

</ActionParameters.Item.ValueType>

<ActionParameters.Item.Value>

<ActionMessage>

<Sender>

<string>DATABASE</string>

</Sender>

<Recipients>

<ArrayOfString xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" />

</Recipients>

<Action>

<string>COMMON_SEND_PING</string>

</Action>

<ActionParameters.Count>

<int>0</int>

</ActionParameters.Count>

<ActionParameters />

</ActionMessage>

</ActionParameters.Item.Value>

</ActionParameters.Item>

<ActionParameters.Item>

<ActionParameters.Item.Key>

<string>DISPATCH_SERVICE_EXECUTE_ACTION_ACTION_MESSAGE_CALL_IDENTIFICATION_TOKEN</string>

</ActionParameters.Item.Key>

<ActionParameters.Item.ValueType>

<string>System.String, mscorlib</string>

</ActionParameters.Item.ValueType>

<ActionParameters.Item.Value>

<string>33CAECFF-2419-4E78-A4BD-9612953EC2D0</string>

</ActionParameters.Item.Value>

</ActionParameters.Item>

</ActionParameters>

</ActionMessage>
