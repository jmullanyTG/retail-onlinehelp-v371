JustEnough Broker Dispatch Service

The JustEnough Broker Dispatch Service (JEBDS) is a windows service that is comprised of a collection of contained services and sub-systems. This allows JEBDS to be able to achieve its two primary requirements of providing generic bi-directional endpoints between the JustEnough Database and JustEnough Developed Endpoints and being able to execute jobs or tasks at a scheduled time, whilst providing granular control on how resources are used for each Action Message execution.

As can be seen from Annexure A, JEBDS is a standalone windows service that communicates directly with our BLL web services where it augments all of our existing methods and services. To place an analogy on what JEBDS is similar to, one has to look no further than the Supply Chain or Distribution Channels of the Postal Service. Whilst the Push Service is considered the heart of the JustEnough Action Message system with very limited routing capabilities (it requires listeners to be subscribed to the Push Service), JEBDS provides the much needed general routing (it is a listener subscribed to the Push Service) and resource management required for Action Message processing where end users are not necessarily interacted with. Thus JEBDS supplies the required infrastructure for backend process to be able to execute jobs or tasks and communicate or interact with one each other in a very scalable and asynchronous manner.

If one is to examine Annexure B, one can observe the various services and sub-systems JEBDS is comprised of and the flow of how these systems interact with each other. JEBDS contains the following services and sub-systems: Logging Service, Scheduling Service, Dispatch Accessor, Push Service Accessor, Job Manager Accessor and Action Message Coordinator.
