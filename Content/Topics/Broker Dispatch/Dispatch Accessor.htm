Dispatch Accessor

The Dispatch Accessor provides a generic endpoint for JEBDS to be able to pass an Action Message on to another System or Engine without JEBDS needing to know how to interpret the Action Message or the implementation specifics. The Dispatch Accessor essentially completes the routing chain process by passing on the Action Message to the implemented handling method endpoint, which in term will un-package the Action Message so that it can continue processing or raise an error if the Action Message is malformed.
