Forecast Details

The detail information for the forecast may be returned to the forecast viewer if users wish to scrutinize the forecast beyond the method fit.

Forecast

The forecast information will display information only related to the forecast method currently applied.

  • History Start Date – The first (oldest) time period of demand history data for the inventory item used by the forecasting engine. Default is approximately two years previous to the forecast start date.
  • Forecast Start Date – The date that the forecast tournament was last run on the item, or the date of the last saved forecast.
  • Accuracy – The accuracy field is a placeholder for configurable accuracy measurement displays. It does not populate by default.
  • Lock Expiration – If the forecast methods have been locked, the expiration date for this lock will be displayed here.
  • Forecast Message – If the forecast method was disqualified from the tournament, the reason for disqualification will be displayed here, in addition to the forecast method drop down list.
  • Forecast Parameters – The forecast parameters displayed will vary depending on the method applied:

Forecast Details

The details section lists information related to the inventory item.

  • Seasonality – Seasonality (different from Seasonality Constant) is a value assigned to each item to specify the number of consecutive buckets to consider when determining seasonality trends.  If it is set to 12, a seasonal trend will only qualify if it repeats itself in two sets of consecutive 12-bucket periods.
  • Creation Date – The date on which the inventory item was added to the database.
  • Supersession – If any supersessions are lined to the item, details will display here. More information may be found on supersessions within the Supersession module.
