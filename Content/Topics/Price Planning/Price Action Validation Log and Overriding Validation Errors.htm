Price Action Validation Log and Overriding Validation Errors

Double left-mouse clicking a Validation icon in any Price Manager grid (Price Action Header, Summary, Item and Item Location) will display the Validation Log window. The Validation Log provides messages based on failed validations configured in the Price Strategy. Depending on where you initialize the Validation Log, the system will filter the results displayed, as appropriate.

 

Overriding a Validation Error forces the system to downgrade the Error to a Warning. To override, mark the checkbox adjacent to the Validation Error(s) and click Ok. This will cause the system to mark the product as well as any grouping above it as ‘Calculation Required’. During any recalculation, the system will compare any future validation failures against configured overrides and if found will down grade to a Warning. If any recalculation causes the validation to pass successfully the override will be dismissed.
