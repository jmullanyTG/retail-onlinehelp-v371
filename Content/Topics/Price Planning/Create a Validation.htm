Create a Validation

Within the Price Manager Configuration Validation screen, click New to create a new Calculation or Calculation Rule, enter the required fields and click Save.

Field

Description

Validation Name (Required)

Enter a descriptive label to summarize the purpose of the Calculation or Rule.

Validation Description (Optional)

Enter a summary of the Calculation or Rule.

Calculation Method (Optional)

Select from a list of preconfigured database stored procedure for more complex mathematical formulas or validations. Selection in the ‘method’ field will override any entry in the ‘formula’ field.

Validation Method (Required/Optional)

Select from a list of preconfigured database stored procedure for more complex mathematical formulas or validations. Selection in the ‘method’ field will override any entry in the ‘formula’ field.

Validation Formula (Required/Optional)

Enter the mathematical expression for the system to evaluate. Configured variables can be found in the right panel.

  • Validation Examples
  • Check to see if Cost1 is NULL, $0.00 or $0.01
  • ISNULL([Variable1],Replacement Value)<.02
  • Variable1 = Cost1
  • Variable1 is being wrapped by the ISNULL function to replace any NULL values, in Cost1, with a value of zero
  • ISNULL([Cost1],0)<.02
  • If [Cost1] is less than .02 the system will respond with the configured severity level in the Pricing Strategy which can also be viewed in the Validation Log. The Price Action will not continue validation and calculation if the severity level is configured as ‘error’.
  • If [Cost1] is greater than .02 the system will marked as validation passed and will continue calculation and evaluation
  • Check to see if Retail Sales Price is lower than MSRP
  • Variable1 <Variable4
  • Variable1 = [RSP]
  • Variable4 = [MSRP]
  • [RSP]>[MSRP]
  • If [RSP] is greater than [MSRP] the system will respond with the configured severity level in the Pricing Strategy which can also be viewed in the Validation Log. The Price Action will not continue validation and calculation if the severity level is configured as ‘error’.
  • If [RSP] is less than [MSRP] the system will marked as validation passed and will continue calculation and evaluation.

Error Message (Required)

Enter a message to be included in the validation log for validation failures. Error messages cannot be configured for ‘calculation’ types.

Validation Active (Optional)

Unmarking the checkbox, will remove the ability to configure the Calculation in any new Price Strategies.
