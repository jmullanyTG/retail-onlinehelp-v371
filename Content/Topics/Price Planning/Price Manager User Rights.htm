Price Manager User Rights

There are two sets of permissions within Price Planning; One set not affected by a Price Action Status (i.e. global), the other affected by status. Below is a list of global permissions and each’s purpose. Permissions can be configured by clicking Users from the Maintain tab and using the > and < buttons to add or remove rights for a user or role.

User Right

Description

Access to Price Manager

Provides the ability to view the Price Manager solution, by clicking Price Manager from the Plan tab, and all Pricing Plans and Actions configured.

Access to Price Planning Configuration

Provides the ability to view the Price Manager configuration screen: Rounding Rules, Variables, Calculations, Validations, Strategies and Status.

Add Price Action

Provides the ability to create Price Actions which set pricing for site products.

Add Price Plan

Provides the ability to create (i.e. Start Date, add/remove products and sites) associated to a Pricing Plan.

Edit Price Action

Provides the ability to edit Price Actions which set pricing for site products.

Edit a Price Plan

Provides the ability to edit (i.e. Start Date, add/remove products and sites) associated to a Pricing Plan.

Delete Price Action

Provides the ability to delete Price Actions which set pricing for site products.

Delete a Price Plan

Provides the ability to delete Pricing Plans. Pricing Plans cannot be deleted if there are associated Price Actions.

Allow Emergency Pricing Action

Provides the ability to modify the ‘Pricing Plan Start Date’ and ‘Pricing Action Start Date’ to occur tomorrow. By default, the system will restrict the ‘Plan Start Date’, and associated ‘Price Action Start Date(s)’ by the number of days, from today, configured in the Global Policy, ‘Price Manager Plan Start Date Number Of Days Out’.

Modify Price Action Status

Provides access to the ‘Change Status’ button for a Price Action.

Override a Validation Error

Provides ability to modify a Validation with severity of ‘Error’ into a severity of ‘Warning’ in the Validation Log.
