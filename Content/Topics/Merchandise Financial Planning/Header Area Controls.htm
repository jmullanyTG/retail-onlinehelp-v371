Header Area Controls

  • New: icon to launch the process of creating a new master plan
  • Edit:  icon to edit the details of a master or branch plan
  • Initialize:  icon to initialize a master or branch plan once the plan has been created
  • Initialization: this is the process of populating/seeding the values associated to the measure set that has been selected for a plan.
  • Important: regardless of the selected Seed Type, metrics that would be seeded with negative values based on the available data indicated by that type will be seeded with zeroes instead
  • Plan Status Name: Once a plan has been initialized, the status will update to Initialized in the plan status name column on the left hand side of the Financial Plan Maintenance view
  • Re-Initialize:  icon to re-initialize a plan.  This control should be utilized when master data has changed in the data feed (ie. Re-class)
  • Close:  icon to delete a plan.  This control should be utilized when deleting a master or branch plan from the view
  • Re-Seed:  icon to re-seed future periods within the plan. 
  • A master plan can be re-seeded from calculation as defined by the seed type selected upon creation of the master plan
  • A branch plan can be re-seeded from plan as it is a direct copy of another branch plan or a master plan
  • Important: regardless of the selected Seed Type, metrics that would be seeded with negative values based on the available data indicated by that type will be seeded with zeroes instead
  • Branch:  icon to launch the process of creating a branch from a master plan or another branch plan.
  • Merge: icon to merge the eligible measure set from one branch to another branch or one branch to a master plan
  • Submit:  icon to submit a branch plan for approval.  This icon kicks of the approval process for a plan within Merchandise Financial Planning
  • Approve:   icon to approve a plan that has been submitted to a user for approval.  Once a plan has been approved it can then be merged into the master plan.
  • Reject:   icon to reject a branch plan that has been submitted for approval
  • Global groups:   icon to launch the process of defining the site groups and product groups that will be a part of the hierarchical structure defined for a master plan.  In addition the ordering of the groups are defined in this view
  • User Access:   icon to launch the process of defining the access permissions by role or user
  • Merge Measure set:   Measure Sets are an advanced configuration feature provided in the framework of the Merchandise Financial Planning module.  Just as the name implies, Measure Sets are sets, or groups, of measures that can be created and used for the purpose of merging financial plans.  By default, measure sets are provided for each Merchandise Financial Plan Type (Cost, or Retail) that contain all plan measures which allows all measures to be merged from one plan to another.  However, with the use of measure sets, new sets can be created that would allow for selective merging of specific measures, for instance, only Sales Metrics.  This can be useful for companies who may have one group of Financial Planners who only effect one aspect of the planning metrics who then also want to control what measures get merged back when that same planning group is done and ready to merge their changes back up the financial planning hierarchy. 

Although this allows for great flexibility in controlling what users or roles can merge, it should be set up only by individuals with an intimate understanding of the dependencies between the planning metrics, thus making it an advanced configuration feature.

  • Refresh:   icon to refresh the financial plan maintenance view
