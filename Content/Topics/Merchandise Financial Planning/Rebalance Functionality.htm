Rebalance Functionality

It is a common occurrence within companies to have multiple versions of a plan for the same product/channel levels and window of time for the purpose of exploring different possible outcomes and/or analyzing the business from different perspectives. Sometimes there is a version built from the location level up and another from the top site level down, sometimes there are multiple scenarios built at the same level, or maybe there are even versions built using different hierarchies altogether. Eventually, a reconciliation occurs where those within the planning organization decide which version to move forward with, and it is for this purpose that Rebalance functionality exists – to take the desired values from the “winning” version and homogenize other versions to them.

 

Before initiating a rebalance, it is necessary to have two plans that share at least one common site level, one common product level, and a common window of time. Once you are ready, click in the header area controls to launch the Rebalance Plan dialog

 

Note: it is not necessary for the common levels of the plans being rebalanced to exist in the same positions (i.e. the same number of levels deep). The system will adjust data as needed to ensure the proper values are transferred

 

  • Source Plan: user editable single-select tree; displays/sets the Source Plan, the plan to which the Destination Plan will be homogenized, for the rebalance
  • Destination Plan: user editable single-select tree; displays/sets the Destination Plan, the plan to be homogenized with the Source Plan, for the rebalance
  • Destination Product Hierarchy: user editable multi-select tree; displays/sets the level(s) of the Destination Plan product hierarchy to be homogenized with the Source Plan
  • Destination Site Hierarchy: user editable multi-select tree; displays/sets the level(s) of the Destination Plan site hierarchy to be homogenized with the Source Plan
  • Destination Time Hierarchy: user editable multi-select tree; displays/sets the time periods(s) of the Destination Plan to be homogenized with the Source Plan
  • Sales: user editable checkbox; determines whether Sales revenue in the Destination Plan will be homogenized with the Source Plan for the selected product/site levels and window of time
  • Sales Units: user editable checkbox; determines whether Sales Units in the Destination Plan will be homogenized with the Source Plan for the selected product/site levels and window of time
  • COGS: user editable checkbox; determines whether COGS (cost of goods sold) in the Destination Plan will be homogenized with the Source Plan for the selected product/site levels and window of time
  • Receipt Retail: future functionality; user editable checkbox; determines whether Receipt Retail in the Destination Plan will be homogenized with the Source Plan for the selected product/site levels and window of time
  • Receipt Units: future functionality; user editable checkbox; determines whether Receipt Units in the Destination Plan will be homogenized with the Source Plan for the selected product/site levels and window of time
  • Receipt Cost: future functionality; user editable checkbox; determines whether Receipt Cost in the Destination Plan will be homogenized with the Source Plan for the selected product/site levels and window of time
  • Cancel: closes the Rebalance Plan dialog without performing a rebalance
  • OK: initiates a rebalance between the Source and Destination Plans of the selected product/site levels and window of time for the chosen metrics

 

Once a rebalance is initiated, the system will perform the rebalance from the Source to the Destination plan at the lowest levels of commonality in the product and site hierarchies between plans using the following calculation rules, depending on which metrics were selected:

 

  • Sales only: Lock AUR, Lock AUC, Calc Sales Units, Calc COGS, Calc MMU (Product), MMU % (Product)
  • Sales Units only: Lock AUR, Lock AUC, Calc Sales, Calc COGS, Calc MMU (Product), MMU % (Product)
  • COGS only: Lock Sales, Lock Sales Units, Calc AUC, Calc MMU (Product), MMU % (Product)
  • Sales and Sales Units: Lock AUC, Calc AUR, Calc COGS, Calc MMU (Product), MMU % (Product)
  • Sales and COGS: Lock AUR, Calc AUC, Calc Sales Units, Calc MMU (Product), MMU % (Product)
  • Sales Units and COGS: Lock AUR, Calc AUC, Calc Sales, Calc MMU (Product), MMU % (Product)
  • Sales, Sales Units, and COGS: Calc AUR, Calc AUC, Calc MMU (Product), MMU % (Product)

 

All other dependent metrics in the Destination Plan will recalculate per the normal calculations defined in Main Planning View > Metric Details for the appropriate plan type, Cost or Retail.

 

  • Note: rather than overwrite the values in the Destination Plan, the system will create a rebalanced copy with the word “Rebalanced” appended to the beginning of the Destination Plan name
  • Example: a very common scenario in the retail industry is where there are two sets of planners – tops-down financial planners and store planners. These individuals will create their ideal versions of the larger plan from their perspectives then meet to discuss which total volume outcome is most feasible, given all the available data. Once complete, one version will be rebalanced to the other
