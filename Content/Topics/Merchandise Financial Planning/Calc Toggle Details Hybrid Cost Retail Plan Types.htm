  • HOLD: AUR, AUT, AUT | CALC: Sales Units, COGS, Markdowns, IMU%, MMU %
  • Example: the default Sales calc toggle, this is employed when the user wishes to change sales revenue and units while maintaining the planned average selling price (AUR) per unit

Note that Sales Units has increased to 369 while AUR has remained at $2.71 to balance the change in Sales

 

  • HOLD: Sales Units, ACT, AUT | CALC: AUR, Markdowns, MMU %
  • Example: the alternate calc toggle for Sales allows for a slightly different approach in which the user wishes to maintain Sales Units while adjusting the average selling price (AUR) per unit

In the example above, Sales has again been changed to $1,000 for 2017_wk27

 

But now Sales Units holds at 290 while AUR increases to $3.45 to balance the change in Sales

 

  • Sales Units
  • HOLD: AUR, AUC, AUT | CALC: Sales, COGS, Markdowns, IMU %, MMU %
  • Example: very similar to the default toggle for Sales, the default calc toggle for Sales Units is employed when the user wishes to maintain the average selling price (AUR) per unit while allowing sales revenue to change

In the example above, Sales Units have been changed to 350 for 2017_wk27

 

Sales changes to $1,207 while AUR holds at $3.45 to balance the change in Sales Units

 

  • HOLD: Sales, AUC, AUT | CALC: AUR, COGS, Markdowns, IMU %, MMU %
  • Example: the alternate toggle for Sales Units allows for a different approach, flexing AUR while maintaining sales revenue

In the example above, Sales Units have again been changed to 350 for 2017_wk27

 

But now, Sales holds at $1,000 while AUR decreases to $2.86 to balance the change in Sales Units

 

  • AUR
  • HOLD: Sales Units, AUC, AUT | CALC: Sales, Markdowns, MMU %
  • Example: the default calc toggle for AUR is employed when the user desires to change the average selling price per unit but keep the number of units being sold intact as sales revenue flexes

In the example above, AUR has been decreased to $3.00 for 2017_wk27, creating a $1 difference between AUT (average ticket price per unit) and AUR

 

Note that, while Sales Units has held at 350, Sales has decreased to $1,050, also creating $350 in markdowns (the difference between AUT & AUR * Sales Units), thus, changing AUR is also a useful means of adjusting profitability and managing markdowns in a given period

 

  • HOLD: Sales, AUC, AUT | CALC: Sales Units, COGS, Markdowns, IMU %, MMU %
  • Example: the alternate calc toggle for AUR performs a very similar function but holds sales revenue, allowing the user to change the average selling price while adjusting sales units to balance the change

In the example above, AUR has again been decreased to $3.00 for 2017_wk27, creating a $1 difference between AUT (average ticket price per unit) and AUR

 

In this example, though, Sales has now held at $1,400, while Sales Units has recalculated to 467, also creating $467 in Markdowns ( ($4.00 - $3.00) * 467)

 

  • IMU %
  • HOLD: Sales, Sales Units, AUC | CALC: AUT, Markdowns, MMU %
  • Example: the default calc toggle for IMU % is employed primarily when the user wants to change IMU % while keeping average cost per unit (AUC) intact. The two primary components of IMU % are AUC and AUT (average ticket price per unit), so since this toggle holds AUC, it recalculates AUT in order to balance the change

In the example above, IMU % has been increased to 80% for 2017_wk28, indicating a desire for a higher average ticket price per unit

 

Notice the increase in AUT to balance the increase in IMU% while AUC remains unchanged. Markdowns have also increased, as the gap between AUT and AUR has increased. MMU% does not recalc in this Cost based plan because COGS, a component of MMU (product), does not subtract markdowns in Cost method

 

 

  • HOLD: Sales, Sales Units, AUT | CALC: AUC, COGS, MMU % (not available in Retail method)
  • Example: the alternate calc toggle for IMU % is employed primarily when the user wants to change IMU % while keeping average ticket price per unit (AUC) intact. The two primary components of IMU % are AUC and AUT (average ticket price per unit), so since this toggle holds AUT, it recalculates AUC in order to balance the change

In the example above, IMU % has been increased to 80% for 2017_wk28, indicating a desire for a lower average cost per unit

Notice the decrease in AUC to balance the increase in IMU% while AUT remains unchanged. Markdowns remain unchanged because the relationship between AUT and AUR doesn’t change. MMU %, however, increases, because AUC (and as a result, COGS, decreases, increasing the profit margin between the cost of goods and sales

 

  • Markdowns
  • HOLD: AUT, Sales, AUC | CALC: Sales Units, AUR, COGS, IMU %, MMU %
  • Example: the default calc toggle for Markdowns is employed when a user desires to create markdowns by decreasing the average selling price per unit (AUR) while holding Sales intact

In the example above, Markdowns have been increased to $2,122 for 2017_wk28, necessitating a $1 difference between AUT and AUR, since Markdowns, in Retail method plans, are equal to (AUT – AUR) * BOP Units ($2,122 = ($4.00 – X) * 2,122)

AUR must become $3.00 in order to equate the entered Markdowns value and BOP Units, but because this calc toggle also holds Sales intact, Sales Units must also recalculate to 247 to balance the change ($740 / $3.00 = 247). Lastly, MMU % recalculates as a result of the entered Markdowns value

 

  • HOLD: AUT, Sales Units, AUC | CALC: Sales, AUR, MMU %
  • Example: the alternate calc toggle for Markdowns is employed when a user desires to create markdowns by decreasing the average selling price per unit (AUR) while holding Sales Units intact

In the example above, Markdowns have been increased to $1,028 for 2017_wk28, necessitating a $1 difference between AUT and AUR, since Markdowns, in Cost method plans, are equal to (AUT – AUR) * Sales Units ($1,028 = ($4.00 – X) * 1,028)

 

AUR must become $3.00 in order to equate the entered Markdowns value and Sales Units, but because this calc toggle also holds Sales Units intact, Sales must also recalculate to $3,085 to balance the change (1,028 * $3.00 = $3,085). Lastly, MMU % recalculates as a result of the change in Sales

 

  • Target Receipt Retail
  • HOLD: Target Receipt AUC, Target Receipt AUT | CALC: Target Receipt Units, Target Receipt Cost
  • Example: the default calc toggle for Target Receipt Retail is employed when the user wishes to change the retail value of planned receipts while keeping the average cost and average ticket price per receipt unit (Target Receipt AUC and AUT) intact

In the example above, Target Receipt Retail has been increased to $15,000 for 2017_wk28, indicating a desire to also increase the receipt units for the period

 

  • Target Receipt Units must recalculate to 750 to balance this change (Target Receipt Retail / Target Receipt AUT: $15,000 / $20.00 = 750). Consequently, the change in units also causes Target Receipt Cost to increase to $3,750, as Target Receipt Cost = Target Receipt AUC * Target Receipt Units

 

 

 

 

  • HOLD: Target Receipt Units | CALC: Target Receipt AUT, Target Receipt IMU %
  • Example: the alternate calc toggle for Target Receipt Retail is employed when the user wishes to change the retail value of planned receipts while keeping the planned receipt units intact

In the example above, Target Receipt Retail has again been increased to $15,000 for 2017_wk28, this time indicating a desire to also increase the average ticket price per receipt unit for the period

  • Target Receipt AUT must recalculate to $21.43 to balance this change (Target Receipt Retail / Target Receipt Units: $15,000 / 700 = $21.43). Consequently, the change in units also causes Target Receipt IMU % increase to 76.67% because of the increased profit potential created by the increase in Target Receipt AUT over Target Receipt AUC

 

  • Target Receipt Units
  • HOLD: Target Receipt AUT, Target Receipt AUC | CALC: Target Receipt Retail, Target Receipt Cost
  • Example: the default calc toggle for Target Receipt Units is employed when the user wishes to change the number of receipt units planned for a given period while keeping the average ticket price and cost per receipt unit intact

In the example above, Target Receipt Units has been increased to 1,500 for 2017_wk28, indicating a desire to also increase the cost and retail value of receipts for the period

  • Both Target Receipt Cost and Target Receipt Retail recalc to balance the change in receipt units to $7,500 (1,500 * $5.00) and $30,000 (1,500 * $20.00) respectively while Target Receipt AUC and AUT remain intact

 

  • HOLD: Target Receipt Retail, Target Receipt Cost | CALC: Target Receipt AUT, Target Receipt AUC
  • Example: the alternate calc toggle for Target Receipt Units is employed when the user wishes to change the number of receipt units planned for a given period while keeping the receipt cost and retail values intact

In the example above, Target Receipt Units has again been increased to 1,500 for 2017_wk28, this time indicating a desire to decrease the average cost and ticket price per receipt unit for the period

  • Both Target Receipt AUC and Target Receipt AUT recalc to balance the change in receipt units to $3.33 ($5,000 / 1,500) and $13.33 ($20,000 / 1,500) respectively while Target Receipt AUC and AUT remain intact

  • Target Receipt AUT
  • HOLD: Target Receipt Units | CALC: Target Receipt Retail, Target Receipt IMU %
  • Example: the default calc toggle for Target Receipt AUT is employed when the user wishes to change the average ticket price per receipt unit for a given period while keeping the total planned receipt units intact

In the example above, Target Receipt AUT has been decreased to $18.00 for 2017_wk28, this time indicating a desire to also decrease the average ticket price per receipt unit for the period

  • Target Receipt Retail decreases to $18,000 (1,000 * $18.00) to balance the change in Target Receipt AUT. Target Receipt IMU % also decreases as the difference between Target Receipt AUT and AUC has decreased, reducing the profit potential for the period

 

  • HOLD: Target Receipt Retail | CALC: Target Receipt Units, Target Receipt IMU %, Target Receipt Cost
  • Example: the alternate calc toggle for Target Receipt AUT is employed when the user wishes to change the average ticket price per receipt unit for a given period while keeping the total retail value of receipts intact

In the example above, Target Receipt AUT has again been decreased to $18.00 for 2017_wk28, this time indicating a desire to increase the number of planned receipt units for the period

  • Target Receipt Units and Cost increase to 1,111 ($20,000 / $18.00) and $5,556 (1,111 * $5.00) respectively. Target Receipt IMU % again decreases as the difference between Target Receipt Retail and Cost has decreased, reducing the profit potential for the period
  • Target Receipt AUC
  • HOLD: Target Receipt Units | CALC: Target Receipt IMU %, Target Receipt Cost
  • Example: the default calc toggle for Target Receipt AUC is employed when the user wishes to change the average cost per receipt unit for a given period while keeping the total planned receipt units intact

In the example above, Target Receipt AUC has been increased to $6.00 for 2017_wk28, this time indicating a desire to increase the total receipt cost for the period

  • Target Receipt Cost increases to $6,000 (1,000 * $6.00) to balance the change in Target Receipt AUC. Target Receipt IMU % decreases as the difference between Target Receipt Retail and Cost has decreased, reducing the profit potential for the period

 

  • HOLD: Target Receipt Cost | CALC: Target Receipt Units, Target Receipt IMU %, Target Receipt Retail
  • Example: the alternate calc toggle for Target Receipt AUC is employed when the user wishes to change the average cost per receipt unit for a given period while keeping the total cost of receipts intact

In the example above, Target Receipt AUC has again been increased to $6.00 for 2017_wk28, this time indicating a desire to decrease the total receipt units for the period

  • Target Receipt Units and Retail decrease to 833 ($5,000 / $6.00) and $16,667 (833 * $20.00) respectively to balance the change in Target Receipt AUC. Target Receipt IMU % decreases as the difference between Target Receipt Retail and Cost has decreased, reducing the profit potential for the period

 

  • Target Receipt IMU %
  • HOLD: Target Receipt Units, Target Receipt Cost | CALC: Target Receipt Retail, Target Receipt AUT
  • Example: the default calc toggle for Target Receipt IMU % is employed when the user wishes to change the initial profit potential for a given period while keeping the total receipt units and cost of receipts intact

In the example above, Target Receipt IMU % has been increased to 85% for 2017_wk28, this time indicating a desire to increase the average ticket price per receipt unit and the total retail value of receipts for the period

  • Target Receipt AUT and Retail increase to $33.33 ($5.00 / (1 - .85)) and $33,333 (1000 * $33.33) respectively to balance the change in Target Receipt IMU %

 

  • HOLD: Target Receipt Units, Target Receipt Retail | CALC: Target Receipt Cost, Target Receipt AUC
  • Example: the alternate calc toggle for Target Receipt IMU % is employed when the user wishes to change the initial profit potential for a given period while keeping the total receipt units and retail value of receipts intact

In the example above, Target Receipt IMU % has again been increased to 85% for 2017_wk28, this time indicating a desire to decrease the average cost per receipt unit and the total cost of receipts for the period

  • Target Receipt AUC and Cost decrease to $3.00 ($20.00 * (1 - .85)) and $3,000 (1000 * $3.00) respectively to balance the change in Target Receipt IMU %

 

 

  • Target Receipt Cost
  • HOLD: Target Receipt AUC, Target Receipt AUT | CALC: Target Receipt Retail, Target Receipt Units
  • Example: the default calc toggle for Target Receipt Cost is employed when the user wishes to change the total receipt cost for a given period while keeping the average cost and ticket price per receipt unit intact

In the example above, Target Receipt Cost has been decreased to $4,000 for 2017_wk28, indicating a desire to also decrease the planned receipt units and total retail value of receipts for the period

  • Target Receipt Units and Retail decrease to 800 ($4,000 / $5.00) and $16,000 (800 * $20.00) respectively to balance the change in Target Receipt Cost
  • HOLD: Target Receipt Units, Target Receipt Retail | CALC: Target Receipt AUC, Target Receipt IMU%
  • Example: the alternate calc toggle for Target Receipt Cost is employed when the user wishes to change the total receipt cost for a given period while keeping the planned receipt units and total retail value of receipts intact

In the example above, Target Receipt Cost has again been decreased to $4,000 for 2017_wk28, indicating a desire to also decrease average cost per receipt unit and increase profit potential for the period

  • Target Receipt AUC decreases to $4.00 ($4,000 / 1,000) and Target Receipt IMU % increases to 80% (($20.00 - $4.00) / $20.00) to balance the change in Target Receipt Cost
