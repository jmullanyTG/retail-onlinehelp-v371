Glossary

Term Definition
Accounting Method

The accounting method (Cost or Retail) that serves as the basis for how inventory is valued in a financial plan and for calculating certain metrics, such as MMU (Maintained Markup).

Both cost and retail inventory metrics are available regardless of which accounting method is selected for a plan. There are simply differences in the way some of them are calculated.

Aggregation The combination of lower levels of data within the plan. This often accomplished via simple summation, as with Sales, Sales Units, and other quantitative metrics. However, the exact method of aggregation can change depending on the specific metric and how it is being aggregated.
Aggregation Level A user-specified parameter defining the level at which data is pre-aggregated in the back end of JustEnough. This is done to improve system performance and should be set to the lowest level of the product/site hierarchies at which the user wishes to work
Branch Plan Branch plans are created as children of a given master plan. Multiple branch plans can exist beneath a master plan, allowing users to separate subsets of data for easier planning among multiple individuals on a planning team. Branch plans can also be created from other branch plans, either containing the full data set for planning of different scenarios, or containing further subdivisions of data. Branch plans are editable by users.
Calc Toggle A set of predefined behaviors for dependent metrics when a given metric is changed by the user. There are calc toggles available for many plannable metrics.
Comp Sales Generally speaking, this refers to sales occurring in stores that have been open at least one full year. The exact timing of when stores become “comps” is managed outside of JustEnough.
Cost Accounting Method Accounting method in which inventory units are valued at the average unit cost (AUC) for each period.
Disaggregation The splitting of aggregated values among their lower-level components (such as lower levels of a product or site hierarchy or time period). Since plan values are stored at the lowest levels specified by the plan hierarchies, disaggregation is more accurately described as simply aggregation to lower levels than are shown in the current plan view. In the case of the lowest hierarchy levels being present in the view, simply displaying the stored values.
Initialization The process of seeding a created plan with data from a user specified source so planning can begin. Default seed types are Historic Sales LY, Historic Sales LLY, Forecasted Sales, and None (initializes a blank plan).
Lock & Spread Functionality that allows users to freeze sales and related metrics at specified levels of the plan, ensuring that changes made are spread among unlocked levels of the plan.
Master Plan Plans within Merchandise Financial Planning exist in a parent – child structure, with master plans being the top-level parent. Master plans are not directly editable by the user, but rather, are changed indirectly by merging approved data from branch plans.
Merge The pushing of data from a branch plan into a master plan. By default, merges are initiated automatically when a child branch plan is approved. Merging can also be initiated manually.
Non-Comp Sales Generally speaking, this refers to sales occurring in “new” stores that have been open for less than a full year.
Pivot Grid Merchandise Financial Planning is based on Pivot Grid technology. Similar to a Microsoft Excel pivot table, users can quickly rearrange the layout of the planning area, allowing for intuitive aggregations and groupings of meaningful chunks of data. However, unlike Pivot Tables, the JustEnough Pivot Grid is editable at any level for all plannable metrics.
Product Hierarchy The structure of product levels within an organization. A simple example is a 3-level hierarchy with the following levels: Category, Gender, and Class. The category-level values are Apparel, Accessories, and Footwear. The gender-level values are Male and Female. The class-level values include things like knit tops, short sleeve tees, etc.
  • Retail Accounting Method
    		• Accounting method in which inventory units are valued at the average unit ticket price (AUT) for each period
    Sales by Type Sales can be planned by type within Merchandise Financial Planning, allowing users to be more detailed about the circumstances under which they expect sales to occur. Default sales types are regular, promotional, and markdown sales.
    Site Hierarchy The structure of location levels within an organization. A simple example is a 3-level hierarchy with the following levels: Company, Channel, and District.The company level represents all locations. Channel-level values are Retail, Wholesale, and Ecommerce. District-level values include things like Northwest, Southeast, etc.
    Snapshot A save of the financial plan at a specific point in time. Snapshots are used to compare prior plan states with your current working plan.
    Window of Time A contiguous range of periods used to define the data returned in a view or an analysis.
