Parameters

Selections made in the parameters area define the data that is returned not only in the Main Planning View but also to the Sales By Type and Comp/Non-Comp Supporting views

 

  • Channel: user editable multi-select tree; selects the site hierarchy level(s) to display in the view

    • Note: populates with only the active site hierarchy levels defined for the plan

  • Channel Agg Level: user editable single-select dropdown; selects the level of the site hierarchy at which data will be pre-aggregated in the view; levels below the selected level will not be displayed, however, the system will still respect the specific levels selected in Channel, and display aggregate values for only those levels

    • Note: this parameter should be set at the lowest level at which the user wishes to view/edit site data in the view

  • Product: user editable multi-select tree; selects the product hierarchy level(s) to display in the view

    • Note: populates with only the active product hierarchy levels defined for the plan

  • Product Agg Level: user editable single-select dropdown; selects the level of the product hierarchy at which data will be pre-aggregated in the view; levels below the selected level will not be displayed, however, the system will still respect the specific levels selected in Product, and display aggregate values for only those levels

    • Note: this parameter should be set at the lowest level at which the user wishes to view/edit product data in the view

  • Time: user editable multi-select tree; selects the period(s) to display in the view

    • Note: populates only periods including and between the Start Period and End Period settings for the active plan

      Note: the level at which periods are displayed and selectable is based on the Calendar Level setting for the active plan

      Important: while it is technically possible to select non-contiguous ranges of time, in order for the system to properly calculate values for future periods, all periods between the earliest and latest selected will still be displayed

  • Currency: user editable single-select dropdown; selects the currency in which to view all revenue based metrics within the view

    • Note: populates with only currencies that are currently assigned to active sites within the site hierarchy on which the active plan is based

  • Comparison Type: user editable single-select dropdown; determines the type of comparison to show in the view (comparison metrics appear in light purple). Options are: None (default), vs. Branch, vs. Master, vs. Snapshot, and LLY (Last Last Year/two years back).

    • Note: If you select vs. Branch, vs. Master, or vs. Snapshot, you must also select a comparison plan in the Comparison Plan parameter. The LLY option pulls comparison data directly from the database and does not use the Comparison Plan parameter.
  • Comparison Plan: user editable single-select dropdown; selects the specific plan to use as the source for vs. Branch, vs. Master, vs. Snapshot comparison types.

    • Note: all comparison metrics are populated via direct read from the source plan specified rather than calculated; if no data exists in the selected plan for a given site/product level and period, a value of zero will be displayed

  • View Mode: user editable dropdown; selects the view mode to show in the view– Total Company or Average Store.

    • Total Company: allows the user to view/plan revenue and unit metrics as total company values – meaning the total volume for all sites contained in whatever level the Pivot Grid is aggregated to

    • Average Store: allows the user to view/plan revenue and unit metrics as average store values – meaning the per site volume for the number of sites contained in whatever level the Pivot Grid is aggregated to

    Parameters: Shows/hides the Parameters controls in the current view; hide to increase vertical viewable space on-screen

    Execute: returns data to the view based on the selections made in the Parameters controls
