Header Area Controls

  • Plan Status: the column on the left hand side of the financial plan maintenance view that displays the status the plan is currently in
  • Created:  The status of a plan after the tabs in the Maintain Financial Plan view have been defined
  • All plans and plan types are in the status immediately following the creation of a plan
  • Edits are not available for any plan in created status
  • Users have the ability to initialize a created plan
  • Plans in created status cannot be Re-Initialized, Re-Seeded or Re-Balanced
  • Snapshots cannot be taken for plans in created status
  • Branches cannot be created off of plans in created status
  • Initialized:   The status of a plan once it has been seeded per the selected seed type
  • Edits are not available for master plans but are available for branch plans, no changes to the plan have been made after initialization
  • Users have the ability to edit a branch plan or merge a branch into the master plan, this status will change to In-Progress once an edit has been made and saved
  • Master Plans in Initialized status can be re-initialized
  • All plans in initialized status can be re-seeded
  • All plans in initialized status can be re-balanced
  • All plans in initialized status can have snapshots taken
  • All plans in initialized status can have branches created from them
  • In-Progress:  The status of a plan after a plan has been initialized and changes have been made post-initialization
  • Edits are not available for master plans but are available for branch plans
  • Users have the ability to edit a branch plan, merge a branch into the master plan or set the status to Awaiting Approval

  • Awaiting Approval:   The status of a branch plan after a user has submitted the plan to an authorized user for approval
  • Master plans will never be in this status
  • Edits are not available for master plans or branch plans in this status, a notification to all users with write access will be sent to alert them of the status change
  • Users with access have the ability to review the plan and move to approved or back to In-progress if they reject the plan
  • All plans in this status cannot be re-initialized
  • All plans in this status cannot be re-seeded
  • All plans in this status cannot be re-balanced
  • All plans in this status can have snapshots taken
  • All plans in this status can have branches created from them
  • Approved:   The status of a master or branch plan after an authorized user has approved the child branch plan
  • Once a plan is moved to approved status, users with rights can move the plan back to In-Progress
  • Master plans can be re-initialized
  • Master plans can be re-seeded
  • All plans in this status can be re-balanced
  • All plans in this status can have snapshots taken
  • All plans in this status can have branches created from them
