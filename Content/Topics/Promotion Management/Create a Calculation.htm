Create a Calculation

Within the Promotion Calculations tab, click New above the Calculation Repository grid in the bottom half of the screen to create a new Promotion Calculation. Enter the required fields and click Save.

Field

Description

Calculation Name (Required)

Enter a descriptive label to summarize the purpose of the Calculation. This will be used to identify the Calculation when creating Calculation Sets.

Calculation Description (Optional)

A summary of the Calculation.

Calculation Type (Required)

Identify when the Calculation will be run in relation to the retrieval of any dynamic item, retail, or cost updates to the Promotion. Select the ‘Post’ option from the dropdown menu.

  • Pre - To be developed as future functionality.
  • Post - This should be selected in all cases and will run the Calculation after price and cost retrieval.
  • Both - To be developed as future functionality.

Method (Optional)

The dropdown is populated by the available methods (i.e. advanced processes applied at the database level) configured for use.  Select a method the system will apply for the Calculation. This will supersede any value entered in the Aggregation, Formula and Rounding fields.

Calculation Level (Required)

Select a level from the populated dropdown. All levels act on the Site Cluster level.

  • Promotion Level – To be developed as future functionality. For now, selecting this option means that Calculations will be performed at the Promotion Group level.
  • Promotion Aggregate Level – Calculations will be performed at the Promotion Group level.
  • Group Level – Calculations will be performed at the Product Group level.
  • Product Level
  • Item Level – Calculations will be performed at the Item Master Level.
  • Variant Level – Calculations will be performed at the Variant Level.

Note: The Promotion Manager system defines Product Levels as follows:

  • Inventory – A product at a particular Site so it has a single price.
  • Item Master – A product across all Sites that may have different prices at different Sites. An Item Master may have a max price for example.
  • Variant – A variant of Item Masters based on a shared attribute i.e. size, color, style, etc. An example of a Variant is Men’s Polo Shirts.

Group Type (Required)

From the dropdown options, select one to identify which Group Association Type this Calculation will be run on:

  • Buy Group - The Calculation will only be run on the Buy Group.
  • Get Group - The Calculation will only be run on the Get Group.
  • Buy Get Group - The Calculation will only be run on both the Buy and Get Groups. This option should also be selected for all Promotion Types that are not of the Buy Get type.

Calculation Field (Required)

The dropdown is populated with database fields where the resulting value of the Calculation will be stored.

Aggregation (Optional)

Select an aggregation Calculation from the populated dropdown. This will override the Formula if specified.

  • AVG - Average all results.
  • Calculate - Will run what is specified in the Formula column.
  • MAX - The maximum value of all results.
  • MIN - The minimum value of all results.
  • SUM - Total all results.
  • MIN no 0 - The minimum value of all results, not including zero values.

Formula (Optional)

Enter a mathematical formula for the Calculation. Any Valid SQL statement can be used if all fields exists in the Promotion Price Output standard or custom table structure. This formula will be used only if the Method and Aggregation columns are blank or if Aggregation is set to ‘Calculate’.

Delete (Optional)

Leave this checkbox unselected.

Rounding Set (Optional)

Select from the configured list of Standard or Price Ending Rounding Rules to apply to the Calculation. Rounding Sets can always be assigned but are only applied when the Calculation Field is of a valid numeric data type (Float, Numeric, and Decimal) and when the Method is blank.
