Standard aggregation Calculations

The following standard aggregation Calculations set the lowest and highest promotion price values by rolling up from the corresponding values at the lower levels. With the exception of the Variant Calculations which are only used in Variant implementations, these Calculations are included in most Calculation Sets.

Calculation Name

Configuration

Standard - Promo Low - Variant

Set the Variant level LowestPromotionPrice field to the minimum Item level LowestPromotionPrice including overrides. To be used in a Variant implementation.

  • Calculation Level – Variant Level
  • Group Type – Buy Get Group
  • Calculation Field – LowestPromotionPrice
  • Aggregation Type – MIN no 0
  • Formula – ISNULL(PromotionPrice_Override,LowestPromotionPrice)

Standard - Promo High - Variant

Set the Variant level HighestPromotionPrice field to the maximum Item level HighestPromotionPrice including overrides. To be used in a Variant implementation.

  • Calculation Level – Variant Level
  • Group Type – Buy Get Group
  • Calculation Field – HighestPromotionPrice
  • Aggregation Type – MAX
  • Formula – ISNULL(PromotionPrice_Override,HighestPromotionPrice)

Standard - Promo Low - Group

Set the Group level LowestPromotionPrice field to the minimum LowestPromotionPrice of the products in the group, Item or Variant including overrides.

  • Calculation Level – Group Level
  • Group Type – Buy Get Group
  • Calculation Field – LowestPromotionPrice
  • Aggregation Type – MIN no 0
  • Formula – ISNULL(PromotionPrice_Override,LowestPromotionPrice)

Standard - Promo High - Group

Set the Group level HighestPromotionPrice field to the maximum HighestPromotionPrice of the products in the group, Item or Variant including overrides.

  • Calculation Level – Group Level
  • Group Type – Buy Get Group
  • Calculation Field – HighestPromotionPrice
  • Aggregation Type – MAX
  • Formula – ISNULL(PromotionPrice_Override,HighestPromotionPrice)

Standard - Promo Low - Promo Aggr

Set the Promotion Aggregate level LowestPromotionPrice field to the minimum LowestPromotionPrice of the products (Item/Variant or Groups) in the Promotion. For example, if the Promotion consists of Items and groups, the Promotion Aggregate value is set by comparing the PromotionPrice at Item level for the Items, the calculated PromotionPrice at the Group level for all groups, then setting to the Promotion Aggregate LowestPromotionPrice to the minimum value of all. Overrides are also taken into account.

  • Calculation Level – Promotion Aggregate Level
  • Group Type – Buy Get Group
  • Calculation Field – LowestPromotionPrice
  • Aggregation Type – MIN no 0
  • Formula – ISNULL(PromotionPrice_Override,LowestPromotionPrice)

Standard - Promo High - Promo Aggr

Set the Promotion Aggregate level HighestPromotionPrice field to the maximum HighestPromotionPrice of the products (Item/Variant or Groups) in the Promotion. For example, if the Promotion consists of Variants and groups, the Promotion Aggregate value is set by comparing the calculated PromotionPrice at Variant level for the Variants, the calculated PromotionPrice at the Group level for all groups, then setting to the Promotion Aggregate HighestPromotionPrice to the maximum value of all. Overrides are also taken into account.

  • Calculation Level – Promotion Aggregate Level
  • Group Type – Buy Get Group
  • Calculation Field – HighestPromotionPrice
  • Aggregation Type – MAX
  • Formula – ISNULL(PromotionPrice_Override,HighestPromotionPrice)
