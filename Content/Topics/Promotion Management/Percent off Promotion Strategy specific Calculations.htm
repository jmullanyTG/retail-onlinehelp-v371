Percent off Promotion Strategy specific Calculations

Calculation Name

Configuration

Percent Off – Filldown – Item

Fill down the keyed percent off to all levels while checking for any percent off overrides and filling those down as needed. To be used in an ItemMaster implementation.

  • Method – Promotion_PriceCalculation_OverrideColumn_Filldown_ItemMaster
  • Calculation Level – Group Level
  • Group Type – Buy Get Group
  • Calculation Field – PercentDiscount

Percent Off – Filldown – Variant

Fill down the keyed percent off to all levels while checking for any percent off overrides and filling those down as needed. To be used in a Variant implementation.

  • Method – Promotion_PriceCalculation_OverrideColumn_Filldown_Variant
  • Calculation Level – Group Level
  • Group Type – Buy Get Group
  • Calculation Field – PercentDiscount

Percent Off - Low - Item

Set the LowestPercentDiscount field at the Item level to the percent off override if available, otherwise the keyed percent off.

  • Calculation Level – Item Level
  • Group Type – Buy Get Group
  • Calculation Field – LowestPercentDiscount
  • Formula – LowestPercentDiscount = ISNULL(PercentDiscount_Override,PercentDiscount)

Percent Off - High - Item

Set the HighestPercentDiscount field at the Item level to the percent off override if available, otherwise the keyed percent off.

  • Calculation Level – Item Level
  • Group Type – Buy Get Group
  • Calculation Field – HighestPercentDiscount
  • Formula – HighestPercentDiscount = ISNULL(PercentDiscount_Override,PercentDiscount)

Percent Off - Low - Variant

Set the Variant level LowestPercentDiscount field to the minimum Variant level PercentDiscount_Override, if available, or PercentDiscount. To be used in a Variant implementation.

  • Calculation Level – Variant Level
  • Group Type – Buy Get Group
  • Calculation Field – LowestPercentDiscount
  • Aggregation Type – MIN
  • Formula – ISNULL(PercentDiscount_Override,PercentDiscount)

Percent Off - High - Variant

Set the Variant level HighestPercentDiscount field to the maximum Variant level PercentDiscount_Override, if available, or PercentDiscount. To be used in a Variant implementation.

  • Calculation Level – Variant Level
  • Group Type – Buy Get Group
  • Calculation Field – HighestPercentDiscount
  • Aggregation Type – MAX
  • Formula – ISNULL(PercentDiscount_Override,PercentDiscount)

Percent Off - Low - Group

Set the Group level LowestPercentDiscount field to the minimum PercentDiscount_Override, if available, or PercentDiscount of the products in the group, Item or Variant.

  • Calculation Level – Group Level
  • Group Type – Buy Get Group
  • Calculation Field – LowestPercentDiscount
  • Aggregation Type – MIN
  • Formula – ISNULL(PercentDiscount_Override,PercentDiscount)

Percent Off - High - Group

  • Calculation Level – Group Level
  • Group Type – Buy Get Group
  • Calculation Field – HighestPercentDiscount
  • Aggregation Type – MAX
  • Formula – ISNULL(PercentDiscount_Override,PercentDiscount)

Percent Off - Low – Promo Aggr

Set the Promotion Aggregate level LowestPercentDiscount field to the minimum PercentDiscount_Override, if available, or PercentDiscount of the products (Item, Variant or groups) in the Promotion. For example, if the Promotion consists of Items and groups, the Promotion Aggregate value is set by comparing the minimum PercentDiscount at Item level for the Items, the minimim PercentDiscount at the group level for all groups, then setting to the Promotion Aggregate LowestPercentDiscount to the minimum value of all, taking overrides into account at every level.

  • Calculation Level – Promotion Aggregate Level
  • Group Type – Buy Get Group
  • Calculation Field – LowestPercentDiscount
  • Aggregation Type – MIN
  • Formula – ISNULL(PercentDiscount_Override,PercentDiscount)

Percent Off - High - Promo Aggr

Set the Promotion Aggregate level HighestPercentDiscount field to the maximum PercentDiscount_Override, if available, or PercentDiscount of the products (Item, Variant or groups) in the Promotion. For example, if the Promotion consists of Variants and groups, the Promotion Aggregate value is set by comparing the maximum PercentDiscount at Variant level for the Variants, the maximum PercentDiscount at the Group level for all groups, then setting to the Promotion Aggregate HighestPercentDiscount to the maximum value of all, taking overrides into account at every level.

  • Calculation Level – Promotion Aggregate Level
  • Group Type – Buy Get Group
  • Calculation Field – HighestPercentDiscount
  • Aggregation Type – MAX
  • Formula – ISNULL(PercentDiscount_Override,PercentDiscount)

Percent Off - Promo Low - Item

Set the LowestPromotionPrice field at the Item level to the lowest retail price minus the calculated amount off using the keyed percent off (or override percent off, if entered).

  • Calculation Level – Item Level
  • Group Type – Buy Get Group
  • Calculation Field – LowestPromotionPrice
  • Formula – LowestPromotionPrice = ISNULL(LowestRetailPrice,0) * (1 - ISNULL(PercentDiscount_Override,PercentDiscount))

Percent Off - Promo High - Item

Set the HighestPromotionPrice field at the Item level to the highest retail price minus the calculated amount off using the keyed percent off (or override percent off, if entered).

  • Calculation Level – Item Level
  • Group Type – Buy Get Group
  • Calculation Field – HighestPromotionPrice
  • Formula – HighestPromotionPrice = ISNULL(HighestRetailPrice,0) * (1 - ISNULL(PercentDiscount_Override,PercentDiscount))
