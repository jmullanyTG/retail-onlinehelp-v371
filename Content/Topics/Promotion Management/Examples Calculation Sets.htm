Examples: Calculation Sets

Best practices require that Calculations are assigned to Calculation Sets in the following order:

  1. Fill down keyed values from the Promotion to all levels, if necessary.

  2. If the Promotion Strategy includes a promoted discount, set the Item level promoted discount low and high values to the keyed promoted discount. Then perform all lowest and highest promoted discount Promotion Strategy specific calculations starting from the Item level and working up to the Promotion Aggregate level.

  1. Set the Item level Promotion Price low and high values using Promotion Strategy specific calculations.
  2. Perform all standard aggregation Promoted Price calculations starting from the Variant level (if applicable) and working up to the Promotion Aggregate level.

Note: The Calculation Sets may vary from these examples due to implementation requirements.

For more information on calculation examples, see Examples: calculations.
