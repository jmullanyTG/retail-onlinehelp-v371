Price Translation Management

This screen can be accessed via Plan > Views > Currency > Price Translation Management and allows users to set different pricing strategies by currency and price range. The primary intent of this functionality is to allow users to account for differing costs of doing business in other countries without having to manage pricing for them all individually

Note: In Assortment Planning, the Price Exceptions side view can be used to make overrides by item/currency to the rules created here. See JustEnough Assortment Planning_Standard Documentation > Line Visualization > Supporting Views > Price Exceptions View for more information.

Important: Even if no price translations are needed, you must, at a minimum, create at least one rule with Translation Factors of 1 for all From and To currencies to be used and with a Start Date of the earliest date to be used within any of your assortment plans and assign the rule to the Total Company level of the product hierarchies used in those plans

  • Header Area Controls
  • Currency From: user editable dropdown; selects the currency to view/create translation rules from in the view
  • Note: rules will generally only need to be created for the home currency that you will be managing your assortment plans in
  • Currency To: user editable multi-select; selects the currencies to view/create translations to in the view
  • Include Unidentified:  user editable checkbox; sets whether or not rules that do not yet have From and To currencies defined appear in the Price Translation Rules section
  • Note: when creating new rules, this should be left checked, as From and To currencies can only be defined in the Details section after clicking Apply in the Rules section. Clicking Apply with this setting unchecked when creating new rules will cause the new rule lines to disappear from the view
  • Note: clicking this parameter automatically refreshes the view without having to click Execute again
  • Parameters: shows/hides the Parameters area of the view
  • Execute: updates the Rules and Details sections of view to display rules with the selected Currency From and Currency To assignments
  • Note: rules only have to match one of the Currency From or To assignments to be shown
  • Price Translation Rules Section
  • New: creates a new price translation rule line
  • Delete: deletes the selected price translation rule line(s)
  • Warning: users will not be asked to confirm deletion of translation rules, so be certain you want to remove them before clicking Delete
  • Fields: opens the fields Customization dialog; users can drag fields to/from the tab’s header area and this dialog to customize the view

 

  • Custom layouts can be saved/restored by right-clicking in the Pivot Grid and selecting Save Layout or Restore Layout, respectively
  • Apply: saves all new price translation rules created and changes made to existing rules since the last Apply
  • Cancel: removes all new price translation rules created, and undoes deletions/reverts changes made to existing rules since the last Apply
  • Price Translation Rule ID: non-editable integer; unique system identifier for the price translation rule
  • Rule Name: user editable alpha-numeric; sets/displays the name for the translation rule
  • Privacy: user editable dropdown; defines the privacy settings for the translation rule
  • Public: all users with sufficient system rights can view/edit the rule
  • Private: only the user who created the rule and view/edit it
  • Shared: only specified users with sufficient system rights can view/edit the rule
  • Users: user editable multi-select; sets the users allowed to view/edit the rule
  • From Currencies: non-editable multi-select; displays the currencies being translated from in the rule
  • To Currencies: non-editable multi-select; displays the currencies being translated to in the rule
  • Price Range Endpoints: non-editable alpha-numeric; displays the price range end points for the rule; corresponds to the Price Greater Than entries in the Details section
  • Note: If the Price Greater Than 0 line in the Details section for the rule does not have a translation factor set, this field will be conditionally formatted:

 

  • Minimum Date Present: non-editable calendar control; displays the earliest Start Date for the rule
  • Maximum Date Present: non-editable calendar control; displays the latest Start Date for the rule
  • Copy Rule: user editable checkbox; if checked on Apply, creates a copy of the rule with “ – Copy” and the current date appended to the Rule Name
  • Note: multiple rules can be copied with a single Apply
  • Missing Values: non-editable checkbox; checked if the Price Greater than 0 line for the selected rule in the Rule Details section does not have Translation Factors set
  • Note: all rules must, at a minimum, have a translation factor for the Price Greater Than 0 line set
  • Price Translation Rule Details Section
  • New: creates a new translation factor line for the rule
  • Delete: deletes the selected price translation rule line(s)
  • Warning: users will not be asked to confirm deletion of lines, so be certain you want to remove them before clicking Delete
  • Note: the Price Greater Than 0 line populates for new rules by default and cannot be deleted; if deleted, the line will regenerate on Apply
  • Fields: opens the fields Customization dialog; users can drag fields to/from the tab’s header area and this dialog to customize the view

 

  • Custom layouts can be saved/restored by right-clicking in the Pivot Grid and selecting Save Layout or Restore Layout, respectively
  • Apply: saves all new translation lines created and changes made to existing lines since the last Apply
  • Cancel: removes all new translation lines created, and undoes deletions/reverts changes made to existing lines since the last Apply
  • Price Greater Than: user editable decimal (10.1); sets the starting price for the translation factor(s) defined in the line to be applied; also sets the ending price for the previous line to be less than the value entered
  • Start Date: user editable calendar control; sets the date the translation factor(s) in the line will become active
  • Price Translation Rule Detail ID: non-editable integer; unique system identifier for the translation rule detail; unique by line by currency
  • Translation Factor: user editable decimal (1.10); sets the translation factor for the corresponding currency and price range of the line;
  • Note: translation factors are applied as a multiplier to the ticket price (AUT) of the product in the From Currency and are invoked by ticket prices in that currency, regardless of what the To Currencies are; setting a factor of 1 indicates no change
  • Important: translation factors are applied to VAT Inclusive ticket prices (AUTs) for the purposes of dependent calculations within Assortment Planning
