Planning VAT Rates

This view can be accessed via Plan > Views > Currency > Planning VAT Rates. Some countries include tax in the actual ticket prices of products sold, rather than calculating it as a percent of the selling price during sales transactions. This is known as value added tax or VAT. In this view, users can manage VAT rates at any level of any defined product hierarchy and for Site Groups 1-20, as defined during implementation

Note: All ticket prices (AUT) within Assortment Planning are assumed to be VAT Inclusive unless otherwise noted by an “ex VAT” descriptor in the metric name.

Important: Even if VAT does not apply to your business, you must, at a minimum, assign a VAT Rate of zero to the highest level of all product hierarchies to be used in assortment plans. Because Site Groups are not explicitly tied to site hierarchies, you may need to enter this zero rate for more than highest level Site Group, depending on how your groups are defined

  • Header Area Controls
  • Product Hierarchy: user editable dropdown; selects the product hierarchy for which you want to manage VAT Rates
  • Hierarchy Level: user editable single-select tree; selects the specific level of the selected Product Hierarchy for which you want to manage VAT Rates
  • Note: changes entered for higher levels of the product hierarchy will be pushed down to the levels beneath
  • Note: for higher hierarchy levels with multiple VAT Rates assigned to the levels beneath, VAT Rate will display a straight average of the rates for those levels; bold italic formatting as shown below will be applied to make cases where this is true easily identifiable to the user:

 

  • Start Date: user editable calendar control; selects the start of the time frame for which to display VAT rates in the view
  • End Date: user editable calendar control; selects the end of the time frame for which to display VAT rates in the view
  • Site Group: user editable dropdown; selects the Site Group level (1-20) for which to display and work with VAT rates in the view
  • Note: changes entered for site groups corresponding to higher levels of the site hierarchy will be pushed down to lower level site groups
  • Note: for higher level site groups with multiple VAT Rates assigned to the levels beneath, VAT Rate will display a straight average of the rates for those levels; bold italic formatting as shown below will be applied to make cases where this is true easily identifiable to the user:

 

  • Parameters: shows/hides the Parameters area of the view
  • Execute: updates the view with all defined VAT rate entries for the intersection of the selected Hierarchy Level, Site Group, and time frame
  • Note: with regard to time, any entries that overlap any portion of the time frame specified by the selected Start and End Dates will be shown
  • Planning VAT Rates Section
  • New: adds a new VAT Rate line to the current view
  • Delete: removes the selected VAT Rate line(s) from the current view
  • Warning: users will not be asked to confirm deletion of VAT Rate lines, so be certain you want to remove them before clicking Delete
  • Fields: opens the fields Customization dialog; users can drag fields to/from the tab’s header area and this dialog to customize the view

 

  • Custom layouts can be saved/restored by right-clicking in the Pivot Grid and selecting Save Layout or Restore Layout, respectively
  • Apply: saves all new VAT Rate lines and deletions of existing VAT Rate lines made since the last Apply
  • Cancel: removes all new VAT Rate lines and undoes the deletion of existing VAT Rate lines made since the last Apply
  • Product Level: user editable dropdown (non-editable for saved lines); sets the product level for the VAT Rate line
  • Start Date: user editable calendar control (non-editable for saved lines); sets the start date for the VAT Rate line to become effective
  • End Date: user editable calendar control (non-editable for saved lines); sets the end date for the VAT Rate line
  • Note: the latest VAT rate line for a given product level will always Default to the end of the system timeline (currently Dec 31, 2999) in order to always provide an active VAT rate for system calculations. When setting VAT rate changes beyond the latest entry, the previous latest entry will change its End Date automatically to just before the new entry timeframe begins
  • Example: if my latest VAT entry for Men’s Short Sleeve Tops is currently from January 1, 2018 to Dec 31, 2999 (the system default) and I enter a new line with a start date of June 1, 2018 and an end date of Dec 31, 2018, the previous latest entry’s End Date will change to May 31, 2018. The new entry will begin on June 1, 2018 as entered but the End Date will change to Dec 31, 2999 to provide VAT rates through the future
  • Site Group Columns: columns shown depend on the levels that exist within the selected Site Group
  • VAT Rate: user editable decimal (0.0001) (non-editable for saved lines); sets the VAT Rate for the specified product level and time frame
  • Site Count: non-editable integer; displays the number of sites to which the VAT Rate will be applied; updates after Apply for new lines
  • Important: VAT rates cannot be set for newly entered lines until they have first been saved via Apply; this is a current system limitation rather than an error
