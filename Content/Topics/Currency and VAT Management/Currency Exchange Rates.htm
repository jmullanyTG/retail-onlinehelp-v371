Currency Exchange Rates

This screen can be accessed via Plan > Views > Currency > Currency Exchange Rates and allows users to maintain both global exchange rates and time-based exchange rates from and to all currencies defined within JustEnough. At a minimum, exchange rates should be maintained globally for a single “From” currency, being the home currency used in your assortment plans and as many “To” currencies as you have assigned to the sites within the cluster sets in those plans

Important: If your company only does business in a single currency (ie. No international business), simply set a single conversion entry “From” and “To” your home currency with a Conversion Factor of 1, indicating no change.

  • Header Area Controls
  • Currency From: user editable multi-select; sets the “From” currencies to be worked with in the view
  • Currency To: user editable multi-select; sets the “To” currencies to be worked with in the view
  • Parameters: shows/hides the Parameters area of the view
  • Execute: updates the view with a conversion factor grid for the Currency From and Currency To selections
  • Default Currency Exchange Rates Tab

 

  • Pivot Grid Functionality: this tab features JustEnough standard Pivot Grid functionality, allowing the user to manipulate the field layout by dragging and dropping fields in the Filter, Row, Column, and Data areas of the grid
  • Export: standard Pivot Grid functionality; exports and saves the Pivot Grid in many common formats including Microsoft Excel and Adobe PDF
  • Open: standard Pivot Grid functionality; opens the Pivot Grid in many common formats including Microsoft Excel and Adobe PDF – does not require save prior to viewing in the desired format
  • Graph: activates the standard JE Pivot Grid graph, which graphs any data series selected by the user

 

  • Fields: launches the Pivot Grid Fields Customization dialog;
  • Simple tab: here, fields can be dragged and dropped between Filter, Row, Column, and Data areas of the Pivot Grid to manipulate the current layout
  • Note: dragging fields to the None area hides them in the layout

 

  • Advanced tab: manipulate the layout here by selecting the desired location of each field from the Pivot Area dropdowns next to each; within each area, an Index can be specified to define the order in which the fields display

  • Search bars: each tab contains a search bar in which you can enter text to more quickly locate the fields you wish to manipulate; the fields shown in the view will be limited to only those containing the text you have entered
  • Custom Layouts: changes made to the Assortment Plan Defaults By Time layout can be saved for future use
  • After making changes and clicking OK in the Fields Customization dialog, right click anywhere in the Pivot Grid and choose Save Layout

 

  • Name: Enter a name for a new layout, or select an existing layout using the dropdown menu
  • Description: Enter a description for the layout; not required
  • Default: check this to make the layout you are saving the default layout that loads when you launch the Manage Assortment Defaults dialog
  • Private/Public/Shared: set whether the current layout is only available for the current user, for all users, or for specified users only; when selecting Shared, click the ellipsis (...) button to select the desired users
  • OK: saves the current layout with the settings you have specified
  • Cancel: cancels saving the layout without applying any changes to Description, Default, or Privacy settings
  • Restore a custom layout by right-clicking anywhere in the Pivot Grid, choosing Restore Layout, and selecting the layout to restore
  • Apply: save all changes made to conversion factors in the current view since the last Apply
  • Cancel: reverts all changes to conversion factors in the current view since the last apply to their previously saved values
  • From Currency Description: text description of the From Currency
  • From Currency Code: unique system code for the From Currency
  • To Currency Description: text description of the To Currency
  • To Currency Code: unique system code for the To Currency
  • Conversion Factor: user editable decimal (1.0001); sets the exchange rate from the From Currency to the To Currency
  • Example: I have a Conversion Factor of .88 from USD (US Dollars) to GBP (UK Pounds), thus a sales of $1,000 USD will equal 888£ GBP
  • Note: setting a Conversion Factor of 1 indicates no change
  • Temporal Currency Exchange Rates Tab

This tab allows users to enter exchange rate changes with specific Start and End Dates, useful for planning for future economic fluctuations ahead of time without having to alter the default exchange rates.

 

Note: Temporal Currency Exchange Rates supersede rates entered on the Default tab for the time frames in which they are active

 

  • New: adds a new line for entering a temporal exchange rate
  • Delete: deletes the selected temporal exchange rate line(s)
  • Warning: users will not be asked to confirm deletion of temporal exchange rates, so be certain you want to remove them before clicking Delete
  • Fields: opens the fields Customization dialog; users can drag fields to/from the tab’s header area and this dialog to customize the view

 

  • Custom layouts can be saved/restored by right-clicking in the Pivot Grid and selecting Save Layout or Restore Layout, respectively
  • Apply: saves all new temporal exchange rate lines and changes made to existing lines since the last Apply
  • Cancel: removes all new temporal exchange rate lines and reverts all changes made to existing lines since the last Apply back to their previously saved values
  • Currency From: user editable dropdown; sets the currency to convert from for the line
  • Currency To: user editable dropdown; sets the currency to convert to for the line
  • Start Date: user editable calendar control; sets the start date for the temporal currency conversion
  • Note: creating temporal currency conversions in the past WILL NOT have any effect on actualized periods
  • End Date: user editable calendar control; sets the end date for the temporal currency conversion; not required
  • Note: not setting an End Date for a conversion means it will persist into the future until one is set or another conversion is entered in the future – this effectively makes the End Date for the conversion the day prior to the Start Date of the new conversion
  • Conversion Factor: user editable decimal (1.0001); sets the exchange rate from the From Currency to the To Currency
  • Example: I have a Conversion Factor of .88 from USD (US Dollars) to GBP (UK Pounds), thus a sales of $1,000 USD will equal 888£ GBP
