Curves Overview

The Curves module is a management screen for data used in other modules of JustEnough, such as the Profiles Module and Assortment Planning Module.

Curves are generated within JustEnough using data which has been imported into the system. They may also be imported as-is using the standard import mechanism.

Essential Business Concepts

The important concept to understand when using the curves module is that it is a representation of the available data, and not the interface though which the data is applied.

  • Name – The name for the individual curve. May be concatenated out of several associated tags.
  • Generation Type – The method by which the curve was generated.
  • Manual Curve – When creating a manual curve, the values must be entered by hand. A default value may be specified to set the scale.
  • Specified Conditions – A programmatically generated curve using a selected data sample.
  • Bucket Type – The type of time period used for the x-axis. Bucket types may be daily, weekly, or monthly, but should be selected to support the data.
  • Start Date – The effective start date for the sample.
  • End Data – The effective end date for the sample.
  • Number of Periods – The number of periods contained in the curve.

Tags

Tags are used within the curves module as an attribute in order to help sort and categorize curves. They are distinctly different from the other forms of tags within the system. When selecting curves to view, they may be filtered based on available tags.

Filter Configuration

The filter configuration section will display all available groups (item master, inventory, and site) for inclusion. The order setting will allow users to effectively construct their hierarchy (product, site, etc.) and include the appropriate values for the curve filters.

Tag Configuration

Curve tags are used to give the individual curves searchable attributes. Curve tags may be added or removed within the “Maintain Curve Tags” wizard.

Practical Application

Creating and maintaining the curves is largely managed through the curves wizard.

The curve may be named and security rights added at the time of creation. If desired, tags may be applied using the “Tag Links” button.

Remaining details will be added in the second wizard screen. Filter values must be entered.

Once built, users may choose to preview the curve.

Technical Execution

Curve data is stored in the database under the tbl_curve namespace.
