Workload Queue Active List

The Workload Queue Active list displays the current Batches being or waiting to be processed by the system. The list will be sorted by Process Type and Status which cannot be altered and will provide the following information:

Field

Description

Process Type

Process Types represent the supported JustEnough solutions with the ability to use the Workload Queue. Batches will be grouped by Process Types and will only be displayed if there are ‘active’ Batches of that type.

Status Icon

The system will display the appropriate icon for the current status of the Batch.

Owner

The name of the user who submitted the work to the queue.

Description

Name used to identify different Batches.

  • Inventory Forecast – The system will use the Item Filter Name and product filter used.
  • Assortment Planning – The system will use the Scenario Name. If a Scenario Name is not available, the Plan Name will be displayed with the product filter used.

Batch Priority

The priority calculated by the system or modified by a Super User. The Batch Priority determines the order in which the Batches will be executed in conjunction with Batch priority escalation based on length of time it has been waiting to be processed.

Item Status

The number of Items completed out of the total number of Items to be processed in the Batch.

Enqueue Date

The date time stamp the Batch was submitted to the Workload Queue.

Start Date

The date time stamp the Batch began execution on the Workload Queue. The field will be blank until execution begins.

Expire Date

The system calculated date time stamp the Batch will be removed from the Workload Queue if its execution has not begun.

Last Changed Date

The date time stamp of the last Priority Change.

Last Changed By

The name of the user who last modified the Batch’s Priority.
