Standardized Jobs

Standardized jobs, found in the Job Manager module, come pre-configured with JustEnough. They organize user- and system-submitted jobs into daily, nightly, and weekend batches, according to estimated workload.

Standardized cleanup and optimization tuning jobs are also included.

Daily job

The Daily job processes batches of low-workload jobs during daytime working hours. Daily job tasks should be scheduled to run during the day--for example, Monday through Friday, 7 AM - 6 PM.

Nightly job

The Nightly job processes batches of medium-workload jobs overnight. Nightly job tasks should be scheduled to run after normal working hours--for example, Monday through Friday, 6 PM - 11 PM.

Weekend job

The Weekend job processes batches of medium-workload jobs overnight. Weekend job tasks should be scheduled to run during the weekend--for example, Saturday 3 AM - 11 PM.

Job Task Configuration

For convenience and speed, you can configure your daily, nightly, and weekly standard job tasks to include all (or specific) process types (forecasts, price actions, etc.) and to take advantage of parallel processing capabilities. Double-click on a task in the Job Manager to view and edit the properties described in the table below.

Method Parameter Name Default Value Description
@bDebug False Required. Whether or not the system should provide additional logging
@bEnforceWorkLimit True

Required. Whether or not the system adheres to the Maximum Server Work Threshold system policy.

  • If set to True and the amount of work meets or exceeds the threshold set in the policy, any un-processing job tasks will go into "delay" cycles until the amount of work returns below the threshold value.
@bNightly False Required. Whether or not the job task processes only nightly jobs.
@ParallelBatching False

Required. Applicable only when multiple job tasks are configured for a processing period (Daily, Nightly, or Weekend).

  • If set to True, all job tasks work on the same job until it is completed. Then, they move on to the next job in the queue.
  • If set to False,each job task works on distinct jobs.
  • All job tasks must have the same value (True or False).
@bWeekly False Required. Whether or not the job task processes only weekend jobs.
@iProcessTypeID  

Optional. Specifies the type of jobs that this job task will run. Leave this value blank to run all job types. Otherwise, enter a single integer value from the list below:

  • 1 = Inventory Forecast
  • 2 = Level Forecast
  • 3 = Group Forecast
  • 4 = Replenish Inventory
  • 5 = Assortment Receipt Plan
  • 6 = Adaptive Allocations
  • 7 = Assortment Prepack
  • 8 = Allocations
  • 9 = Replenish Set
  • 10 = Price Manager
@iUserID 1

Optional. The integer ID of the user that the job task will run as if the job process is submitted without an identified user.

  • This user must have all necessary access rights to complete work for the @iProcessTypeID value.
  • Default value is the IOAdmin ID.
@nMarshalID   Required. Unique ID value for this job task. To ensure accurate processing and logging, value must be unique across all job tasks. Best practice is to include this ID value in the job name ("Workload Queue Parallel - Nightly - 1", for example).
