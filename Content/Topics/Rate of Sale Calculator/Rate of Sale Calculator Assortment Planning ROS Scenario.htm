Rate of Sale Calculator – Assortment Planning ROS Scenario Creation

The first component is where users create a Rate of Sale scenario which defines the inputs required to model historic data in which to generate rates of sale outputs.  Users can define and save rate of sale scenarios for application now or in the future based on existing placeholder/variant and site filters – or define filters on the fly to suit your needs and specified time frames. Users can also apply Min/Max Price and Max Price Discount filters to limit the data being considered during calculation.

Additionally, users can easily recall saved scenarios by filtering the list with placeholder/variant and site filters.  Time frame selection can be static or dynamic, with the ability to name dynamic ranges (i.e. Last 4 weeks) and dynamic scenarios will recalculate regularly to make use of the most current historical data.  Further filtering against the historical data is available to refine the ROS scenarios by defining minimum price, maximum price, maximum discounts and price types as restrictions on what is and is not included in the data set.

Step by Step Instructions – ROS Scenario Creation

  1. Variant and Site Filter selection
  1. Select an existing placeholder/variant filter or create a new filter to select the items for the scenario (this determines which scenarios will appear after scenarios have been created and saved)
  2. Select an existing site filter or create a new filter to select the sites for the scenario (for application in assortment planning, select (All Sites)

  1. Click the Execute button

  1. Click the Add button to create a new scenario line

  1. Scenario line inputs

  1. Name – Input a name for the scenario (a default name will populate, but can be overridden)
  2. Description – Input a description for the scenario
  3. Scenario Mode – Select Assortment Planning
  4. Assortment Plan – Select the active assortment plan
  5. Type – Select Revenue or Sales Units to model against
  6. Product Filter – Select the product filter for the scenario, this will drive which items are included in the calculation
  7. Cluster Set – Select the cluster sets to be included for the scenario, this will drive which clusters are included in the calculation. (Cluster sets must be configured for use with Assortment Planning.)
  8. Sales types – Select the type(s) of sales to include in scenario (Reg, Markdown, Promo, etc)
  9. Time Frame – Select an existing time frame or create a new time frame for the scenario
  1. Max Price –Input the highest price of variants to include in the scenario; set to zero to ignore this parameter

    • Note: during calculation, the Max Price specified is compared against the selling price by period, as defined in sales history by type, for all sales included in the Product Filter, Cluster Set, Sales Types, and Time Frame parameters of the scenario; any sales made at a selling price above Max Price will be excluded

    • Important: selling prices by period do not necessarily correlate to average selling prices shown in the UI (for example, AUR within Assortment Planning)

  2. Min Price – Input the lowest price of variants to include in the scenario; set to zero to ignore this parameter
    • Note: during calculation, the Min Price specified is compared against the selling price by period, as defined in sales history by type, for all sales included in the Product Filter, Cluster Set, Sales Types, and Time Frame parameters of the scenario; any sales made at a selling price below Min Price will be excluded
    • Important: selling prices by period do not necessarily correlate to average selling prices shown in the UI (for example, AUR within Assortment Planning)
  1. Max Price Discount – Input the max % discount to ticket price of variants to include in the scenario; set to zero to ignore this parameter

    • Note: during calculation, the Max Price Discount specified is compared against the selling price by period, as defined in sales history by type, for all sales included in the Product Filter, Cluster Set, Sales Types, and Time Frame parameters of the scenario; any sales made at a selling price below (Max Price Discount * ticket price (full undiscounted price) of the variant in question) will be excluded

    • Important: selling prices by period do not necessarily correlate to average selling prices shown in the UI (for example, AUR within Assortment Planning)
  2. Price Type – Select the price type to include in the scenario
  1. Note: price type is only valid with Max Price, Min Price, and/or Max Price Discount are specified; price types are defined by your organization during software implementation and can be things like MSRP, Clearance, etc.
  2. Important: the difference between Sales Type and Price Type can be difficult to understand, but basically, think of Sales Types as defining the larger bucket of sales from which to draw; Price Type is an optional data point that can change by period and can be used to further limit the data selection for the scenario; for example, it is possible to include the Sales Type Markdown but then further limit that data by setting a Price Type of Clearance and a Max Price Discount of 75% to only capture variants that had the Clearance Price Type and selling prices that were no more than 75% the original ticket price for each period in the scenario time frame
  1. Minimum Volume Threshold – Input the minimum number of sales per item/week/all sites to include in the scenario
  2. Access Level – Select access level to the scenario
  1. Public, Private, Shared
  2. If shared, select the shared users by clicking on the ellipsis
  1. Click Save– This will save the scenario inputs
  2. Click Calculate – This will calculate the scenario line/lines
