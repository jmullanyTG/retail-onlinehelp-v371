Example - Creating a Store Supersession

Since new stores do not have sales histories of their own to base forecasts on, the sales histories from comparable stores can be used instead. In this example, we will identify an existing store to serve as a model for a new store in Calgary. Then, we will create a store supersession, which tells JustEnough to use product level sales histories from the model store when forecasting demand for the new Calgary store.

Instructions

  1. From the Manage tab, click Site Manager.

  2. In the Items dialog box, select (All Sites).

  3. In the Choose Fields dialog box, choose site group codes 2-5.

  4. In the site document, filter site groups 2-5 to find a suitable store model for Calgary:

    • Site Group 2 Code = Canada
    • Site Group 3 Code = Retail
    • Site Group 5 Code = Medium Format

    The results show 2 similar stores: Edmonton and Montreal. For this example, we'll choose Edmonton as the model store for Calgary.

  5. Select Calgary in the grid.
  6. In the Site Modeling tab, click New in the Site Model Options toolbar to create a new site model.
  7. In the Site Model Options grid, enter the following details for the new site model row:
    • Supersession Start Date: Choose a date at least one year in the past. This is the date from which sales history will be copied from the model site (Edmonton) to the new site (Calgary).
    • Supersession End Date: Choose a date two years from today. This is the date on which sales history will stop being copied from the model site to the new site.
    • Click the Active checkbox to mark the supersession as active.
    • Click Save.
  8. Now we need to tell JustEnough which item sales histories from Edmonton to copy through this supersession. For this example, we'll select sales from all product levels.
    1. In the Site Model Details grid, select all rows.
    2. Right-click in the Model Site column and choose Multiple Changes.
    3. In the Multi-Data Change Form dialog box, choose Edmonton from the New value field list and click OK.

    4. Click Save in the Site Model details area to save Edmonton as the model site for all product levels at the Calgary store.

Results

JustEnough starts using the sales histories from the Edmonton store to generate demand forecasts for items in the new Calgary store. You can generate these forecasts manually or wait for JustEnough to generate the forecasts automatically as part of its scheduled overnight tasks. Once generated, you can review the demand forecasts for the new store to see if any revisions may be required.
