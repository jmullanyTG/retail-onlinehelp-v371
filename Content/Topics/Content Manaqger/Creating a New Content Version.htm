Creating a New Content Version

You can create content versions from the Content Versions tab of the Content Manager Configuration window.

  1. In the Content Manager Configuration window, open the Content Versions tab and click New in the toolbar.

  2. In the New Content Version dialog box, enter values for the following fields:

    Field

    Description

    Content Version Name

    Enter a descriptive label to summarize the purpose of the Content Version. This will be used to identify the Content Version for Promotion Content.

    Default Version

     

    Select to make the new content version be the default content version. If selected, the new content version will automatically include all geographies when saved. When a promotion is added to a Slot, if there are any Geographies that are not associated with a specific Content Version, they will be assigned the default Content Version.

    Description

    Optional.

    Clusters pane In the Selected column, select the site clusters that you want to assign to this content version.
    Content Field pane

    In the Display column, select the content fields that you want to assign to this content version.

    • You can set the order in which selected content fields will be displayed by dragging them up or down the list.

    • You can control edit permissions for a content field by double-clicking on the blue user icon in the field's Access column. (The field must be selected in the Display column.)

      Users and roles listed in the Keep Role/User Access list will be able to edit this field. Those in the Remove Role/User Access list will be able to see the field but not edit it (it will appear read-only).

Notes:

  • To associate a new content version with a promotion in an event that is currently in planning, you must first manually remove the promotion from the event slot and then add the promotion to the event again to reference the new content version. Any prior modifications to event-specific content will be lost.
  • Promotions added to events after a new content version is created will automatically inherit the new content version.
