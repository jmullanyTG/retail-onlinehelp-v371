Essential Business Concepts

The details of the available policy tasks may be viewed in the main window, but other than reordering, all changes to the policy tasks must be made within the add or edit windows.

  • Task Name – The name given to the task. Any cloned tasks will start with the name “Cloned From:” before they are changed.
  • Task Description – A basic description of the task.
  • Default Task – The default task will be the fallback inventory policy for any inventories not covered by other tasks. There may be only one default task.
  • Task Enabled – If the task is currently enabled (active) this box will be checked.
  • Task Execution Order – The order in which the task rules will be applied. This is important, as some task rules may supersede others depending on their order.
  • Task Start Date Defined – If the task is intended to begin application on a certain date, this check box must be marked, and the start date specified in the next column.
  • Task Start Date – If the task is intended to begin application on a certain date, the start date must be specified and the check box must be marked in the preceding column.
  • Task End Date Defined – If the task is intended to end application on a certain date, this check box must be marked, and the end date specified in the next column.
  • Task End Date – If the task is intended to end application on a certain date, the end date must be specified and the check box must be marked in the preceding column.
  • Task Allowed To Execute – An informational check box which will display the effective status of the task. If the task is enabled and falls within the specified date range, the “task allowed to execute” box should be checked.
