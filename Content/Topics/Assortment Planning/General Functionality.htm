General Functionality

Header Area Controls

  • Product: user editable dropdown; selects a product filter for the current view
  •   this button launches the Filter Conditions dialog, allowing the user to dynamically edit the criteria for the selected filter prior to executing it; the option also exists to create a new filter from the adjusted criteria on the fly
  •   this button launches the Maintain Filter dialog, allowing the user to create or import a new filter, or update, delete, or export an existing filter
  • Note: promotions, in and of themselves are not tied to any group of products or product hierarchy levels; the Product selection only affects the Promotion Applications shown and products listed in the Promotion Application Details
  • Warning: given sufficient permissions, it is possible for a user to create promotion applications for products that aren’t included in the criteria specified by the selected product filter and, thus, disappear from the current view after clicking Execute; the applications do not disappear – to view them, simply change the filter selection or adjust the filter’s criteria to include them again
  • Site Hierarchy: user editable tree multi-select; selects the levels of the current Plan’s Site Hierarchy to include in the current view
  • Note: similar to Product, Promotions are not tied to any Site Hierarchy levels; Site Hierarchy selection drives the Cluster Sets available for inclusion in the view, which, in turn, affects the Promotion Applications shown and the clusters shown in the Promotion Application Details
  • Cluster Sets: user editable multi-select dropdown; selects the cluster sets to include in the current view. The cluster sets listed include all default and "additionally available" cluster sets for the product level selected in the Product parameter.
  • While cluster sets are not explicitly part of the Site Hierarchy, they are directly related, in that cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy; because of this, the Cluster Set dropdown will populate with the cluster sets contained beneath the selected Site Hierarchy levels
  • Note: Promotions are not tied to Cluster Sets directly; Cluster Set selection only affects the Promotion Applications shown and clusters listed in the Promotion Application Details
  • Window of Time: user editable tree multi-select; selects the window of time to display in the current view
  • Note: any promotion or promotion application whose life cycle intersects with any part of the selected Window of Time will be displayed in the view
  • Warning: the selected timeframe must be a contiguous range of periods; if it is not, the system will still use the window of time defined by the first and last selected periods
  • Show Completed: user editable checkbox; sets whether or not promotions whose end dates have passed will be displayed in the view
  • Note: because the Window of Time selection returns all promotions whose eligibility timeframe or start and end dates intersect with any part of the range selected, this is useful for quickly hiding promotions that have already passed without adjusting the Window of Time selection
  • Display Currency: user editable dropdown; sets the currency to display for all revenue based fields in the view; defaults to the Home Currency for the active Assortment Plan
  • Parameters: shows/hides the Parameters area of the view
  • Execute: updates the Defined Promotions, Promotion Application, and Promotion Application Details sections of the view based on the selected view parameters
