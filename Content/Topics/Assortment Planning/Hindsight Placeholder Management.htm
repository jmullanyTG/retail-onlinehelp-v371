Hindsight Placeholder Management

  • Launching: the Hindsight Placeholder Management dialog is launched from the header area controls in the Main Planning View (note this is not available in Collection view)

 

 

  • Functionality: this dialog can be used to create new placeholders as well as managing Like Style mappings for new and existing placeholders within your plan
  • Drag & Drop Capability: regardless of the view configuration (with Current on the left and Historical on the right or vice-versa) users can drag & drop items from both sections to the other to set Map From Variants/Map To PHs (see below for details on these fields)
  • Controls:
  • # Placeholders: user editable integer; sets the number of new placeholder lines to add when New is clicked
  • New: adds the number of new placeholder lines specified in # Placeholders to the bottom of the Current placeholder listing
  • Clear: removes mapped Like Styles (Map From Variant) for the selected placeholder(s)
  • Delete: deletes the selected placeholder(s); only valid for new placeholders; existing placeholders can be deleted in the Main Planning View
  • Mirror Options: when highlighted, copies the selections made for the topmost new placeholder line to any beneath it
  • Note: it is not necessary to un-highlight this option if you want to deviate from the selections made in the topmost placeholder line; simply edit as you would normally and the button will un-highlight automatically
  • Current Collection: when highlighted, automatically sets Coll Code and Coll Desc for newly created placeholders to the values for the collection of the placeholder that was selected when the Hindsight dialog was launched
  • Historical: flips the Current and Historical sections in the view so the Historical section is on the left
  • Note: the dynamic of assigning historical variants to placeholders is essentially the same regardless of the layout, but be aware that while a single historical variant can be assigned to many current placeholders, when assigning in reverse, only one variant can be mapped to a current placeholder (if a blend of multiple variants is desired to seed rate of sale for a placeholder, the Rate of Sale Calculator can be used – see Line Visualization Supporting Views – Rate of Sale Calculator below for more information)
  • Parameters: shows/hides the data selection parameters for the view
  • Parameter options are the same as in the Main Planning View (Product, Site Hierarchy, Cluster Set, Window of Time); these will default to the same product, site, and cluster set selections as in the main view, with time defaulting to the LY equivalent of the time selected in the main view
  • Data selection parameters only determine the variants/placeholders that will populate in the Historical section of the view; Current placeholders are populated based on those shown in the Main Planning View
  • Placeholders that are Inactive, or in Committed status will not be shown in the Current section of the view
  • Current Section
  • Coll Code: user editable dropdown for new placeholders, otherwise non-editable; selects the collection code to which the placeholder will be assigned
  • Note: select a collection by double-clicking; type in the space below Code or Desc of the dropdown to filter the results returned; this acts as a “starts with” filter by default, but can accept “%” as a wildcard to do “contains” type filtering instead
  • Note: new collections can be created on the fly if you don’t wish to assign the placeholder to an existing one; simply enter a description for the collection in the space provided and click New

 

 

  • Coll Desc: non-editable alpha-numeric; the corresponding description for Coll Code for the placeholder
  • PH Code: non-editable alpha-numeric; unique system generated code for the placeholder
  • PH Desc: user editable alpha-numeric; sets the description for the placeholder
  • Image: non-editable image control; displays the primary image associated with the placeholder; if no primary image exists, “No image data” is displayed
  • Product Level: user editable single-select tree; sets the product hierarchy level to which the placeholder will be assigned
  • Note: this defaults to the lowest common product hierarchy level present in the Product filter in the data selection parameters for Placeholder Mode
  • Cluster Set: user editable multi-select; selects the cluster sets to which the placeholder will be assigned. The cluster sets listed include all default and "additionally available" cluster sets for the product level selected in the Product parameter.
  • Phase In: user editable period picker; selects the phase in date for the placeholder
  • Selections limit to periods prior to Phase Out, if set
  • Note: defaults to the earliest non-actualized period within the Window of Time in the data selection parameters
  • Phase Out: user editable period picker; selects the phase out date for the placeholder
  • Selections limit to periods after Phase In, if set
  • Note: defaults to the last period within the Window of Time in the data selection parameters
  • AUT: user editable currency ($10.01); sets the home currency AUT (average unit ticket) for the placeholder;
  • Note: if a Map From Variant is specified, AUT will be set to that of the selected variant
  • Note: AUT is assumed to be VAT inclusive for the purpose of all calculations within Assortment Planning; for companies in which VAT is not in play for the home currency assigned to the active assortment plan, no special actions are required as long as the associated VAT % is set to zero (see JustEnough Currency and VAT Management_Standard Documentation for more information)
  • AUR: non-editable currency ($10.01); displays the AUR (average unit retail price) for the placeholder
  • Margin %: non-editable percentage (10.01%); gross margin percentage for the placeholder
  • Calculation: ((AUR ex VAT / (1 + VAT %)) – AUC) / (AUR ex VAT / 1 + VAT %)
  • ROS $: non-editable currency ($10.01); displays the current ROS $ for the placeholder; blank for new placeholders
  • ROS U: non-editable decimal (1.01); displays the current ROS U for the placeholder; blank for new placeholders
  • Ttl Sls $: non-editable currency ($10.01); displays the current total sales revenue planned for the placeholder; blank for new placeholders
  • Ttl Sls U: non-editable integer; displays the current total sales units planned for the placeholder; blank for new placeholders
  • S/T %: non-editable percentage (10.1%); displays the target sell-through percentage for the placeholder
  • Calculation: (Ttl Sls U / Ttl Buy U)
  • Promo?: non-editable checkbox control; indicates whether the placeholder has any active promotions applied to it
  • Map From Variant: user editable dropdown; selects a placeholder item to use as a Like Style from which the system can seed a sales plan and other information; not required
  • Populates with placeholders and/or variants shown in the Historical section that exist within the selected Product Hierarchy for the line
  • Note: Like Styles can also be dragged from the Historical section onto the target placeholders in the Current section; dragging a Like Style onto a placeholder that already has a Like Style assigned will replace the previous assigned style with the new one
  • Note: type in the space below Code, Desc, Phase In, or Phase Out of the dropdown to filter the results returned; this acts as a “starts with” filter by default, but can accept “%” as a wildcard to do “contains” type filtering instead
  • Note: If set, this will copy the selected item’s Size Option, Curve Mode, Seasonality Curve, AUT (unless specified in Ticket Price), Min Pres, Location Exceptions, and receipt plan constraints (Receipt Variant Min, Receipt Size Min, Variant Order Multiple, Periods of Supply, Periods Before Phase In, Flow Period Frequency, Min Stock to Sales, Shrink % of Sales, Recalc Buy, and Hold Receipt Units) and can calculate a sales plan based on sales present for the selected Like PH between the Comp Start and End dates specified

 

  • Attrib Copy: user editable checkbox; only valid if Map From Variant is set; defines whether any product attributes assigned to the Map From Variant will be copied to the new placeholder line or if any defined attribute defaults for the Product Hierarchy level will be assigned instead (see Attribute Management Standard Documentation for more details)
  • Cluster Copy: user editable checkbox; only valid if Map From Variant is set; defines whether the placeholder will copy the cluster assignments of the selected Map From Variant or use those set by the user on the placeholder line
  • Warning: when Map From Variants are assigned and their cluster assignments are copied for placeholders that existed prior to launching the Hindsight dialog, any existing cluster assignments will be overwritten
  • Note: when Map From Variants are assigned but their cluster assignments are NOT copied for placeholders that existed prior to the launching of the Hindsight dialog, it is possible that some of the pre-assigned clusters will not have rates of sale if there were no sales in those clusters between the Comp Start and Comp End dates for the Map From Variant selected
  • Cluster Set Message: read only alpha numeric; displays system messages regarding the copying of cluster assignments from a Map From Variant for the specified placeholder
  • The Like Style is assorted to more cluster sets than are visible: (<list of cluster sets if the length of list is less than 512 characters OR the text "x cluster sets">), If applied with Cluster Assortment Copy checked, these assortments will also be created for this placeholder
  • The Placeholder to be altered is assorted to more cluster sets than are visible: (<list of cluster sets if the length of list is less than 512 characters OR the text "x cluster sets">). If applied with Cluster Assortment Copy checked, these assortments will also be removed from this placeholder
  • The Placeholder to be altered is assorted to cluster sets to which you do not have access: Assortments to this placeholder cannot be altered using this method
  • Changing assortments for this variant would cause overlapping cluster sets to exist over the same time ranges: This is when the placeholder to be altered is assorted to a variant, that variant is also assigned to other placeholders in the plan, and those other placeholders' combined cluster set / cluster / date assignments would be in conflict if the change was made.
  • Example: Placeholders X and Y are both assigned to Variant A.  Placeholder X is assorted to Cluster Set 1, and Placeholder Y is assorted to Cluster Set 2. Then, you have Placeholder X in the hindsight view as the thing you want to change. If you select a Map From Variant that is assorted to Cluster Set 2, you won’t be able to copy cluster sets, because it would cause both Placeholders X and Y would be assorted to Cluster Set 2 for the same variant
  • Mod Factor: user editable decimal (1.01); only valid if Map From Variant is set; defines a factor multiply the sales history of the Map From Variant by; defaults to 1.0
  • Comp Start: user editable period picker; only valid if Map From Variant is set; defines the start date for the desired window of time from which to capture sales for the Map From Variant
  • Selections limit to periods prior to Comp End, if set
  • Defaults to the Phase In of the Map From Variant
  • Comp End: user editable period picker; only valid if Map From Variant is set; defines the end date for the desired window of time from which to capture sales for the Map From Variant
  • Selections limit to periods after Comp Start, if set
  • Defaults to the Phase Out of the Map From Variant
  • Adj ROS $: non-editable currency ($10.01); blank unless Map From Variant is set; displays the placeholder level ROS $ calculated from the Map From Variant based on the Comp Start and Comp End Dates and any Mod Factor specified that will be assigned to the placeholder
  • Note: because rates of sale are calculated and assigned at the cluster level, actual rates of sale assigned to individual clusters may vary from the value shown here, just as differences can exist between the ROS $ shown in Placeholder view versus those in Cluster Set view
  • Adj ROS U: non-editable decimal (10.01); blank unless Map From Variant is set; displays the placeholder level ROS U calculated from the Map From Variant based on the Comp Start and Comp End Dates and any Mod Factor specified that will be assigned to the placeholder
  • Note: because rates of sale are calculated and assigned at the cluster level, actual rates of sale assigned to individual clusters may vary from the value shown here, just as differences can exist between the ROS U shown in Placeholder view versus those in Cluster Set view
  • Historical Section
  • Function: this section displays the placeholders and/or variants contained within the Product, Site Hierarchy, Cluster Set, and Window of Time parameters of the Hindsight Placeholder Management dialog; this section functions similarly to the Style Bank in that you can drag items listed here onto placeholders in the Current section
  • Note: the section name “Historical” is more of a generic term; based on the parameter selections the user makes, it’s possible to display placeholders/variants from any timeframe here
  • Metrics: all metrics in the Historical section are essentially the same as their Current section counterparts, except that none are editable when the standard view configuration is engaged (meaning Current section on the left and Historical on the right)
  • When the Historical Section is on the Left: this is achieved by clicking the Historical button in the header controls of the dialog; when this is done, Map to Variant is changed to Map to PH, and the Attrib Copy, Mod Factor, Comp Start, Comp End, Adj ROS $, and Adju ROS U fields become part of the Historical section
  • Map to PH: user editable dropdown; selects a placeholder from those in the Current section with compatible product levels to map the item to as a Like Style
  • Note: this field functions very similarly to the Map From Variant field, except that each placeholder in the Current section can only be selected as a Map to PH once in the Historical section, as this view is not capable of mapping multiple items to a single placeholder. To blend multiple items together for the purpose of seeding rates of sale, users must use the Rate of Sale Calculator supporting view (see Line Visualization – Supporting Views > Rate of Sale Calculator below for more information)
  • Note: users can also drag and drop placeholders from the Current section to set the Map to PH; if the placeholder being dropped is already selected as the Map To PH for another line in the Historical section, it will be removed from that line and set as the Map To PH for the line you have dropped it onto
  • Cancel: closes the Hindsight Placeholder Management dialog without committing any of the changes made
  • Apply: adds all new placeholders created and commits all Map From Variant/Map To PH changes made while in the Hindsight Placeholder Management dialog; also initiates recalculation of placeholder sales plans based on changes made
