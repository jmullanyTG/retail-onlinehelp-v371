Smart Start

Once a scenario has been rationalized, a user has the option of Smart Starting. On Smart Start, the system generates placeholder items for each segment/cluster based on the following:

  • TY Act New Choices & TY Act New AUR
  • The results of the Inter and Intra-style marginal return curves
  • The Collection Option of each segment
  • The configuration of choices to clusters within each segment
  • The default minimum presentation, seasonality curve, curve mode, periods of supply, rec var min, rec size min, size curve, periods before phase in, flow cadence, min stk/sales, and shrink % of sales

Given these factors, the system will generate a fully defined sales plan for each placeholder item, including rates of sale, total sales, sales by period, receipt units and flow strategy. The user can can proceed straight to the assignment of actual variants and placing buys, or use the intuitive functionality within Line Visualization and its supporting modules to perform additional analysis and fine tune plans before proceeding.

Note: If multiple users attempt to Smart Start scenarios at the same time, the system will hold the second process until the first one finishes. This will be transparent to the user, aside from an apparent lag in processing time compared to normal for the user initiating the second Smart Start.

  • Overlapping Scenarios
  • Warning: If the intersection of product and site hierarchies and time of a scenario you Smart Start overlaps ANY part of another scenario that has already been Smart Started, the system will display the following warning below. Proceeding will delete any placeholders in Open status. This is because the sales volume those areas account for within your business have already been accounted for in the overall assortment plan, and the system is attempting to prevent you from over-buying:

 

 

  • Example: I Smart Start a scenario for Mens Tops, US Retail, 4th Quarter 2017 and leave all generated placeholders in Open status. Then, I create a new scenario for Mens Short Sleeved Tees, for US Retail â€“ Southeast District, for December 2017 only. ALL placeholders for the first scenario are still deleted and replaced by those generated for the new scenario, because it contains portions of the sales volume that was accounted for in the first scenario. This is essentially like telling the system you want to plan at a more granular level, so it is making space for you to do so without double-accounting for sales volume.
  • Placeholder to Cluster Assortment
  • Because all placeholders created for a given segment are assumed to be shared among all clusters assigned to that segment, the system is able to create a desirable tiering strategy based on the descending number of new choices set in the Smart Started scenario for each cluster.

  • Initial Receipt Plans
  • Using the rates of sale assigned to each placeholder from the Marginal Return Curve Matrix, the default seasonality curve, minimum presentation, and other receipt planning parameters set for the relevant level of the product hierarchy, the system will calculate an optimized receipt flow strategy by cluster. It is to your benefit to have these default parameters set up before you Smart Start a scenario to save yourself the time of having to manually set these parameters later for each placeholder/cluster combination.
  • Reconciling Smart Start Output Back to Scenarios
  • The sum of the Ttl Sls $ and Ttl Sls U in Line Visualization for the placeholders generated by Smart Starting a scenario should be equal to the sum of TY Act New Sales $ or U for the scenario.
  • These sums may also tie back to the Scenario Target minus the TY C/O Sales $ or U, but because TY Act New Sales $ and U can be edited by the user and do not have to tie back to 100% of the Scenario Target, if overrides are made to these metrics, they will not likely tie to the output of Smart Start.
