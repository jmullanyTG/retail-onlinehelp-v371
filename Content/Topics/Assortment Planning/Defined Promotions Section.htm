Defined Promotions Section

  • New: adds a new line in which the user can define a new promotion
  • Delete: removes the selected promotion from the list of defined promotions
  • Warning: users will not be asked to confirm deletion of promotions, so be certain you want to remove them before clicking Delete
  • Fields: opens the fields Customization dialog; users can drag fields to/from the tab’s header area and this dialog to customize the view

 

  • Custom layouts can be saved/restored by right-clicking in the grid and selecting Save Layout or Restore Layout, respectively
  • Apply: saves all newly created promotions and changes/deletions of existing promotions made since the last Apply
  • Cancel: removes all newly created promotions and undoes deletions/change of existing promotions  made since the last Apply
  • Name: user editable alpha-numeric; sets the name for the promotion
  • Description: user editable alpha-numeric; sets the description for the promotion
  • Privacy: user editable dropdown; sets the user access settings for the promotion
  • Public: everyone with access to Promotions can view/edit this promotion
  • Private: only the user who created this promotion can view/edit this promotion
  • Shared: only specified users with access to Promotions can view/edit this promotion
  • Sharing Settings: user editable tree multi-select; sets the specific users to grant permission to view/edit this promotion; only valid when Privacy is set to Shared
  • Type: user editable dropdown; sets the type for the promotion
  • Promotion types are defined during product implementation
  • Note: in this version of Promotions, types are informational only and do not affect the promotion definition or the way it is applied
  • AUR Effects: only one AUR effect from the below list can be applied to a given promotion; on setting one of the following, the other fields will be grayed out and made inactive but can be made active again by clearing the value from the field set

 

  • Replacement AUR: user editable currency; sets an explicit new AUR to be assigned to all placeholder/clusters the promotion is applied to
  • Promotion Discount ($): user editable currency; sets the amount to decrease the AUR of placeholder/clusters the promotion is applied to by
  • Promotion Discount (%): user editable percentage (10.00%); sets a percentage to decrease the AUR of the placeholder/clusters the promotion is applied to by
  • Global: user editable checkbox; sets whether or not the promotion is a global promotion or not
  • Global promotions gain the added Behavior option of Supersede
  • Note: Aside from the above, the global promotion designation is purely informational; while global promos are typically applied to all products or large section of the product hierarchy, this is not required
  • Exclusion Filter: user editable filter selector; selects a product filter of placeholders to exclude from any applications of the promotion

 

  • Filter: user editable dropdown; sets the product filter to be used
  • : launches the Maintain Filters dialog where users can create new product filters, edit, delete, or export existing product filters, and import product filter files
  • Cancel: closes the product filter selection dialog without setting/changing the product filter selection
  • OK: sets/changes the product filter selection
  • Note: once an exclusion filter is set, the field cannot be blanked out; if you don’t wish to use an exclusion filter after one is set, simply define a filter with no criteria, so it returns no items and assign that – nothing will be excluded from the promotion
  • Behavior: user editable dropdown; sets the stacking behavior for the promotion
  • Stacking: all stacking promotions that are active in a given period combine their effects; promotional stacking is always applied in order depending on the type of AUR effect assigned to each: New AUR, % Off, $ Off
  • Non-Stacking: non-stacking promotions operate on best-deal logic, meaning the system will compare the resulting AURs of applied promos and apply the one with the lowest AUR

This result is compared to the result of promo 3, which is still lower than the result of promos 1 and 2, so promo 3 is ultimately applied

 

  • Supersede: the supersede behavior is only valid for Global promotions; when set, supersede promos are applied in place of any others applied during a given period regardless of whether their resulting AURs are lower
  • Final Index: user editable index (1.01); sets the promotional index for the promotion
  • Note: changing the Final Index at this level will only affect newly created Promotion Applications; existing applications will not be changed
  • Is Always Eligible: user editable checkbox; sets whether the promotion will be a standing promotion with no limits on when it can be applied
  • Checking this clears and locks Eligibility Start Date and End Date for the promotion
  • Eligibility Start Date: user editable single select tree; sets the start date (as a period) for the timeframe during which the promotion can be applied
  • Note: Start Dates at the Promotion Application and Application Details levels cannot be set prior to this date
  • Eligibility End Date: user editable single select tree; sets the end date (as a period) for the timeframe during which the promotion can be applied
  • Note: End Dates at the Promotion Application and Application Details levels cannot be set after this date
  • Created By: non-editable alpha-numeric; displays the user name of the creator of the promotion
  • Create Date: non-editable calendar control; displays the date the promotion was created
  • Note: once there are active applications for a given promotion, all fields will be locked for editing except Name, Description, Privacy, and Sharing Settings; promotions with active applications will be conditionally formatted with bold text, with locked fields also being italicized
