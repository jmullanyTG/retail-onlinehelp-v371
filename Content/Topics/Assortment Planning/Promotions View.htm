Promotions View

The Promotions view gives users visibility to the various promotions, whether Active or Inactive, that have been applied to the selected placeholder/collection within the plan. Users also have the ability to adjust start and end dates for applied promotions. The promotions shown here, as well as the creation of new promotions, can be managed in the Promotions area of Assortment Planning.

 

The information provided in this section is meant only to give the user a basic understanding of how promotions interact with one another. For more detailed information on promotional applications and the effects they have on the sales plan, see the Promotions section later in this manual.

 

Note: Promotions view is not available in Collection mode

 

 

  • Promotion Name: non-editable alpha-numeric; displays the name of the promotion
  • Behavior: non-editable dropdown; displays the stacking behavior of the promotion
  • Stacking: all stacking promotions that are active in a given period combine their effects; promotional stacking is always applied in order depending on the type of AUR effect assigned to each: New AUR, % Off, $ Off
  • Non-Stacking: non-stacking promotions operate on best-deal logic, meaning the system will compare the resulting AURs of applied promos and apply the one with the lowest AUR
  • Example: let’s say we have 3 promos applied for a period, 2 stacking, and one non-stacking, as in the table below:

 

 

The system groups all stacking promos together and treats them as one – in this case, promo 1 is considered first, since it is a New AUR promo, followed by promo 2, which takes an additional 20% off of the $18 result after promo 1. This result is compared to the result of promo 3, which is still lower than the result of promos 1 and 2, so promo 3 is ultimately applied

 

  • Supersede: the supersede behavior is only valid for Global promotions; when set, supersede promos are applied in place of any others applied during a given period regardless of whether their resulting AURs are lower
  • AUR Effect: none-editable alpha numeric; displays the defined effect to AUR for the promotion – New AUR, % Off, or $ Off and the value or magnitude associated with each
  • Global: non-editable checkbox; shows whether or not the promotion is a global promotion
  • Global promotions are typically used for storewide promotions like “30% everything in stock except ______.”
  • Application Status: non-editable dropdown; displays the status of the promotion application – Active or Inactive
  • Start: user editable period picker; displays/sets the start date of the promo application for the selected placeholder/collection
  • Selections limited to periods before End and within the promo application timeframe defined in Promotions
  • End:user editable period picker; displays/sets the end date of the promo application for the selected placeholder/collection
  • Selections limited to periods after Start and within the promo application timeframe defined in Promotions
  • Hold Sales: user editable checkbox; sets whether the promotion has the ability to increase total sales of the placeholder/cluster
  • Checked: the total sales of the placeholder/cluster will remain the same as before the promo was applied, and only magnitude of sales in each period will be adjusted
  • Unchecked: the total sales of the placeholder/cluster can increase above the magnitude prior to promo application; this is effected through recalculation of the placeholder/cluster rate of sale
  • Final Index: user editable decimal (1.10); displays/sets the promotional index to apply to the placeholder/cluster
  • Included: user editable checkbox; sets whether the promo will be applied to each the placeholder/cluster or not
  • Fields: opens the fields Customization dialog; users can drag fields to/from the promotions column header area and this dialog to customize the view
  • Custom layouts can be saved/restored by right-clicking in the Snapshot Templates area and selecting Save Layout or Restore Layout, respectively
  • Apply: saves changes made to promo start and end dates since the last Apply and recalculates the placeholder sales plan
  • Cancel: cancel changes made to promo start and end dates since the last Apply and reverts them to their previously saved values
