Promotion Application Section

This section displays information for promotion applications already created for the promotion selected in the Defined Promotions section. Users can create new applications and delete or change existing applications.

  • New: creates a new Promotion Application line for the promotion selected in the Defined Promotions section
  • Delete: removes the selected item from the list of promotion applications
  • Warning: users will not be asked to confirm deletion of promotion applications, so be certain you want to remove them before clicking Delete
  • Fields: opens the fields Customization dialog; users can drag fields to/from the tabâ€™s header area and this dialog to customize the view

  • Custom layouts can be saved/restored by right-clicking in the grid and selecting Save Layout or Restore Layout, respectively
  • Apply:saves all newly created promotion applications and changes/deletions of existing promotion applications made since the last Apply; for promotions marked as Active, applies the promotion per the specifics in the Promotion Application Details section
  • Cancel:removes all newly created promotion applications and undoes deletions/change of existing promotion applications made since the last Apply
  • Product Filter:user editable filter selector; selects a product filter of placeholders to include in the promotion application

  • Filter: user editable dropdown; sets the product filter to be used
  • : launches the Maintain Filters dialog where users can create new product filters, edit, delete, or export existing product filters, and import product filter files
  • Cancel: closes the product filter selection dialog without setting/changing the product filter selection
  • OK: sets/changes the product filter selection
  • Note: choosing a product filter with products that are not included in Product selection of the Parameters area may cause promotion applications to disappear from the view when the screen is refreshed; to view them again, simply adjust the Product selection to include the Product Filter you chose for the application
  • Cluster Sets:user editable multi-select; defines the cluster sets for the promotion application
  • Note: any cluster sets related to the Site Hierarchy selection in the Parameters area will be available for selection here, however, if the cluster sets chosen are not included in the Cluster Sets selection in the Parameters area, the promotion application may disappear from the view when the screen is refreshed
  • Start Date:user editable single select tree; sets the start date (as a period) for promotion application; defaults to the Eligibility Start Date for the promotion or the nearest non-actualized period
  • Note: this date cannot be set prior to the Eligibility Start Date for the promotion
  • Note: Start Dates at the Promotion Application Details level cannot be set prior to this date
  • End Date:user editable single select tree; sets the end date (as a period) for promotion application; defaults to the Eligibility End Date for the promotion
  • Note: this date cannot be set after the Eligibility End Date for the promotion
  • Note: End Dates at the Promotion Application Details level cannot be set after this date
  • Status:user editable dropdown; sets the status of the promotion application â€“ Active or Inactive
  • Active: this means the application will be applied to the placeholder/clusters as per the Promotion Application Details section, with all related sales plans being effected accordingly
  • Inactive:default setting; this means the application is still saved and ready for application but has not yet be actually applied; It will be visible in the Promotions supporting view of Line Visualization but it will not have an effect on the sales plans of the placeholder/clusters shown in the Promotion Application Details section
  • # Variants:non-editable integer; displays the number of placeholders to which the application is actually being applied per the Promotion Application Details section
  • Final Index:user editable index (1.01); sets the Promotional Index for this specific application of the promotion; defaults to the Final Index of the selected promotion
  • Note: changing the Final Index at this level will push down and overwrite all placeholder/cluster Final Indices on Apply for the promotion application in the Promotion Application Details section
  • Location Exceptions:non-editable checkbox; shows whether site level exclusions have been made to the promotion application
  • Hold Sales:user editable checkbox; sets whether the promotion application has the ability to increase total sales of the placeholder/clusters it affects
  • Checked: the total sales of affected placeholder/clusters will remain the same as before the promo was applied, and only magnitude of sales in each period will be adjusted
  • Unchecked: the total sales of the affected placeholder/clusters can increase above the magnitude prior to promo application; this is effected through recalculation of the placeholder/cluster rates of sale
  • : indicates that only some of the placeholder/clusters included in the promotion application have the Hold Sales option checked
  • Note: checking/unchecking this option at the Promotion Application level pushes the change down to all placeholder/clusters included in the application
  • Location Exceptions button: launches the Assortment Promotion Locations dialog

  • Export: standard Pivot Grid functionality; exports and saves the Pivot Grid in many common formats including Microsoft Excel and Adobe PDF
  • Open: standard Pivot Grid functionality; opens the Pivot Grid in many common formats including Microsoft Excel and Adobe PDF â€“ does not require save prior to viewing in the desired format
  • Graph: standard Pivot Grid functionality; graphs the selected data in the view
  • Fields: standard Pivot Grid functionality; allows the user to customize the field layout for the view
  • Custom layouts can be saved and restored by right-clicking in the Pivot Grid and selecting Save Layout or Restore Layout, respectively
  • Apply: saves changes made to the Include field by site since the last Apply
  • Cancel: undoes changes made to the Include field by site since the last Apply
  • Site Hierarchy Columns: displays the description for the Site Hierarchy level of the sites listed in the corresponding band; columns shown depend on the structure of the Site Hierarchy assigned to the current assortment plan
  • Cluster Set: displays the name of the cluster set containing the sites listed in the corresponding band
  • Cluster: displays the name of the cluster containing the sites listed in the corresponding band
  • Location ID: displays the unique system identifier for the site
  • Location Description: displays the description for the site
  • Include: user editable checkbox; sets whether or not the site will participate in the promotion application

  • Cluster % Contribution: non-editable percentage (1.00%); displays the siteâ€™s percent contribution to the cluster to which it is assigned

    • Calculated from the Location % Contribution tables for the relevant product hierarchy level(s)

    • Used in the calculation of the actual promotional index applied and adjustments to rate of sale (if Hold Sales is not checked)

    • Important: for the purpose of determining the effect of excluding certain sites from promotions, location % contribution is calculated by period rather than in total for the entire promotion to yield the most accurate result possible

  • Promotion Application ID: displays the unique system identifier for the promotion application
  • Seasonal Index: non-editable decimal (1.01); displays the aggregate seasonal index for all placeholder/clusters within the period based on their assigned seasonality curves
  • ROS:non-editable currency ($100.01); displays the average sales revenue per site prior to the promotion application for the period for all placeholder/clusters included in the promotion application
  • ROS Units:non-editable integer; displays the average sales units per site prior to the promotion application for the period for all placeholder/clusters included in the promotion application
  • Sales:non-editable currency ($100.01); displays the total sales revenue prior to the promotion application for the period for all placeholder/clusters included in the promotion application
  • Sales Units:non-editable integer; displays the total sales units prior to the promotion application for the period for all placeholder/clusters included in the promotion application
  • Promo Index: non-editable decimal (1.01); displays the promotional index for each period for the promotion application being approximated
  • Effective Index: non-editable decimal (1.01); displays the index applied to period rate of sale to calculate period sales
  • Derived by multiplying the seasonal and promo indices for each period then dividing by the average result of all periods from the minimum Phase In date to the maximum Phase Out date for all placeholder/clusters included in the application
  • AUT:non-editable currency ($10.01); displays the average unit ticket price for the period for all placeholder/clusters included in the promotion application
  • Adjusted Sales:non-editable currency ($100.01); displays the total sales revenue AFTER the promotion application for the period for all placeholder/clusters included in the promotion application
  • Adjusted Sales Units:non-editable integer; displays the total sales units AFTER the promotion application for the period for all placeholder/clusters included in the promotion application
  • AUR:non-editable currency ($10.01); displays the average unit ticket price AFTER the promotion application for the period for all placeholder/clusters included in the promotion application
  • Sales Difference:non-editable currency ($100.01); displays the difference between the sales revenue for the period before and after the promotion is applied

  • Applied By:non-editable alpha-numeric; displays the user name of the person who created the promotion application
  • Apply Date:non-editable calendar control; displays the date the promotion application was created
