Assortment Upcoming Commitments Report

The Assortment Upcoming Commitments Report allows users to easily track key milestones in the commitment process for orders created from assortment plans and quickly navigate to those orders to complete the next required steps. 

Note: planned receipts that have not yet had commitments created for them will still appear in this report, however certain fields may be blank, and the Due Date will calculate based on the intended receipt date and the Default Lead Time Days assigned to the corresponding product

The Assortment Buy Sheet Report is controlled by four parameters:

  • Product Hierarchy: user editable tree multi-select; selects the levels of current plan’s Product Hierarchy levels to include in the current view
  • Site Hierarchy: user editable tree multi-select; selects the levels of the current plan’s Site Hierarchy to include in the current view
  • Window of Time: user editable tree multi-select; selects the window of time to display in the current view
  • Note: order documents are returned based on associated milestone due dates that fall within the selected window; multiple future due dates may exist for an order, but if any of them fall within the selection, the due date displayed will be the one nearest to the current date
  • Display Currency: user editable dropdown; sets the currency to display for all revenue based fields in the view; defaults to the Home Currency for the active Assortment Plan

 

After the user has set the parameter selections at the top of the screen, clicking Execute will return the following columns:

  • Order Header Name:non-editable alpha numeric; displays the name of the order document associated with the listed due date; blank for receipts that have not yet had commitment orders created for them
  • Product Code:non-editable alpha numeric; displays the variant or placeholder code(s) associated with the corresponding order document
  • Product Description:non-editable alpha numeric; displays the description(s) for the variant(s) or placeholder(s) associated with the corresponding order document
  • Receipt Units:non-editable integer; displays the total receipt units for the corresponding product(s)
  • Receipt AUC:non-editable currency ($5.00); displays the average unit cost of the receipt units for the corresponding product(s)
  • Receipt Cost:non-editable currency ($5,000.00); displays the total cost of receipt units for the corresponding product(s); calculated as Receipt AUC * Receipt Units
  • Order Created: non-editable checkbox; indicates whether a commitment order has been created for the corresponding receipt
  • Order Date:non-editable calendar control; displays the date the corresponding order document was created; if no order has been created, set to the Due Date
  • Delivery Date:non-editable calendar control; displays the date the corresponding order is due for delivery
  • Required Action:non-editable alpha-numeric; displays the nearest milestone, based on the assigned workflow; that is due within the selected Window of Time for the corresponding order
  • Due Date:non-editable calendar control; displays the date the corresponding Required Action is due, based on the workflow assigned to the order document, or for receipts without created commitments, based on the Default Lead Time Days of the corresponding product
  • Note: due dates that are in the past will be conditionally formatted for easy identification; this DOES NOT mean the listed Required Action can no longer be completed, however, failure to meet milestones may adversely effect on-time delivery of orders

  • View Order:clicking this button automatically launches Order Manager and displays the corresponding order document for review/action
  • Note: clicking this button for receipts that have not yet had orders created (ie. Order Created checkbox is blank) will do nothing
