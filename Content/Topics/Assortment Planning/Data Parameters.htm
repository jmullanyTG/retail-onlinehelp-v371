Data Parameters

The Assortment Rollup is controlled by eight parameters:

  • Product: user editable dropdown; selects a product filter to define the items to be included in the aggregate rollup at the selected Product Hierarchy Level
  •   this button launches the Filter Conditions dialog, allowing the user to dynamically edit the criteria for the selected filter prior to executing it; the option also exists to create a new filter from the adjusted criteria on the fly
  •   this button launches the Maintain Filter dialog, allowing the user to create or import a new filter, or update, delete, or export an existing filter
  • Site Hierarchy Level: user editable dropdown; selects the level of the site hierarchy at which to view and analyze the metrics contained in the view
  • This selection drives the level of the site hierarchy in which the assortment plan data will be aggregated to.  For example, if the user selects cluster set as the site hierarchy level parameter selection, the assortment rollup will display all aggregated data in the view at the cluster set level, but no lower than that level from a site hierarchical perspective.  The data displayed in the view can be aggregated to levels above cluster set on the fly
  • Site Hierarchy: user editable tree multi-select; selects the specific levels of the current plan’s Site Hierarchy to include in the current view
  • Product Hierarchy Level: user editable dropdown; selects the level of the product hierarchy at which to view and analyze the metrics contained in the view
  • This selection drives the level of the product hierarchy in which the assortment plan data will be aggregated to.  For example, if the user selects sub-department as the product hierarchy level parameter selection, the assortment rollup will display all aggregated data in this view at the sub-department level, but no lower than that level from a product hierarchical perspective. The data displayed in the view can be aggregated to levels above sub-department on the fly
  • Cluster Sets: user editable multi-select dropdown; selects the cluster sets to include in the current view. The cluster sets listed include all default and "additionally available" cluster sets for the product level selected in the Product parameter.
  • While cluster sets are not explicitly part of the Site Hierarchy, they are directly related, in that cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy; because of this, the Cluster Set dropdown will populate with the cluster sets contained beneath the selected Site Hierarchy levels
  • Window of Time: user editable tree multi-select; selects the window of time to display in the current view
  • Warning: the selected timeframe must be a contiguous range of periods; if it is not, the system will still use the window of time defined by the first and last selected periods
  • Comparison Snapshot: user editable dropdown; select a snapshot for the current plan from which to populate comparison metrics in the view 
  • Note:  See snapshot section in the standard AP documentation for detailed information on how a user can take snapshots to create the comparisons against the assortment plan.  Snapshots available for comparison to the user will be based on user rights.
  • View Mode: user editable dropdown; selects the view mode for the assortment rollup – Total Company or Average Store
  • Total Company: allows the user to view/plan revenue and unit metrics as total company values – meaning the total volume for all sites contained in whatever level the Pivot Grid is aggregated to
  • Average Store: allows the user to view/plan revenue and unit metrics as average store values – meaning the per site volume for the number of sites contained in whatever level the Pivot Grid is aggregated to
  • Display Currency: user editable dropdown; sets the currency to display for all revenue based fields in the view; defaults to the Home Currency for the active Assortment Plan
  • Note: due to the amount of data that can potentially be returned in the view via these selection parameters, there are levels at which the performance of the pivot grid will become sub-optimal; if the data that would be returned by a parameter set will exceed the established system threshold, the system will raise an error indicating this and allow the user to revise their parameters
