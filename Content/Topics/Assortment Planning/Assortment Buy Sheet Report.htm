Assortment Buy Sheet Report

The Assortment Buy Sheet Report is intended to provide users the ability to view and evaluate planned items within an assortment for a defined time frame.  The report contains key measures for placeholders/variants/collections related to what those entities are, where they are planned to sell, and when they are planned to sell.  Additionally, users have visibility to sales, pricing, margin, and receipts and on order for all items contained within the report. 

The Assortment Buy Sheet Report is controlled by five parameters:

  • Product: user editable dropdown; selects a product filter for the current view
  •   this button launches the Filter Conditions dialog, allowing the user to dynamically edit the criteria for the selected filter prior to executing it; the option also exists to create a new filter from the adjusted criteria on the fly
  •   this button launches the Maintain Filter dialog, allowing the user to create or import a new filter, or update, delete, or export an existing filter
  • Site Hierarchy: user editable tree multi-select; selects the levels of the current plan’s Site Hierarchy to include in the current view
  • Cluster Sets:   user editable multi-select dropdown; selects the cluster sets to include in the current view. The cluster sets listed include all default and "additionally available" cluster sets for the product level selected in the Product parameter.
  • While cluster sets are not explicitly part of the Site Hierarchy, they are directly related, in that cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy; because of this, the Cluster Set dropdown will populate with the cluster sets contained beneath the selected Site Hierarchy levels
  • Window of Time: user editable tree multi-select; selects the window of time to display in the current view
  • Note: any placeholder whose life cycle intersects with any part of the selected Window of Time will be displayed in the view; this is to allow the user to view placeholders holistically, as displaying subdivisions of their life cycle might alter the decisions made by the user if they have forgotten or are unaware of the full length of time a placeholder will be for sale
  • Warning: the selected timeframe must be a contiguous range of periods; if it is not, the system will still use the window of time defined by the first and last selected periods
  • Display Currency: user editable dropdown; sets the currency to display for all revenue based fields in the view; defaults to the Home Currency for the active Assortment Plan

 

After the user has set the parameter selections at the top of the screen, clicking Execute will return the following columns in the Assortment Buy Sheet Report:

  • Image – displays a thumbnail of the primary image for the placeholder or attached variant
  • PH/PLM Code – unique system generated code for the placeholder or if a variant is assigned in Line Visualization, displays the PLM code associated with the assigned variant
  • PH/PLM Description – description for the placeholder or if a variant is assigned in Line visualization, displays the PLM description associated with the assigned variant
  • Avg. Unit Ticket (AUT) – average unit ticket price for the placeholder
  • Avg. Unit Retail (AUR) – average unit retail price for the placeholder
  • Avg. Unit Cost (AUC) – average unit cost price for the placeholder
  • Margin % – margin percentage for the placeholder
  • Calculation: (AUR – AUC) / AUR
  • Phase In – the phase in date for the placeholder, first period of planned sales
  • Phase Out – the phase out date for the placeholder, last period of planned sales
  • # Weeks – total number of weeks the placeholder is on offer to the customer
  • Rate of Sale $ -- average sales per site per period in revenue for the placeholder
  • Rate of Sale U – average sales per site per period in units for the placeholder
  • Total Sales $ -- total sales revenue for the item from Phase in through Phase out
  • Total Sales U – total sales units for the item from Phase in through Phase out
  • Promo – if checked a promotion exists for the placeholder, if un-checked no promotion exists
  • Plan Receipt $ -- total cost of planned receipts for an placeholder from Phase in through Phase out
  • Plan Receipt U – total units of planned receipts for an placeholder from Phase in through Phase out
  • # of Flows – total number of periods that receipts are planned to deliver for an placeholder
  • Site Count – total number of sites that an placeholder is assorted to
  • Avg. Units – average amount of planned receipts per site
  • On Order U – total number of committed units planned to deliver for an placeholder
  • % Commit -- % of total planned receipts that have been committed to for an placeholder
  • Calculation: (On Order U + prior actual receipts) / Planned Receipt U
  • Status: displays the placeholder Status, Open, Created, Assigned, Committed, or Replen
  • Cluster Sets -- displays the cluster sets assigned to the placeholder
