ROS Scenario Creation/Selection

Using the parameters and controls at the top and at the far left of the view, users can either select a saved rate of sale scenario to apply to the placeholders/collections selected in the Main Planning view, and edit that scenario is desired, or define the inputs for an entirely new scenario.

Step by Step Instructions

  1. Variant and Site Filter selection
  1. Select an existing placeholder/variant filter or create a new filter to select the items for the scenario (this determines which scenarios will appear after scenarios have been created and saved)
  2. Select an existing site filter or create a new filter to select the sites for the scenario (for application in assortment planning, select (All Sites)

  1. Click the Execute button

  1. Click the Add button to create a new scenario; this will reset the parameters at the left to their defaults to allow for a new scenario but will not alter any saved scenario that was selected previously

  1. Scenario inputs
  1. Name – Input a name for the scenario (a default name will populate, but can be overridden)
  2. Description – Input a description for the scenario
  3. Variant Filter – Select the product filter for the scenario, this will drive which items are included in the calculation
  4. Cluster Set Names – Select the cluster sets to be included for the scenario, this will drive which clusters are included in the calculation
  5. Time Frame – Select an existing time frame or create a new time frame for the scenario

  1. When creating a new time frame for the scenario, the following inputs are required:
  1. Name – input a name for the Time Frame
  2. Description – input a description for the Time Frame
  3. Dynamic Rule – check the box to make the time frame dynamic.  Save scenarios will recalculate based on the most up to date historical data
  4. Offset days – sets the number of days from today’s date to include in the data set; use negative numbers for days back and positive numbers for days forward
  5. Number of Rolling Days – number of days forward from the day defined by the offset days (ex. If offset days is -365 and rolling days is 365 that gives the user the last year of historical data in the data set)
  6. Fixed Start Date – Utilize if the scenario is static to define the begin date of the data set to be utilized in the calculation
  7. Fixed End Date – Utilize if the scenario is static to define the end date of the data set to be utilized in the calculation
  8. Access Level – determines whether a time frame that has been created is public, private or shared
  9. User List – if access level is shared, lists the users that have access to the Time Frame selection (auto populated)
  10. Date Created – date the time frame selection was created (auto populated)
  11. Created by – displays the user that created the time frame selection (auto populated)
  12. Date last updated – the date the time frame selection was most recently updated (auto populated)
  13. Updated by – displays the user that last updated the time frame selection (auto populated)
  14. Status – displays whether or not the time frame selection is active or inactive (auto populated)
  1. Scenario Type – Select Revenue or Sales Units to model against
  2. Sales types – Select the type(s) of sales to include in scenario (Reg, Markdown, Promo, etc)
  3. Price Type – Select the price type to include in the scenario
  1. Note: price type is only valid with Max Price, Min Price, and/or Max Price Discount are specified; price types are defined by your organization during software implementation and can be things like MSRP, Clearance, etc.
  2. Important: the difference between Sales Type and Price Type can be difficult to understand, but basically, think of Sales Types as defining the larger bucket of sales from which to draw; Price Type is an optional data point that can change by period and can be used to further limit the data selection for the scenario; for example, it is possible to include the Sales Type Markdown but then further limit that data by setting a Price Type of Clearance and a Max Price Discount of 75% to only capture variants that had the Clearance Price Type and selling prices that were no more than 75% the original ticket price for each period in the scenario time frame
  1. Max Price – Input the highest price of variants to include in the scenario; set to zero to ignore this parameter
  2. Min Price – Input the lowest price of variants to include in the scenario; set to zero to ignore this parameter
  3. Max Price Discount – Input the max % discount to ticket price of variants to include in the scenario; set to zero to ignore this parameter
  4. Minimum Volume Threshold – Input the minimum number of sales per item/week/all sites to include in the scenario
  5. Access Level – Select access level to the scenario
  1. Public, Private, Shared
  2. If shared, select the shared users by clicking on the ellipsis
  1. Click Apply – This will save the scenario inputs
  2. Click Calculate – This will calculate the scenario line/lines
