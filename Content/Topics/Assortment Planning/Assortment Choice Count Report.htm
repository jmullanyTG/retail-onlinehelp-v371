Assortment Choice Count Report

The Assortment Choice Count Report is intended to provide users the ability to view and evaluate planned number of choices and productivity of the planned choices of an assortment in aggregate over a defined window of time.  The report contains key measures at an aggregate level of the product hierarchy or product filter related to the choices planned for an assortment separating the choice counts by sales type.  The report will display how many choices for a given period are regular price, on promotion, or designated as markdown.  Additionally choice counts can be measured against the same period last year to determine the balance of assortment as it relates to sales by type (reg. price, promotion, and markdown).  Other key measures that are contained within this report are related to Total Sales, Total Cost of Sales, and Total Margin.

The Assortment Choice Count Report is controlled by seven parameters:

  • Product: user editable dropdown; selects a product filter for the current view
  •   this button launches the Filter Conditions dialog, allowing the user to dynamically edit the criteria for the selected filter prior to executing it; the option also exists to create a new filter from the adjusted criteria on the fly
  •   this button launches the Maintain Filter dialog, allowing the user to create or import a new filter, or update, delete, or export an existing filter
  • Site Hierarchy: user editable tree multi-select; selects the levels of the current Plan’s Site Hierarchy to include in the current view
  • Cluster Sets:   user editable multi-select dropdown; selects the cluster sets to include in the current view. The cluster sets listed include all default and "additionally available" cluster sets for the product level selected in the Product parameter.
  • While cluster sets are not explicitly part of the Site Hierarchy, they are directly related, in that cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy; because of this, the Cluster Set dropdown will populate with the cluster sets contained beneath the selected Site Hierarchy levels
  • Window of Time: user editable tree multi-select; selects the window of time to display in the current view
  • Warning: the selected timeframe must be a contiguous range of periods; if it is not, the system will still use the window of time defined by the first and last selected periods
  • Attribute Family:   Each attribute family contains attributes that pertain to that family and can be imported from host systems if available.  (Example:  Attribute Family = Fabric Type, and a value belonging to this family could be cotton).  This selection defines the values within a family or group of families that will be available for selection and viewing in the sixth parameter.
  • Attribute Values:  This selection returns the values associated with the attribute families selected in the fifth parameter.
  • Note:  It is not necessary to select Attribute Family or Attribute Values in the parameters selections as it is possible to review choice counts and productivity through the utilization of a product filter, Site Hierarchy, Cluster Sets for a defined window of time.
  • Display Currency: user editable dropdown; sets the currency to display for all revenue based fields in the view; defaults to the Home Currency for the active Assortment Plan

 

This report is a pivot grid and elements contained within the grid may be pivoted as is standard for most of the pivot grids contained within the software.  After the user has set the parameter selections at the top of the screen, clicking Execute will return the following rows in the Assortment Choice Count by Period Report:

  • Reg Choices – number of regular price variants/placeholders planned to sell in the assortment by period
  • Reg Choices LY – Last year number of regular price variants/placeholders selling in the assortment by period
  • Reg Choices % +/- LY -- % increase or decrease versus last year for regular price choices in the assortment.
  • Promo Choices – number of variants/placeholders planned to sell on promotion in the assortment by period
  • Promo Choices LY – Last year number of variants/placeholders selling on promotion in the assortment by period
  • Promo Choices % +/- LY -- % increase or decrease versus last year for promotional choices in the assortment
  • Mkdn Choices – number of variants/placeholders planned to sell on markdown in the assortment by period
  • Mkdn Choices LY – Last year number of variants/placeholders selling on markdown in the assortment by period
  • Mkdn Choices % +/- LY -- % increase or decrease versus last year for markdown choices in the assortment
  • Ttl Choices – The sum of Reg choices + Promo choices + Mkdn choices planned to sell in the assortment by period
  • Ttl Choices LY – Last year sum of Reg choices + promo choices + Mkdn choices selling in the assortment by period
  • Ttl Choices % +/- LY -- % increase or decrease versus last year for the total number of choices in the assortment
  • Sales $ -- Total sales revenue planned or actual for a given period
  • Sales $ LY – Last year total sales revenue for a given period
  • Sales $ % +/- LY -- % increase or decrease versus last year in sales revenue for a given period
  • Sales $ % Cont -- % contribution of sales coming from each cluster set or cluster related to the parameters selected in the top of the pivot grid
  • Sales U – Total sales units planned or actual for a given period
  • Sales U LY – Last year total sales units for a given period
  • Sales U % +/- LY -- % increase or decrease versus last year in sales units for a given period
  • Sales U % Cont -- % contribution of sales units coming from each cluster set or cluster related to the parameters selected in the top of the pivot grid
  • Productivity $ -- average sales revenue per choice per location per period
  • Productivity $ LY – Last year average sales revenue per choice per location per period
  • Productivity $ % +/- LY -- % increase or decrease versus last year for productivity $’s
  • Productivity U – average sales units per choice per location per period
  • Productivity U LY – Last year average sales units per choice per location per period
  • Productivity U % +/- LY -- % increase or decrease versus last year for productivity U’s
  • COGS – Cost of goods sold for a given period
  • COGS LY – Last year Cost of goods sold for a given period
  • COGS % +/- LY -- % increase or decrease versus last year for Cost of goods sold
  • COGS % Cont -- % contribution COGS coming from each cluster set or cluster related to the parameters selected in the top of the pivot grid
  • Margin $ -- Profit margin expressed in value for a given period
  • Margin $ LY – Last year profit margin expressed in value for a given period
  • Margin $ % +/- LY -- % increase or decrease versus last year of profit margin
  • Margin % -- Profit margin % for a given period
  • Margin % LY – Last year profit margin % for a given period
  • Margin % % +/- LY – Profit margin % rate change versus last year

Deriving Choice Counts:

  • Historical Periods - choice counts for each type (Regular, Promo, Mkdn) are derived from the presence of sales of the corresponding type for a given variant.
  • Planned (future) Periods
  • Regular Choices - sales are planned for the period but no promotion is in effect, and the Mkdn Date, if set, has not yet been reached
  • Promo Choices - sales are planned for the period, a promotion IS in effect, and the Mkdn Date, if set, has not yet been reached
  • Mkdn Choices – sales are planned for the period and the period is the same as or past the period set as the Mkdn Date
