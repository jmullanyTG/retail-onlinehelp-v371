Seasonality Curve Generator

The Seasonality Curve Generator within Assortment Planning is a scaled down version of the full Curves module found in Maintain > Curves. Using this view, users can select and apply seasonality curves to the entire selected placeholder/collection or only to selected cluster sets/clusters it is assorted to. Creation of new curves, though, is still done within the full Curves module.

 

 

  • User Inputs
  • Saved Curves: user editable dropdown; selects a curve generated within the Curves module to apply to the selected placeholder/collection
  • Note: only curves of the Manual or Specified Conditions Using Enhanced Filtering types will be available for selection within Assortment Planning
  • Curve Mode: user editable dropdown; sets the mode of the selected curve
  • Static: the selected curve is applied exactly as it exists at the time of application and the placeholder/collection’s curve will not change if changes are made to the curve, whether manually in the Curves module or through dynamic recalculation. Think of this as making a copy of the applied curve.
  • Dynamic: changes made to the curve will be pushed down to the placeholder/collection via automated system processes, and the dependent sales plans will recalculate when those changes are pushed.
  • Filter Tags: user editable multi-select; selects tags to be used in filtering the curves available for selection in Saved Curves; tags can be attached to curves within the Curves module

 

  • Filter the list of tags by typing in the blank field above; use % at the beginning to do a “Contains” type filter instead of a “Begins With” type
  • Apply To Cluster: user editable tree multi-select; selects the cluster sets/clusters to apply the selected curve to from those assorted to the selected placeholder/collection
  • Apply Lowest Level Curve: user editable checkbox; sets the level of curve to be applied
  • Unchecked: applies the curve exactly as shown to all selected cluster sets/clusters
  • Checked: looks for related lower level curves to apply at the cluster set/cluster level and applies them instead of applying the higher level curve as shown
  • During implementation, a system process will be configured to auto-generate curves at levels of the product/site hierarchies on a specified schedule; if desired, this process can be configured to generate these curves at multiple levels
  • “Related lower level curves” refers to these auto-generated curves – created using the same criteria
  • Example: you choose an auto-generated channel level curve from the Saved Curves dropdown and check the Apply Lowest Level Curve option; the selected placeholder has 2 cluster sets, each with 3 clusters, assorted to it, and curves were also auto-generated using the same criteria for each of those 6 clusters; the system will apply the cluster-level curves to each cluster in a single action, saving you the effort of applying the lower level curves individually.
  • Curve Details
  • Generation Type: non-editable dropdown; displays the generation type for the selected curve
  • Note: only curves of the Manual or Specified Conditions Using Enhanced Filtering types will be available for selection within Assortment Planning
  • Bucket Type: non-editable dropdown; displays the bucket type for the selected curve
  • Note: for the purposes of Assortment Planning, bucket type is the same as period type
  • Time Range: non-editable dropdown; displays the time range used to generate the selected curve
  • Curve Source: non-editable dropdown; displays the curve source for the selected curve
  • Forecasted Demand: if configured, generates curves based on projected future demand from the Forecasts module
  • Historical Demand By Sales Type Using Sales History: generates curves based on historical sales by type
  • Generate By: non-editable dropdown; displays the generation type for the selected curve – this is essentially the metric used to calculate the seasonal indices;
  • Volume Contribution (Units) – sales units
  • Cost of Sales Contribution – COGS (cost of goods sold)
  • Revenue Contribution – sales $
  • Profit Contribution – gross margin $ (sales $ - COGS)
  • Sales Types: non-editable multi-select; displays the sales types used in generation of the selected curve
  • Note: not applicable for Forecasted Demand Curve Source
  • Includes Returns: non-editable checkbox; shows whether or not returns (negative sales) were included in the calculation of the selected curve
  • Inactive Items: non-editable checkbox; shows whether or not inactive items were considered in the calculation of the selected curve
  • Product Filtering: fields shown change depending on the options used to generate the selected curve
  • Hierarchy: non-editable dropdown; displays the product hierarchy used in generation of the selected curve
  • Hierarchy Items: non-editable tree multi-select; displays the product hierarchy levels used in generation of the selected curve
  • Filter: non-editable single-select tree; displays the product filter used in generation of the selected curve
  • Site Filtering: fields shown change depending on the options used to generate the selected curve
  • Hierarchy: non-editable dropdown; displays the site hierarchy used in generation of the selected curve
  • Hierarchy Site: non-editable tree multi-select; displays the product hierarchy levels used in generation of the selected curve
  • Filter: non-editable single-select tree; displays the site filter used in generation of the selected curve
  • Cluster Set: non-editable dropdown; displays the cluster set used in generation of the selected curve
  • Cluster: non-editable multi-select; displays the clusters used in generation of the selected curve
