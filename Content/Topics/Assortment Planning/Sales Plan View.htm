Sales Plan View

Sales Plan view allows users to make period level edits to the sales plan of the selected placeholder/collection, either in total or by average store and compare them to the high level plan created in the Main Planning view of Line Visualization. Users can also track inventory and plan shrink and adjustments for more detailed control both pre and in-season.

 

 

  • Parameters: shows/hides the parameters pane at the top of the view
  • Execute: refreshes the view
  • Note: this is not typically necessary, as the view refreshes on its own when a new placeholder/collection is selected in the main planning view, when changes are applied, and when the Comparison Snapshot or View Mode are changed
  • Comparison Snapshot: user editable dropdown; selects a snapshot all saved snapshots within the current plan that are relevant to the current Product and Site Hierarchy selections in the Main Planning view parameters
  • View Mode: user editable dropdown; selects the view mode for sales plan view – Total Company or Average Store
  • Total Company: allows the user to view/plan revenue and unit metrics as total company values – meaning the total volume for all sites contained in whatever level the Pivot Grid is aggregated to
  • Average Store: allows the user to view/plan revenue and unit metrics as average store values – meaning the per site volume for the number of sites contained in whatever level the Pivot Grid is aggregated to
  • General Metrics and Concepts
  • System vs User Metrics:
  • System: denoted by the word “system” at the beginning (System Sales, System BOP Units, etc.) and green row coloration; system metrics are those based on the values shown in the Main Planning and Cluster Set views of Line Visualization
  • User: user metrics default to the System metrics of like name, but these are based on edits made in Sales Plan view to the editable metrics, thus, the user plan can diverge from the system plan in Sales Plan view
  • Ex VAT Metrics: denoted by “ex VAT” being appended to the end of the metric name (System Sales ex VAT & Sales ex VAT, for example); in implementations where VAT (value added tax) is in play, ex VAT metrics are provided to give accurate profit calculations (margin) after taxes have been removed
  • Note: in implementations where VAT is NOT in play, no customization of calculations is necessary; simply go to Plan > Views > Currency > Planning VAT Rates and set the VAT rates for all levels of the product hierarchy assigned to your assortment plan to 0. This will set the ex VAT metrics equal to the non ex VAT metrics
  • Comparison Metrics: denoted by purple row coloration and “(Comparison)” appended to the end of the metric (Sales (Comparison), etc.); these metrics are direct data reads from the selected Comparison Snapshot; defaults to zero if no snapshot is selected or no data is present in the snapshot
  • Editable Metrics: denoted by light blue row coloration, these are the only metrics that are editable by the user in Sales Plan view; all others are calculations or reads from other sources
  • * System, ex VAT, and Comparison metrics are never editable in Sales Plan view
  • Note: with the exception of Selling Period, edits made in Sales Plan view DO NOT change the plan values in the Main Planning and Cluster Set views of Line Visualization – this is by design to allow for comparison of original plans with period level changes. You can, however, bring the SPV Sls $ and Sls U metrics into your Main Planning view layout to view these edits at placeholder/collection or cluster set/cluster level
  • Calculations: calculations in Sales Plan view all work based on one-directional locking calculations, depending on the metrics changed; for more information on the calculation model used in this view see the supplemental Microsoft Excel file Assortment Planning – Sales Plan View
  • Plan Horizon vs. Receipt Planning Engine Horizon: if the Plan Horizon, set at the time of plan creation, extends beyond the receipt planning engine horizon for a given item, data that extends past the receipt engine horizon will appear in blue italics with zero values ($0.00, for example). See example in Plan Horizon vs. Replen Forecast Horizon below).
  • Replen Placeholders/Collections: when a placeholder/collection is of the Status Replen, the system pulls all relevant data to form its sales plan from the Forecasts module. Users should be aware of the following differences in this view when dealing with Replen items:
  • Read Only: all normally editable metrics in Sales Plan view become read only, being populated directly from aggregated replenishment forecast data, with the exception of all Shrink and Inventory Adjustments related metrics, which remain editable and can still be planned normally
  • Plan Horizon vs. Replen Forecast Horizon: if the Plan Horizon, set at the time of plan creation, extends beyond the replenishment forecast horizon for a given item, the system will conditionally format the periods that go beyond the replenishment forecast as shown below to draw the user’s attention to this.   • Toolbar Functions
  • Clear Sales Plan Overrides: this button reverts all user metrics back to the values of their System counterparts; this also reinstates all periods as valid Selling Periods (see Selling Period description below)
  • Note: metrics without system counterparts (the Shrink and Adjustments metrics, for example) cannot be reverted using this function
  • Note: to revert a single cell to the value of its System counterpart, simply select the desired cell, press the Delete key, then click Apply
  • Apply: saves all changes made since the last Apply
  • Cancel: cancels all changes made since the last Apply and reverts them to their previously saved values
  • Export: standard Pivot Grid functionality; exports and saves the Pivot Grid in many common formats including Microsoft Excel and Adobe PDF
  • Open: standard Pivot Grid functionality; opens the Pivot Grid in many common formats including Microsoft Excel and Adobe PDF – does not require save prior to viewing in the desired format
  • Graph: standard Pivot Grid functionality; graphs the selected data in the view
  • Fields: standard Pivot Grid functionality; allows the user to customize the field layout for the view
  • Custom layouts can be saved and restored by right-clicking in the Pivot Grid and selecting Save Layout or Restore Layout, respectively
  • Selling Period: user editable checkbox; selling period allows the user to specify periods where sales are not planned for the selected placeholder/collection but is still in phase
  • Unchecking zeros out System Sales, System Sales Units, Sales, and Sales Units for the period, thus metrics in the Main Planning and Cluster Set views ARE updated by this change; users will not be able to enter sales for unchecked periods
  • When unchecked, inventory (BOP, EOP) still calculates normally for the period
  • When unchecked, receipts can still be realized for the period
  • When unchecked, all Inventory Adjustments metrics (see below) can still be planned normally
  • Note: common uses for this feature might be items that repeat seasonally, such as holiday themed goods, where you will still retain inventory in the items between selling periods
  • Seasonality Metrics
  • Seasonal Index: non-editable decimal (1.01); seasonal index for the period based on the assigned seasonality curve
  • Promotion Index: non-editable decimal (1.01); combined promotional index for all applied promotions in the period
  • System Effective Index: non-editable decimal (1.01); index applied to period rate of sale to calculate period sales
  • Sales Metrics
  • Sales: user editable* currency ($100); planned/actual sales revenue for the period
  • Calculation: (Sales Units * AUR)
  • Sales Build Ratio: non-editable decimal (1.01); Ratio indicating the relationship of Sales of the current period to the previous period
  • Calculation: (Sales / Previous Period Sales)
  • Sales Units: user editable* integer; planned/actual sales units for the period
  • Calculation: (Sales / AUR)
  • Sales Units Build Ratio: non-editable decimal (1.01); Ratio indicating the relationship of Sales Units of the current period to the previous period
  • Calculation: (Sales Units / Previous Period Sales Units)
  • AUT: non-editable currency ($1.01); average unit ticket price for the period
  • Calculation: N/A
  • AUR: non-editable currency ($1.01); average unit retail price for the period
  • Calculation: (Sales / Sales Units)
  • Sellthrough: non-editable percentage (1.01%); percentage of inventory sold for the period
  • Calculation: (Sales Units / BOP Units)
  • Cost of Sales Metrics
  • COGS: non-editable currency ($100); cost of goods sold for the period
  • Calculation: (Sales Units * AUC)
  • AUC: non-editable currency ($1.01); average unit cost price for the period
  • Calculation: (Avg BOP Unit Cost)
  • Margin Metrics
  • Margin: non-editable currency ($100); planned/actual margin revenue (profit) for the period
  • Calculation: (Sales ex VAT - COGS)
  • Margin % non-editable percentage (1.01%); planned/actual margin percentage for the period
  • Calculation: (Margin / Sales ex VAT)
  • BOP Inventory Metrics
  • BOP Units: non-editable integer; planned/actual beginning of period inventory units
  • Calculation: (Previous Period EOP)
  • BOP Cost: non-editable currency ($100); planned/actual beginning of period inventory cost
  • Calculation: (Previous Period EOP Cost)
  • Avg BOP Unit Cost: non-editable currency ($1.01); average beginning of period cost per inventory unit
  • Calculation: (BOP Cost / BOP Units)
  • Receipts Metrics: note that receipt metrics are not editable in Sales Plan view because they are managed in greater detail in Receipt Plan view
  • Receipt Units: non-editable integer; planned/actual receipt units for the period
  • Calculation: From Receipt Plan
  • Receipt AUC: non-editable currency ($1.01); average cost per unit for receipts in the period
  • Calculation: From Receipt Plan
  • Receipt Cost: non-editable currency ($100); planned/actual cost of receipts for the period
  • Calculation: (Receipt Units * Receipt AUC)
  • Minimum Presentation: non-editable integer; Minimum units required to maintain floor presentation for each site selling the item
  • Calculation: From Receipt Plan
  • On Order Metrics
  • On Order Units: non-editable integer; confirmed units on order for the period
  • Calculation: (purchase order units not yet received but scheduled for the period)
  • On Order Units var to Receipt Units: non-editable integer; Difference of On Order Units to the planned/actual Receipt Units
  • Calculation: (On Order Units – Receipt Units)
  • On Order AUC: non-editable currency ($1.01); average cost per unit of inbound orders for the period
  • Calculation: (On Order Cost / On Order Units)
  • On Order Cost: non-editable currency ($100); cost of inbound orders for the period
  • Calculation: N/A
  • On Order Cost % +/- Receipt Cost: non-editable percentage (1.01%); percentage of On Order Cost to planned/actual Receipt Cost
  • Calculation: (On Order Cost / Receipt Cost)
  • Stock To Sales Metrics
  • Stock Cost to COGS: non-editable decimal (1.01); ratio expressing the relationship of BOP Cost to COGS
  • Calculation: (BOP Cost / COGS)
  • Stock Units to Sales Units: non-editable decimal (1.01) ; ratio expressing the relationship of BOP Units to Sales Units
  • Calculation: (BOP Units / Sales Units)
  • Inventory Adjustments Metrics
  • Shrink Units: user editable integer; planned/actual shrink units for the period
  • Calculation: (Shrink Units % of Sales Units * Sales Units)
  • Shrink Units % of Sales Units: user editable percentage (1.01%); percentage of sales units represented by shrink units for the period
  • Calculation: (Shrink Units / Sales Units)
  • Shrink Cost: user editable currency ($100); cost value of planned/actual shrink units for the period
  • Calculation: (Shrink Cost % of COGS * COGS)
  • Shrink Cost % of COGS: user editable percentage (1.01%); percentage of COGS represented by Shrink Cost for the period
  • Calculation: (Shrink Cost / COGS)
  • Inventory Adjustment Units: user editable integer; planned/actual inventory adjustment units for the period
  • Calculation: N/A
  • Inventory Adjustment Cost: user editable currency; planned/actual cost of inventory adjustments for the period
  • Calculation: N/A
  • EOP Inventory Metrics
  • EOP Units: non-editable integer; planned/actual end of period inventory units
  • Calculation: (BOP Units – Sales Units + Receipt Units – Shrink Units + Inventory Adjustment Units)
  • EOP Cost: non-editable currency ($100); planned/actual end of period inventory cost
  • Calculation: (BOP Cost – Sales Cost + Receipt Cost – Shrink Cost + Inventory Adjustment Cost)
  • Avg EOP Unit Cost: non-editable currency ($1.01); average end of period cost per inventory unit
  • Calculation: (EOP Cost / EOP Units)
  • VAT Metrics
  • VAT: non-editable currency ($100); value added tax value of planned/actual Sales for the period
  • Calculation: (Sales - (Sales / (1 + VAT %)))
  • VAT %: non-editable percentage (1.01%); value added tax rate of the planned/actual Sales for the period
  • Calculation: (VAT / Sales ex VAT)
