Reconciliation Parameters

The Reconciliation Report is controlled by eight parameters:

  • Product Hierarchy: user editable tree multi-select ; selects the levels of the common product hierarchy between Comparison Plan 1 and Comparison Plan 2 at which to compare the reconciliation metrics
  • Site Hierarchy: user editable dropdown; selects the level of the common site hierarchy between Comparison Plan 1 and Comparison Plan 2 at which to compare the reconciliation metrics
  • Cluster Sets: user editable multi-select dropdown; selects the cluster sets to include in the current view
  • While cluster sets are not explicitly part of the Site Hierarchy, they are directly related, in that cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy; because of this, the Cluster Set dropdown will populate with the cluster sets contained beneath the selected Site Hierarchy levels
  • Window of Time: user editable tree multi-select; selects the window of time to compare for Comparison Plan 1 and Comparison Plan 2
  • Warning: the selected timeframe must be a contiguous range of periods; if it is not, the system will still use the window of time defined by the first and last selected periods
  • Comparison Type 1:  user editable dropdown; selects the plan type of Comparison Plan 1 – either Assortment Plan or Merchandise Financial Plan
  • Note:  this selection will determine how the variance measures are calculated in the report.  For example, if the assortment plan is selected as Comparison type 1 and the financial plan is selected as Comparison type 2, the assortment plan will be the first variable in the calculation that determines the variance.
  • Comparison Plan 1: user editable dropdown; selects the first plan or snapshot to compare in the view
  • Comparison Type 2: user editable dropdown; selects the plan type of Comparison Plan 2 – either Assortment Plan or Merchandise Financial Plan
  • Note:  this selection will determine how the variance measures are calculated in the report.  For example, if the assortment plan is selected as Comparison type 2 and the financial plan is selected as Comparison type 1, the assortment plan will be the second variable in the calculation that determines the variance.
  • Comparison Plan 2:  user editable dropdown; selects the first plan or snapshot to compare in the view
  • Display Currency: user editable dropdown; sets the currency to display for all revenue based fields in the view; defaults to the Home Currency for the active Assortment Plan
