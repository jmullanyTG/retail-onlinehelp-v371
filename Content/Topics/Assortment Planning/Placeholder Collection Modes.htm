Placeholder & Collection Modes

Header Area Controls

  • New: launches the Add Placeholders dialog (not available in Collection mode)
  • To add new collections, use the New button in Placeholder mode or in Variant view of Collection mode and follow the instructions under Coll Code, below
  • New: clicking adds a single placeholder line
  • Delete: deletes the selected placeholder line
  • # of Placeholders: user editable integer; sets the number of new placeholder lines to add when Generate is clicked
  • Mirror Options: user editable checkbox; when checked, copies the selections made for the topmost placeholder line to any beneath it
  • Note: it is not necessary to uncheck this option if you want to deviate from the selections made in the topmost placeholder line; simply edit as you would normally and the box will uncheck automatically
  • Current Collection: user editable checkbox; when checked, automatically sets Coll Code and Coll Desc to the values for the collection of the placeholder that was selected when the Add Placeholders dialog was launched
  • Generate: creates the number of new placeholder lines specified in # of Placeholders
  • Error: if # of Placeholders is omitted or zero, returns the following error:
  • Coll Code: user editable dropdown; selects the collection code to which the placeholder line will be assigned
  • Note: select a collection by double-clicking; type in the space below Code or Desc of the dropdown to filter the results returned; this acts as a “starts with” filter by default, but can accept “%” as a wildcard to do “contains” type filtering instead
  • Note: new collections can be created on the fly if you don’t wish to assign the placeholder to an existing one; simply enter a description for the collection in the space provided and click New
  • Coll Desc: non-editable alpha-numeric; the corresponding description for Coll Code for the placeholder line
  • PH Code: non-editable alpha-numeric; unique system generated code for the placeholder line
  • PH Desc: user editable alpha-numeric; sets the description for the placeholder line
  • Product Hierarchy: user editable single-select tree; sets the product hierarchy level to which the placeholder line will be assigned
  • Note: this defaults to the lowest common product hierarchy level present in the Product filter in the data selection parameters for Placeholder Mode
  • Cluster Set: user editable multi-select; selects the cluster sets to which the placeholder line will be assigned
  • Note: defaults to the cluster set(s) assigned to the Product Hierarchy level in Assortment Plan Defaults, if any
  • Note: new placeholders are assorted to all clusters within the assigned cluster sets by default
  • Phase In: user editable period picker; selects the phase in date for the placeholder
  • Selections limit to periods prior to Phase Out, if set
  • Note: defaults to the earliest non-actualized period within the Window of Time in the data selection parameters
  • Phase Out: user editable period picker; selects the phase out date for the placeholder
  • Selections limit to periods after Phase In, if set
  • Note: defaults to the last period within the Window of Time in the data selection parameters
  • Ticket Price: user editable currency ($10.01); sets the home currency AUT (average unit ticket) for the placeholder line
  • Like PH: user editable dropdown; selects another placeholder item to use as a Like Style from which the system can seed a sales plan; not required
  • Populates with placeholder items that exist within the selected Product Hierarchy for the line
  • Note: type in the space below Code, Desc, Phase In, or Phase Out of the dropdown to filter the results returned; this acts as a “starts with” filter by default, but can accept “%” as a wildcard to do “contains” type filtering instead
  • Note: If set, this will copy the Like PH’s Size Option, Curve Mode, Seasonality Curve, and AUT (unless specified in Ticket Price), can calculate a sales plan based on sales present for the selected Like PH between the Comp Start and End dates specified
  • Attrib Copy: user editable checkbox; only valid if Like PH is set; defines whether any product attributes assigned to the Like PH will be copied to the new placeholder line or if any defined attribute defaults for the Product Hierarchy level will be assigned instead (see Attribute Management Standard Documentation for more details)
  • Mod Factor: user editable decimal (1.01); only valid if Like PH is set; defines a factor multiply the sales history of the Like PH by; defaults to 1.0
  • Comp Start: user editable period picker; only valid if Like PH is set; defines the start date for the desired window of time from which to capture sales for the Like PH
  • Selections limit to periods prior to Comp End, if set
  • Defaults to the Phase In of the Like PH
  • Comp End: user editable period picker; only valid if Like PH is set; defines the end date for the desired window of time from which to capture sales for the Like PH
  • Selections limit to periods after Comp Start, if set
  • Defaults to the Phase Out of the Like PH
  • Split: launches the Split Placeholders dialog (not available in Collection mode)
  • Placeholder Code: non-editable alpha-numeric; code of the placeholder to be split
  • Placeholder Description: non-editable alpha-numeric; description of the placeholder to be split
  • Number of Sets: non-editable integer; sets the number of placeholders the selected placeholder will be split into
  • Example: a placeholder with phase window of 11/1-11/30/2016 has a # of sets of 2 applied - the original line will now have a phase window of 11/1-11/15/2016 and the additional placeholder created will have a phase window of time of 11/15-11/30/2016)
  • When applying a # of sets that doesn’t evenly divide into the # of periods in a placeholder’s lifecycle, system will allocate 1 additional period onto each created placeholder in set order until the remainder is used (ex. 15 periods, 4 sets = 4 total placeholders, 3 with 5 periods, 1 with 4 periods)
  • When applying a # of sets greater than the # of periods in a placeholder’s lifecycle, system will default to a number of placeholders equal to the number of periods, with each being allotted one period (ex. 9 periods, 10 sets = 9 total placeholders, each with a 1 period lifecycle)
  • Placeholders resulting from splits will calculate their new sales plans and related metrics based on the sales by period of the original placeholder and the specific periods allotted to each new placeholder:

  • Cancel: closes the Split Placeholders dialog without making any changes to the selected placeholder
  • Apply: splits the selected placeholder into the specified Number of Sets per the logic above
  • The original placeholder remains intact after a split, only changing its phase dates and related metrics
  • Additional placeholders resulting from the Split function copy the original PH code and description but have “(Split #)” appended to the code, where # is the number of the split in sequence, and “(Split from PH ID, number #)” appended to the description, where PH ID is the unique system ID for the placeholder, and # is, again, the number of the split in sequence
  • Additional placeholders will retain all non-sales related properties of the original placeholder (product level, min pres, seasonality curve, size option, attributes, etc)
  • Delete: deletes all selected placeholder/collection rows
  • Made permanent on Apply
  • To undo, use Undo > Deletions rather than Cancel
  • Note: Committed placeholders/collections cannot be deleted
  • Undo:
  • Selected Row Changes: undoes changes made prior to Apply for the selected rows only
  • Use this when you have made changes to multiple rows but only want to undo certain rows – using Cancel will undo all changes made since the last Apply
  • Deletions: undoes all deletions of placeholder/collection rows made since the last Apply
  • Use this method instead of clicking Cancel to undo row deletions
  • Unassign Selected: unassigns variants attached to the selected placeholder rows
  • Not available at header level in Collection mode – use in Placeholder mode or Variant view of Collection mode
  • If any product attributes were attached to the variants being unassigned, system will revert the overwritten attribute values back to their pre-variant assignment states
  • You cannot unassign variants from placeholders in Committed status
  • Locations: launches the Manage Locations dialog (not available in Collection mode)
  • use this dialog to exclude specific sites from clusters to which a placeholder is assorted and adjust their sales plans accordingly

  • users can make exceptions for more than one placeholder at once by multi-selecting them in the Main Planning View prior to clicking the Locations button; items selected will be stacked in both the top and bottom sections of the Manage Locations dialog

  • Top section fields
  • Code: non-editable alpha-numeric; displays the system codes for all selected placeholders
  • Description: non-editable alpha-numeric; displays the description the placeholder
  • Cluster Set: non-editable alpha-numeric; list of the cluster sets the placeholder is assorted to
  • Cluster: non-editable alpha-numeric; list of the clusters within each cluster set the placeholder is assorted to
  • Location Count: non-editable integer; count of the sites currently included in the corresponding cluster’s assortment of the placeholder, based on exclusions made in the bottom section of the dialog
  • Note: this field reflects the actual number of assorted sites within a cluster for the placeholder, but the Site Count metric in the main view will not update based on exclusions made here. This is because, on Apply of exclusions, the cluster level rates of sale are recalculated based on the % contribution of included sites and decreasing Site Count would cause double-decrementing of the related sales plan
  • ROS $: non-editable currency ($10.00); the current cluster rate of sale (average sales per site, per period) in revenue for the placeholder
  • ROS U: non-editable decimal (10.00); the current cluster rate of sale (average sales per site, per period) in units for the placeholder
  • Adj ROS $: non-editable currency ($10.00); the adjusted cluster rate of sale in revenue for the placeholder, based on site exclusions made in the lower section of the dialog
  • Calculation: cluster ROS $ * sum ( Cluster % Cont. for all included sites in the cluster )
  • Adj ROS U: non-editable decimal (10.00); the adjusted cluster rate of sale in units for the placeholder, based on site exclusions made in the lower section of the dialog
  • Calculation: cluster ROS $ * sum ( Cluster % Cont. for all included sites in the cluster )
  • Bottom section fields:
  • Code: non-editable alpha-numeric; displays the system codes for all selected placeholders
  • Description: non-editable alpha-numeric; displays the description the placeholder
  • Cluster Set: list of the cluster sets the placeholder is assorted to
  • Cluster: list of the clusters within each cluster set the placeholder is assorted to
  • Loc ID: list of unique system identifiers for the sites contained in each cluster
  • Loc Desc: list of the corresponding site descriptions for the sites contained in each cluster

  • Cluster % Cont.: non-editable percentage (5.01%); derived from location % Cont tables based on assigned Product Hierarchy level of the placeholder

    • Note: the level of product hierarchy data used to calculate the location % contribution tables is set by the Level Contribution parameter for the current assortment plan. The sales included are determined by the Sales Types parameter for the current plan. See Manage Assortment Plans above for more information

    • Note: when calculating location % contribution for location exceptions, sales defined by the above mentioned parameters and that correspond to the LY periods of the lifecycle (periods from Phase in to Phase Out) of each placeholder/cluster are used
  • Include?: user editable checkbox; selects whether a site is planned to receive the placeholder or not; defaults to checked
  • Note: the sites displayed by cluster in this dialog are based on all effective cluster set snapshots for the placeholder over its life (or through the plan horizon, for placeholders without Phase Out dates). See Cluster Set Snapshot Validation above for more information
  • Cancel: closes the Manage Locations dialog without making any changes to the selected placeholders
  • Apply: saves the changes made to the selected placeholders and updates the Main Planning View ROS $ and ROS U fields with the Adj ROS $ and Adj ROS U values shown in the top section of the Manage Locations dialog

  • Images: launches the Image Gallery dialog for the selected placeholder (not available in Collection mode)

  • New: add a blank image space for the placeholder
  • Delete: deletes the selected image(s) from the placeholder
  • Carousel: allows the user to choose different view configurations for attached images
  • Image: display of images attached to the placeholder

  • Click an image then right click > Edit to launch the Image Maintenance dialog and change existing images or assign new ones to blank image spaces

  • Add: launches Windows Explorer file browser, allowing the user to navigate to images to add to the JustEnough Image Repository
  • Delete: deletes the selected images from the Image Repository
  • View: displays a larger version of the selected image

  • Attributes: displays additional information about the selected image

  • Filter: launches a filter picker that allows the user to select or create a new filter to limit the images from the Repository that are being shown
  • Refresh: refreshes the Repository view with any new images or information added since it was opened
  • Cancel: closes the Image Maintenance dialog without committing any changes to the Image Repository
  • Save: saves changes made to the Image Repository
  • Classification: displays the classification of each image
  • Note: the first image assigned to a placeholder is always the primary by default, but more images can be assigned with user defined classifications
  • Primary?: non-editable checkbox; indicates whether an image is the primary image to display whenever images are shown for the placeholder or not
  • Multiples?: non-editable checkbox; indicates whether multiple images are allowed for the given classification
  • Note: the Primary and Multiples parameters for variant images can be set in Manage > Variant Manager > Image Maintenance
  • View Options:
  • Column Chooser: opens the fields Customization dialog; users can drag fields to/from the main planning area and this dialog to customize the view
  • Custom layouts can be saved/restored by right-clicking in the Snapshot Templates area and selecting Save Layout or Restore Layout, respectively

  • Side Pane Choose which side pane you want to view. (You cannot have both open at the same time.)

    • Cluster Details:shows the Cluster Details pane at the right of the screen
    • Lifespan: shows the Lifespan pane at the right of the screen
    • None: Hides the side pane
  • Image Tooltip: this option toggles whether image tooltips appear when the user hovers the mouse over a thumbnail image in the main view (not available in Collection mode)

  • Receipts:

  • Recalc Req. %: displays the % of placeholders/collections in the current view that require receipt plan recalculation based on changes that have been made
  • this value is directly tied to the Receipt Status field in the main view; placeholders/collections with a Receipt Status of “Dirty” require recalculation
  • Recalculate: recalculates receipt plans for all placeholders/collections with a “Dirty” status in the Receipt Status field
  • Receipt plan default constraints can be set via Manage Assortment Plans > Defaults (see corresponding section above for details)
  • System policies affecting the Assortment Engine output can be set via Maintain > Policies > Engine: Commitments (see Line Visualization Supporting Views – Receipt Plan below for details)
  • Note: Receipt recalculations require complex calculations and, thus, are sent to the Workload Queue module for scheduling and processing. Though, often, receipts are recalculated immediately, they may be deferred for later calculation based on the number of placeholders requiring recalculation and other jobs currently in the queue. Placeholders awaiting recalculation will become locked and conditionally formatted to indicate this.
  • Note: receipts for placeholders/collections in Replen status are calculated in the Forecasts module and read in, thus they will never appear “Dirty” and will not be recalculated when this button is clicked
  • Important: all changes to your plan, including changes made to constraints in the Receipt Plan supporting view, must be saved prior to recalculating receipts or they will not be taken into account by the Receipt Engine.

  • Replen:

  • Recalculate: recalculates sales plans for placeholders/collections of Status “Replen” with a “Dirty” status in the Replen Status field
  • The date Replen placeholders/collections were last recalculated is shown in the Last Replen field
  • Replenishment recalculations require complex calculations and, thus, are sent to the Workload Queue module for scheduling and processing. Though, often, replenishment placeholders are recalculated immediately, they may be deferred for later calculation based on the number of placeholders requiring recalculation and other jobs currently in the queue. Placeholders awaiting recalculation will become locked and conditionally formatted to indicate this.
