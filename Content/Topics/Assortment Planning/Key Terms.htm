Key Terms

The following terms appear throughout this manual. For the sake of keeping the size of manual as manageable as possible and to avoid breaking the flow of the information presented herein, a list of these terms and explanations of each is provided below for your reference.

  • Product Hierarchy – the structure of product levels within an organization; a simple example is a 3 level Hierarchy with the levels Category, Gender, and Class, where the Categories are Apparel, Accessories, and Footwear, the Genders and Male and Female, and the Classes include things like knit tops, short sleeve tees, etc.
  • Site Hierarchy – the structure of location levels within an organization; a simple example is a 3 level Hierarchy with the levels Company, Channel, and District, where Company represents all locations, the Channels are Retail, Wholesale, and Ecommerce, and Districts include things like Northwest, Southeast, etc.
  • Period – the smallest unit of time being used within the assortment plan; typically, period represents a single week, however, it is also possible to plan in months or other periods as defined during your product implementation
  • Period Horizon – the number of periods forward in time an assortment plan will calculate by default; used in cases where the end of a timeframe is undefined, such as when Phase Out is not set for a placeholder
  • Note: this does not mean that you cannot have plans further out than this many periods from the current date
  • Snapshot – a save of the assortment plan metric values at a specific point in time; used in a variety of places throughout Assortment Planning as a means of comparing prior plan states with your working plan.
  • Variant – an actual product to be planned and bought through the Assortment Planning process; these are often referred to as style-colors or options
  • Placeholder – a non-specific product to which a variant will be assigned later; users can work with placeholders throughout Assortment Planning as if they were variants
  • Collection – a group of placeholders that can be planned as a single product; these are often referred to as styles or key items
  • Merchandise Financial Plan – a planned set of financial, inventory, and other targets by period, typically created at a level of the Product Hierarchy above Style and at a level of the Site Hierarchy above Site/Store; can be from JustEnough MFP or external planning systems
  • Target Creation – a process through which users can establish plan targets for specific areas of their business and determine the number of items required to meet those targets
  • Scenario – a specific set of user defined variables, including product, location, and time dimensions, used to arrive at a probable outcome; relevant for both Target Creation and Rate of Sale Calculator
  • Plan Basis – a merchandise financial plan used as the basis for a Target Creation scenario
  • Product Filter – a collection of products defined by user established criteria
  • Window of Time – a contiguous range of periods used to define the data returned in a view or an analysis
  • Target Segment – a user defined subdivision of a product hierarchy level
  • Cluster – a group of stores defined by user established criteria within Cluster Site Manager
  • Cluster Set – a collection of clusters
  • Cluster Site Manager – the JustEnough module in which users can create cluster sets and clusters
  • Cluster Set Snapshot – the configuration of stores within the clusters of a cluster set at a specific point in time
  • Rationalization – a process which recommends the number of placeholders and collections required to achieve a specified financial target based on user specified data
  • Smart Start – the process of generating placeholder items from a Target Creation scenario, assigning them to collections, and seeding them with initial sales plans
  • Phase In – the first period a product is planned to be available for sale in stores
  • Phase Out – the last period a product is planned to be available for sale in stores
  • Rate of Sale (ROS) – the average sales per store per period for a given product
  • Rate of Sale Calculator – a supporting module within the JustEnough suite of applications that is used to create rate of sale scenarios
  • Seasonality Curve – a collection of indices across a window of time used to define the magnitude of sales in each period as it relates to the rate of sale
  • Curves Module – a supporting module within the JustEnough suite of applications that is used to model seasonality curves given a set of user defined inputs
