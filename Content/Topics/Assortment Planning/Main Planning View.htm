    Expand or collapse groups by right-clicking the grouping section and selecting Full Expand or Full Collapse

  • Calculations: all calculations listed in this section are generic calcs only, provided for the purposes of checking the general accuracy, however, the actual calculations used may vary depending on what metric was changed to cause a recalculation; details can be found in the supplemental Line Visualization Calculation Behaviors Excel document

  • Collection Level Checkbox Controls: in Collection mode, checkbox controls, such as those used in the Active, Carryover, and Promo fields, will display a solid gold state to indicate that only some placeholders within the collection are checked
  • Active: user editable checkbox; sets whether the current placeholder/collection is active in the assortment plan or not

  • Defaults to checked; unchecking will strike through the placeholder/collection line and zero out its Collection % Cont

  • All aspects of the placeholder/collection sales plan are retained for seamless switching between active and inactive states
  • Inactive placeholders/collections are not included in any reporting within Assortment Planning: Target Creation, Assortment Rollup, Reconciliation, and Dashboards
  • Note: placeholders/collections can also be made active/inactive by right click > Set Active / Set Inactive
  • Note: placeholders/collections of Replen Status cannot be made Inactive
  • Locked: user editable checkbox; locks/unlocks the collection the placeholder belongs to (or the collection in Collection mode) for editing

  • When checked, all placeholders attached to the collection become locked and uneditable in the view and highlighted in pink/red depending on the row band

  • Actualization of sales in a period can still affect locked placeholders/collections
  • Creation of new cluster set snapshots for cluster sets to which locked collections are assorted can also change locked placeholders/collections
  • This is done to maintain synchronicity with ROS metrics and the site count associated with clusters to which locked placeholders/collections are assorted
  • Recalculation of linked Seasonality Curves will not affect locked placeholders/collections
  • Already active promotions will remain active on locked placeholders/collections; new promotions cannot be applied to locked placeholders
  • Note: placeholders/collections can also be locked by right click > Lock Selected / Unlock Selected
  • Product Level: user editable single-select tree; sets the level of the product hierarchy to which the placeholder/collection is assigned
  • This selection does not have to be made at the lowest level of the product hierarchy, though it is recommended users do so to avoid ambiguity in reporting

  • PH Collection Code: user editable dropdown; sets the collection to which the placeholder is assigned (non-editable in Collection mode)

  • Double-click a collection in the list to change the placeholder’s assignment
  • Filter the list by typing in the Code or Description fields; use % to do a “Contains” filter condition rather than “Starts With” (ex. %Tops returns everything containing the word “Tops”)
  • Enter a new Collection Desc and click New to create a new collection and assign the selected placeholder to it
  • Changing a placeholder’s collection assignment will recalculate both its prior collection’s and its newly assigned collection’s % contribution for all placeholders that belong to either
  • PH Coll Desc: user editable alpha-numeric; sets a description for the collection
  • Note: changing this for one placeholder changes it for all other placeholders currently assigned to that collection
  • PH Code: non editable alpha-numeric; unique system generated code for the placeholder (not present in Collection mode)
  • PH Desc: user editable alpha-numeric; sets a description for the placeholder (not present in Collection mode)
  • Placeholders generated through the Smart Start process default to “Smart Started from Segment ” & (segment name) for easy identification
  • Cluster Set: non-editable; displays the cluster sets the placeholder or collection assorts to, as set in the Cluster Details side pane.
  • Status: user editable dropdown; displays/sets the status for the placeholder collection (non-editable in Collection mode)
  • Open: user selectable; default for all new placeholders; denotes placeholders that can be deleted by Smart Starting an overlapping Target Creation scenario; see Target Creation above for more information
  • Created: user selectable; denotes placeholders that cannot be deleted by Smart Starting and overlapping Target Creation scenario
  • Assigned: auto-assigned upon assignment of a variant to the placeholder; assigned placeholders inherit the Variant ID, PLM Code, PLM Description, Color Code, Color Description, and Attributes of the assigned variant; see Line Visualization Supporting Views – Attributes for details on attribute inheritance
  • Note: placeholders whose variants are unassigned via Undo > Unassign Selected are automatically returned to Created status
  • Committed: auto-assigned upon confirmation of units on-order for the placeholder
  • Note: because actual units on-order will always be against a variant (real style-color), once a placeholder enters Committed status, the assigned variant cannot be removed, nor can the status be changed unless the on-order units are cancelled
  • Replen: user selectable for placeholders that in either Assigned or Committed Status; shifts control of the placeholder/collection sales plan to replenishment forecasts in the Forecasts module, making the plan read-only within Assortment Planning

  • Warning: changing placeholder/collection Status of placeholders with replenishment data present (see Available for Replen, below) to Replen will raise the following error message:

Clicking Yes will irreversibly overwrite the sales plan for the changed item with available replenishment forecast data. While you can change the item Status back, the system is not able to maintain the previous sales plan data. The state of the sales plan at the time of the change, including metric values, cluster assignments, and location exceptions, will become the go-forward sales plan, though the plan elements will become editable once more

  • Warning: changing placeholder/collection Status of placeholders without replenishment data present (see Available for Replen, below) to Replen will raise the following error message:

Clicking Yes will irreversibly zero out the sales plan for the changed item. While you can change the item Status back, the system is not able to maintain the previous sales plan data. Plan metrics will stay at zero, though the plan elements will become editable once more

  • Notes on Replen items:
  • Sales plans for Replen items will be updated nightly to coincide with updates to replenishment forecasts
  • Replen item sales plans can also be updated on demand by clicking the Recalculate button in the Replen section of the toolbar, however, be advised that this will utilize the settings of the most recent replenishment run; it is advisable to verify the output of the last replenishment run was created using the same parameters as are used in the nightly run or the data brought into AP may be inconsistent or incomplete
  • Because only future data is ever written in a replen run and data is written daily, this requires the system to specially handle inter-period sales and on-hand inventory. BOP on-hand will be created from actuals and not changed throughout the period; sales (and receipts) will be combined as "future periods from replenishment" and "past periods from actuals" to maintain a total period number throughout the period until it is fully actualized; EOP on-hand values will always come from replenishment data
  • Certain plan elements will still remain editable; most are just informational fields (PH Description & Code, Tags, Notes), but all Shrink/Inventory Adjustments metrics in Sales Plan View will also be editable
  • Replen items cannot be locked, made inactive, or deleted
  • Seasonality Curve, Size Curve, and Min Pres will be cleared, as these elements are controlled within the Forecasts module; if Status changes from Replen to something else, the item will inherit the defaults set on the plan for the product level to which it belongs
  • Any Sales Plan and Receipt Plan overrides will be irrevocably cleared
  • Receipt Plan constraints will be cleared, as these elements are controlled within the Forecasts module; if Status changes from Replen to something else, the item will inherit the defaults set on the plan for the product level to which it belongs
  • Important: currently, Forecasts uses buckets and Assortment Planning uses periods – these are two different time entity types that do not have to align in order for Assortment Planning to read in replenishment forecast data. The system will use BucketBeginDate BETWEEN PeriodStartDate and PeriodEndDate to try to match the calendars in each module, however, if the buckets and periods do not coincinde, this may lead to sub-optimal data within Assortment Planning; for example, if the buckets in Forecasts are set to monthly and the periods in AP are weekly, AP will put the sales for the entire monthly bucket into the first weekly period available during that month, etc.

  • Variant ID: non-editable numeric; unique system identifier for the assigned variant; blank if no variant is assigned (not present in Collection mode)
  • PLM Code: non-editable alpha-numeric; if integrated with a PLM (product lifecycle management) system, displays the PLM code associated with the assigned variant (not present in Collection mode)
  • PLM Desc: non-editable alpha-numeric; if integrated with a PLM (product lifecycle management) system, displays the PLM description associated with the assigned variant (not present in Collection mode)
  • Color Code: user editable alpha numeric; displays/sets the color code for the placeholder (not present in Collection mode)
  • When a variant is assigned, overwrites any user-entered value and displays the variant color code
  • On unassigning a variant, reverts back to the user-entered value, if any
  • Color Desc: user editable alpha numeric; displays/sets the color description for the placeholder (not present in Collection mode)
  • When a variant is assigned, overwrites any user-entered value and displays the variant color description
  • On unassigning a variant, reverts back to the user-entered value, if any
  • Supplier Code: non-editable integer; displays the unique system supplier code of the supplier associated with the variant assigned to the placeholder, if any
  • Supplier Name: non-editable alpha numeric; displays the supplier name of the supplier associated with the variant assigned to the placeholder, if any
  • Default Lead Days: non-editable integer; displays either the Default Lead Time Days set for the Product Level the placeholder is assigned to or the Default Lead Time of the supplier associated with the variant assigned to the placeholder
  • Note: if a variant is assigned to the placeholder but that variant has no associated supplier, the system will use the Default Lead Time Days set in the plan defaults (see Line Visualization > Essential Business Concepts > Tracking Against Ordering Deadlines above for more information on setting default lead days)
  • Next Order Date: non-editable date (10/2/2017); displays the earliest order deadline for receipts planned for the corresponding placeholder based on assigned default lead times; essentially, this is the earliest receipt date for any cluster level receipt planned for the placeholder minus Default Lead Days
  • Note: this should also correspond with the earliest Order By Date shown in Receipt Plan view for the placeholder
  • Size Option: user editable dropdown; selects the size curve to apply to the placeholder/collection
  • Displays all defined size curves
  • Filter by typing in the Size Curve Option or Description fields; use % to do a “Contains” filter condition rather than “Starts With” (ex. %Tops returns everything containing the word “Tops”)
  • Warning: be aware that applying at the collection level will blind-push a size curve down to all placeholders in a collection, as the system has no way to validate the relevance of sizes contained in the applied curve to new placeholders
  • Curve Mode: user editable dropdown; sets the mode of the selected Seasonality Curve
  • Static: the selected curve is applied exactly as it exists at the time of application and the placeholder/collection’s curve will not change if changes are made to the curve, whether manually in the Curves module or through dynamic recalculation. Think of this as making a copy of the applied curve.
  • Dynamic: changes made to the curve will be pushed down to the placeholder/collection via automated system processes, and the dependent sales plans will recalculate when those changes are pushed.
  • Seasonality Curve: user editable select control; displays/sets the seasonality curve applied to the placeholder/collection;
  • Used to break down Ttl Sls $ and Ttl Sls U by period in Sales Plan View; see Line Visualization Supporting Views – Sales Plan for more information
  • Note: only curves of the Manual or Specified Conditions Using Enhanced Filtering types will be available for selection within Assortment Planning

  • click the in-cell ellipsis ( ) to launch the Curve Selection dialog

  • Tags: user editable multi-select dropdown; displays/sets the tags used to filter the curves returned for selection; making it easier for the user to find the desired one
  • Note: the maximum number of curves returned based on the selected tags is set by the system policy Max Selectable Lookup Curves, located in Maintain > Policies > Module: Curves; this is done to prevent slowdown in the user interface
  • Helpful Hint: assigning meaningful tags to curves upon generation will help avoid situations where the desired curve may be truncated from the list returned for selection
  • Curve Name: non-editable alpha numeric; displays the name of the corresponding curve
  • Curve Generation Type: non-editable alpha numeric; displays the method used to generate the corresponding curve, Manual or Specified Conditions Using Enhanced Filtering
  • Bucket Type: non-editable alpha numeric; displays the bucket type for the curve, usually weekly or monthly
  • Bucket Start Date: non-editable date (10/3/2017); displays the start date of the first bucket within the curve Start Date and End Date of the corresponding curve
  • Note: technically, this can be before Start Date if Start Date falls in the middle or at the end of the first bucket
  • Start Date: non-editable date (10/3/2017); displays the start date of the corresponding curve
  • End Date: non-editable date (10/3/2017); displays the end date of the corresponding curve
  • Number of Periods: non-editable integer; displays the number of time periods (buckets) encompassed by the curve
  • Existing: non-editable checkbox; indicates that the corresponding curve is already assigned to one or more placeholders in the current view; provided for ease of assignment of curves that have already been used for other existing placeholders
  • Tags: non-editable checkbox; indicates that the corresponding curve has one or more of the selected Tags attached to it
  • Image: user editable image control; displays a thumbnail of the primary image for the placeholder or attached variant (not present in Collection mode)
  • If Image Tooltip is enabled in View Options, hovering over the thumbnail image displays a larger version of the image
  • To edit, click the image, right click and choose Edit Primary or Edit All
  • Edit Primary displays the Image Maintenance dialog (see Header Area Controls > Images earlier in this section) and allows selection of another primary image
  • Edit All displays the Image Gallery dialog (see Header Area Controls > Images earlier in this section) and allows editing/deletion of existing images or addition of more images; this can also be launched by double-clicking the image thumbnail
  • Images can be assigned to placeholders prior to variants being attached, but these will be replaced by the actual variant images once a variant is attached. To retain placeholder images when attaching variants, go to Variant Manager > Image Maintenance for the variant you wish to attach and add the placeholder images there, or add them after the variant has been attached.
  • Note: in implementations where a PLM (Product Lifecycle Management) system integration is present, the primary image of a placeholder will not be editable once a variant has been assigned, as this image will be dictated via data imported from the PLM. The ability to edit the primary image is controlled by the system policy Variant Primary Image Classification Update and can be accessed via Maintain > Policies > Module: Assortment
  • Carryover: non-editable checkbox; indicates whether a placeholder/collection is an already existing item from a previous window of time
  • This is driven solely off a placeholder status of Committed; Committed placeholders/collections have confirmed on order quantities for actual variants, and cannot be edited to the same degree as placeholders/collections that have yet to be ordered
  • Collection % Cont: user editable percentage (10.1%); displays/sets the contribution percentage of a placeholder to the collection to which it belongs (not present in Collection mode)
  • Placeholders in collections by themselves will always have Collection % Cont of 100%
  • When shifting placeholder collection assignments, percentages for all placeholders in affected collections are recalculated, but sales hold for all
  • When adjusting Collection % Cont directly, collection level sales hold and placeholder sales recalculate
  • As changes are made to this metric within a collection, the % Locked field will be checked, so subsequent changes won’t alter your previous entries within the collection
  • Sum of Collection % Cont for all placeholders in a collection must equal 100%; system will re-normalize entries that would cause the sum to exceed by dividing each by the sum of the percentages
  • Calculation: Ttl Sls U / (Ttl Sls U for all placeholders in collection)
  • Error: entering values above 100% will raise an Invalid Value error
  • % Locked: non-editable checkbox; indicates a change has been made to the corresponding % contribution that has not yet been Applied (not present in Collection mode)

  • AUT: user editable currency ($10.01); the average unit ticket price for the placeholder/collection

    • Displays the weighted average of AUT values for all periods, both actualized and future, beneath

    • AUT is assumed to be VAT inclusive and is treated as such for all dependent calculations

    • Because of the potential for multiple currencies, VAT (value added tax) rates, and promotional participation to vary by site within a single cluster, changes to AUT require you to click Apply to properly calculate the effects on the rest of the plan for a given placeholder/collection. Details for this calculation are available in a supplemental Excel model.

    • Period level changes to AUT can be made in the AUT right pane view. See AUT View for more information.

    • Notes:

      • Once a placeholder has begun to actualize (current date is greater than Phase In date), AUT is not directly editable.

      • AUR (average unit retail), being an outcome of the calculation mentioned above, is not directly editable.

  • Margin %: non-editable percentage (10.01%); gross margin percentage for the placeholder/collection
  • Calculation: ((AUR ex VAT / (1 + VAT %)) – AUC) / (AUR ex VAT/ 1 + VAT %)
  • Phase In: user editable period picker; selects the phase in date for the placeholder/collection
  • Selections limit to periods prior to Phase Out, if set
  • Note: defaults to the earliest non-actualized period within the Window of Time in the data selection parameters
  • Mkdn Date: user editable period picker; selects the markdown date for the placeholder/collection
  • Selections limit to periods from Phase In through Phase Out
  • Mkdn date tells the system which period to stop respecting the various constraints utilized in the Receipt Plan supporting view, including Min Pres
  • When viewing the by-period plan for the placeholder in Sales Plan or Receipt Plan view, Min Pres will drop to zero in the period specified by Mkdn Date
  • Note: Mkdn date can be pushed by changing Phase In or Phase Out
  • Phase Out: user editable period picker; selects the phase out date for the placeholder/collection
  • Selections limit to periods after Phase In, if set
  • Note: defaults to the last period within the Window of Time in the data selection parameters
  • Note: for placeholders/collections of Replen Status, Phase Out dates are derived from the maximum Phase Out date set in the Forecasting module for all relevant inventories
  • SPV Overrides: non-editable checkbox; indicates if overrides to Sales $ or Sales U have been made to the placeholder/collection in the Sales Plan supporting view
  • Note: changes made in Sales Plan view do not directly affect the metrics in Line Visualization, with the exception of removing Selling Periods, as these metrics are considered the System Plan when working in Sales Plan view; this allows the user to see differences in the higher level placeholder and cluster plans created in Line Visualization and the results of more granular period level adjustments made in Sales Plan view
  • SPV Sls $: non-editable currency ($10.01); displays the sum of all Sales for the placeholder/collection in Sales Plan view
  • Provided for easy comparison to the Line Visualization plan at the placeholder/collection level
  • SPV Sls U: non-editable integer; displays the sum of all Sales Units for the placeholder/collection in Sales Plan view
  • Provided for easy comparison to the Line Visualization plan at the placeholder/collection level
  • ROS $: user editable currency ($10.01); sets the rate of sale (average sales per site per period) in revenue for the placeholder/collection
  • Calculation: (Ttl Sls $ / No. Periods / Site Count)
  • ROS U: user editable integer; sets the rate of sale (average sales per site per period) in units for the placeholder/collection
  • Calculation: (Ttl Sls U / No. Periods / Site Count)
  • Promo: non-editable checkbox; indicates whether or not there are any active promotions applied to the placeholder/collection
  • Checked means at least one promotion is applied to the placeholder/collection; unchecked means no promotions are applied
  • Inactive promotions will not cause the box to be checked
  • Specific promotions applied can be viewed in the Promotions bottom pane view (see Line Visualization Supporting Views – Promotions for details)
  • Ttl Sls $: user editable currency ($10.01); sets the total sales revenue for the placeholder/collection from Phase In through Phase Out
  • Calculation: (ROS $ * No. Periods * Site Count)
  • Ttl Sls U: user editable integer; sets the total sales units for the placeholder/collection from Phase In through Phase Out
  • Calculation: (ROS U * No. Periods * Site Count)
  • AUR: non-editable currency ($10.01); displays the average unit retail price for the placeholder/collection
  • Because of the potential for multiple currencies, VAT (value added tax) rates, and promotional participation to vary by site within a single cluster, AUR is not directly editable and is only changed as a result of changes to AUT, Phase In, Phase Out, and promotion applications. Details for this calculation are available in a supplemental Excel model.
  • Calculation: (Ttl Sls $ / Ttl Sls U)
  • AUR ex VAT: Non-editable currency ($1.01); average retail (selling) price excluding VAT per unit for the placeholder item
  • Calculation: AUR / (1 + VAT %)
  • AUC: user editable currency ($10.01); sets the average unit cost price for the placeholder/collection
  • Displays the weighted average of AUC values for all periods, both actualized and future, beneath
  • Note: once a placeholder/collection has begun to actualize (current date is greater than Phase In date), AUC will not directly editable.
  • Min Pres: user editable integer; sets the desired minimum presentation stock per site for the placeholder/collection
  • Min Pres represents additional non-sales units to be purchased for the placeholder/collection to maintain adequate stock for presentation within each store
  • Calculation: SUM (cluster site count * cluster min pres – for all clusters) / Site Count (yields a weighted average)
  • Ttl Buy $: user editable currency ($10.01); sets the total buy in revenue for the placeholder/collection
  • Changes to this metric directly affect the placeholder/collection sales plan
  • Calculation: (Ttl Sls U + (Min Pres * Site Count) + Additional Buy) * AUC
  • Ttl Buy U: user editable integer; sets the total buy in revenue for the placeholder/collection
  • Changes to this metric directly affect the placeholder/collection sales plan
  • Calculation: (Ttl Sls U + (Min Pres * Site Count) + Additional Buy)
  • Additional Buy: user editable integer; sets the quantity of additional non-sales and non-min pres units to buy for the placeholder/collection
  • The primary function of this field is to show discrepancies between the sum of Ttl Sls U and (Min Pres * Site Count) and Ttl Buy U that can occur because of the ability to directly edit S/T %
  • To remove unwanted excess units and rebalance a placeholder/collection’s sales plan, simply zero this field out and Apply
  • Calculation: (Ttl Buy U – Ttl Sls U – (Min Pres * Site Count))
  • S/T %: user editable percentage (10.1%); sets the target sell-through percentage for the placeholder/collection
  • Changes to this metric alter Ttl Buy U to achieve the entered S/T %
  • Calculation: (Ttl Sls U / Ttl Buy U)
  • Total Orders ($): non-editable currency ($10.01); sum of Total Orders Cost from Receipt Plan view for the placeholder/collection; included as a comparison of the actual receipt plan to the estimated Ttl Buy $ in the Main Planning View
  • Total Orders (U): non-editable integer; sum of Total Orders from Receipt Plan view for the placeholder/collection; included as a comparison of the actual receipt plan to the estimated Ttl Buy U in the Main Planning View
  • No. Periods: non-editable integer; displays the MAX period count for the placeholder/collection
  • Because Phase In and Phase Out at the placeholder/collection level are set to the earliest and latest periods, respectively, of all clusters to which it has been assorted, No. Periods must calculate this way to balance the other calculations in Line Visualization
  • Example: I have a placeholder assorted to two clusters – cluster A has Phase In/Phase Out of Wk 1 and Wk 4, and cluster B has a Phase In/Phase Out of Wk 2 and Wk 5, the placeholder Phase In/Phase Out will be Wk 1 and Wk 5, yielding a No. Periods of 5

  • Site Count (Active): non-editable decimal (10.1); displays the average total site count and the average active site count (in parenthesis) of all clusters a placeholder/collection is assorted to between Phase In and Phase Out

  • Typically, this metric will always be a static whole number that equates to the sum of the cluster level site counts to which the placeholder/collection is assorted, however, when multiple cluster set snapshots are present during a placeholder/collection’s life, it is possible for site count by cluster to vary, thus necessitating the need for an average count
  • Location Exceptions: non-editable checkbox; indicates the presence of location level exceptions for the placeholder/collection
  • If any sites contained in clusters the placeholder/collection is assorted to have been excluded via the Manage Locations function, this box will be checked
  • Note: Location exceptions WILL NOT alter Site Count; this is because, when location exceptions are made, the system recalculates the placeholder/collection rates of sale according to each site’s % contribution to the cluster; also decrementing Site Count would cause the resulting sales plan to drop too low, as doing so would essentially double-count the removal of the excluded sites
  • Notes: user editable alpha-numeric textbox; allows the user to write text notes about the placeholder/collection which are saved and can be viewed throughout the planning process
  • Lock Type: non-editable alpha-numeric; if any part of a placeholder/collection line is locked, displays the lock type
  • Receipt Status: non-editable alpha; displays the calculation status of the receipt plan for the placeholder/collection; Clean or Dirty
  • In the main planning and Cluster Set views, changes to Seasonality Curve, Collection % Contribution, Phase In, Mkdn Date, Phase Out, AUT, Ttl Sls $, Ttl Sls U, ROS $, ROS U, Min Pres, Ttl Buy $, Ttl Buy U, Additional Buy, or S/T will set this to Dirty
  • In Sales Plan view, changes to Selling Period, Sales, Sales Units, Shrink Units, Shrink Units % of Sales Units, Shrink Cost, Shrink Cost % of COGS, Inventory Adjustment Units and Inventory Adjustment Cost will set this to Dirty
  • In Receipt Plan view, changes to Periods of Supply, Periods Before Phase In, Min Stock to Sales, Shrink % of Sales, Minimum Presentation, Receipt Period, and Receipt Units will set this to Dirty
  • See Header Area Controls > Receipt Data earlier in this section for more information
  • Available for Replen: non-editable checkbox; indicates if replenishment forecast data is available for the placeholder/collection
  • Helpful Hint: check this field before changing the status of a placeholder/collection to Replen. The system will still warn you if no data is present, but checking first will save you time; it’s also useful for verifying that your replenishment forecast matches up with the assigned variant(s)
  • Important: this field will appear checked if ANY portion of the combination assigned variant(s), cluster sets, and product lifespan has a replenishment forecast against it. For example, the field will appear checked if a single item master related to the assigned variant has a forecast for a single site contained within the clusters that variant is assorted to during the lifespan of the variant. Make sure you verify that forecasts are set for all intended item masters and sites to ensure completeness of replenishment data in Assortment Planning
  • Replen Status: non-editable alpha numeric; displays the Replen Status, Clean or Dirty, as an indicator of if the placeholder/collection has been updated with the most recent Replenishment forecast data; valid for placeholders/collections of Replen status only
  • Last Replen: non-editable date (2/13/2018); displays the date replenishment data was last updated for the placeholder/collection; valid for placeholders/collections of Replen status only
  • Actual Sales $: non-editable currency; displays the sum of all actualized sales revenue (from historical periods) for the placeholder/collection
  • Actualization occurs as part of overnight processing (exact frequency can vary by implementation), in which the sales revenue that was realized for a given period is fed to the JustEnough database and read into Assortment Planning; planned sales values are overwritten by these actual sales values
  • Actual Sales U: non-editable integer; displays the sum of all actualized sales units (from historical periods) for the placeholder/collection
  • Actualization occurs as part of overnight processing (exact frequency can vary by implementation), in which the sales units that were realized for a given period are fed to the JustEnough database and read into Assortment Planning; planned sales values are overwritten by these actual sales values
  • Actualized Periods: non-editable integer; displays the number of actualized periods present in the life of the placeholder/collection
  • Example: I have a placeholder whose Phase In was Wk 1 and it is now Wk 5, assuming regular overnight processing has occurred, Actualized Periods will be 4
  • Actualized: non-editable checkbox; indicates the presence of any actualized periods during the placeholder/collection’s life

  • Tags: user editable alpha-numeric; allows the ad-hoc creation and assignment of informational descriptors for a given item

  • Multiple tags can be entered for a single line, with inputs being confirmed by the use of the comma ( , ) key
  • As users begin to type, any tags already present in the current view that begin with the entered charaters will appear in an auto-complete selector below the field; these entries can be selected via click or by using the arrow keys to navigate to the desired item and the comma key to select
