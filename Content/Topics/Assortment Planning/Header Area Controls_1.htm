Header Area Controls

  • Refresh: If a Plan Basis is set for the scenario, refreshes the Scenario Target based on the current plan data
  • Segment by Hierarchy: creates one target segment for each lowest level leaf of the selected product hierarchy for the scenario, define temporary Target Filters for each based on the items belonging to the corresponding hierarchy levels, and assign any default cluster sets you have defined through Manage Assortment Plans > Defaults (see corresponding section above for details)
  • Example: the selected product hierarchy level for the scenario is the department Mens Tops; beneath Mens Tops, there are 4 sub-departments: Mens Tees, Mens Wovens, Mens Knits, and Mens Sweaters. Clicking Segment by Hierarchy will create one segment for each in the Target Segment Definitions area
  • Note: if additional hierarchy levels exist between the product hierarchy level selected for the scenario and the lowest level in the product hierarchy set for the plan, the system will still generate segments for the leaves of the lowest level. If you want to use this functionality to create segments at a higher level, you must assign a different hierarchy to the plan that only goes down to the level desired
  • Warning: we strongly advise against defining product hierarchies that extend to the style level or below, as this will greatly diminish system performance and overcomplicate the assortment planning process
  • View Exclusions: launches the View Exclusions dialog

 

  • displays a list of all placeholder items present during the specified Comparison Time Frames for each segment which belong to the specified Product Hierarchy selection for the scenario but which are not included in any defined segments based on their Target Filter criteria
  • Product Hierarchy fields: Non-editable alpha-numeric; fields indicating the specific leaf of each product hierarchy level the item belongs to
  • Fields shown are based on the levels present in the product hierarchy for the current assortment plan
  • Collection Code: Non-editable alpha-numeric; user or system defined code for the collection containing the placeholder item
  • Collection Description: Non-editable alpha-numeric; user or system defined description for the collection containing the placeholder item
  • Placeholder Code: Non-editable alpha-numeric; user or system defined code for the placeholder item
  • Placeholder Description: Non-editable alpha-numeric; user or system defined description for the placeholder item
  • Color Code: Non-editable alpha-numeric; system color code for the placeholder item
  • Color Description: Non-editable alpha-numeric; system color description for the placeholder item
  • Image: Non-editable image thumbnail; image assigned to the corresponding variant item in the system
  • Images can be assigned to variant items through Manage > Variant Manager > Image Maintenance
  • Assigned Attributes: Non-editable alpha-numeric; delimited list of imported and/or user defined product attributes assigned to the placeholder item
  • Product attributes can be assigned directly to placeholders within the Assortment Plan through the right side Attributes panes in the Collection and Placeholder views or to variant items through Manage > Variant Manager > Variant Attribute Management
  • AUT: Non-editable currency ($1.01); average ticket price per unit for the placeholder item during the selected Comparison Time Frame
  • Note: AUT is assumed to be VAT inclusive for the purposes of all dependent calculations
  • AUC: Non-editable currency ($1.01); average cost per unit for the placeholder item during the selected Comparison Time Frame
  • Gross Margin: Non-editable percentage (1.01%); gross margin percentage for the placeholder item during the selected Comparison Time Frame
  • Calculation: (AUR – AUC) / AUR
  • Sales: Non-editable currency ($1.01); sales revenue for the placeholder item during the selected Comparison Time Frame
  • Sales Units: Non-editable integer; sales units for the placeholder item during the selected Comparison Time Frame
  • AUR: Non-editable currency ($1.01); average retail (selling) price per unit for the placeholder item during the selected Comparison Time Frame
  • Calculation: Sales / Sales Units
  • Manually Assigned Segment: User editable dropdown; segment to which the excluded placeholder item will be assigned for the purpose of scenario Calculation and Rationalization; not required
  • Populates with the names of all defined segments within the scenario
  • Apply: assigns all placeholder items with Manually Assigned Segment specified to those segments for the current scenario
  • Note: this will not update the filters specified in Target Filter for any segment – it makes only temporary assignments used within the scenario
  • Cancel: reverts any changes made to Manually Assigned Segment since the last Apply to their previous values
  • Calculate: Calculates the Segment Cluster Details section based on the Target Data Selection parameters and Target Segment Definitions you have set for the scenario
  • While calculating, the system will display the following dialog:

 

  • See Segment Cluster Details below for more specific information on calculations for each field
  • Note: generally, this process takes only a few seconds to complete, but depending on the volume of data being queried and the complexity of the specified criteria, it could take up to a minute or more.
  • Note: If Calculate is clicked before segments have been defined for the scenario, the Segment Cluster Details section will refresh but return no data
  • Rationalize: executes the Rationalization process to calculate recommended new choice counts by segment cluster based on the Target Data Selection parameters and Target Segment Definitions you have set for the scenario
  • While calculating, the system will display the following dialog:

 

 

  • See Rationalization below for more information on how the Rationalization calculation works to arrive at recommended choice counts
  • Note: generally, this process takes only a few seconds to complete, but depending on the volume of data being queried and the complexity of the specified criteria, it could take up to a minute or more.
  • Note: If Rationalize is clicked before segments have been defined for the scenario, the Segment Cluster Details section will refresh but return no recommended choice counts
  • Clear Overrides: Reverts any edits made to Target Percent Contribution, TY Act New Sales, TY Act New Sales U, TY Act New AUR, and TY Act New Choices in the Segment Cluster Details section back to their system generated values
  • Smart Start: Initiates the Smart Start process for the current scenario, generating the number of placeholder items required, and seeding with beginning sales plan data elements
  • For details, see the Smart Start section later in this document
  • View Data Cleansing Report: launches the Data Cleansing dialog

 

  • Segments: user editable multi-select; selects the segments from the active scenario to include in the report
  • Cluster Sets: user editable multi-select; selects the cluster sets from those assigned to segments within the active scenario to include in the report
  • Clusters: user editable multi-select; selects the clusters from those cluster sets assigned to segments within the active scenario to include in the report
  • Parameters: shows/hides the Parameters area of the report
  • Execute: refreshes the report based on the currently selected parameters
  • Target Data Cleansing Settings: this section displays the data cleansing parameters used for each segment for reference purposes; the background color of the threshold fields in this section correspond to data points that were cleansed or altered as a result of each

 

  • Segment: non-editable alpha-numeric; displays the segment name
  • Average Inclusion Threshold: non-editable percentage (10.00%); corresponds to the system policy of the same name; displays the % of average applied in step 1 of the data cleansing process (see Data Cleansing section below) to remove data points with extreme low sales

 

  • Median Period Total Threshold: non-editable percentage (10.00%); corresponds to the system policy of the same name; displays the % of median applied in step 2 of the data cleansing process (see Data Cleansing section below) to raise periods falling below to more consistent levels prior to deseasonalization

 

  • Active Variant Threshold Factor: non-editable percentage (10.00%); corresponds to the system policy of the same name; displays the % of max applied in step 3 of the data cleansing process (see Data Cleansing section below) to remove outlier data points

 

  • Active Variant Time On Offer Minimum: non-editable percentage (10.00%); corresponds to the system policy of the same name; displays the % of the scenario window of time applied in step 4 of the data cleansing process (see Data Cleansing section below) to remove variants without a sufficient number of data points remaining after steps 1-3

 

  • Target Data Cleansing – Pre-Deseasonalization: this section displays raw sales data for all variants included in the segment, cluster set and cluster Parameters selections and shows those data points removed or altered by steps 1 and 2 of the data cleansing process prior to deseasonalization (see Data Cleansing section below)

 

  • Segment: non-editable alpha-numeric; displays the segment name
  • Cluster Set: non-editable alpha-numeric; displays the name of the cluster set assigned to the segment
  • Cluster: non-editable alpha-numeric; displays the name of the cluster within the indicated cluster set
  • Variant: non-editable alpha-numeric; displays the variant code
  • Variant Description: non-editable alpha-numeric; displays the variant description
  • Total: non-editable integer or currency (depending on Sales Basis of units or revenue); displays the total raw sales that occurred in the corresponding cluster across the scenario window of time for the variant
  • Maximum: non-editable integer or currency (depending on Sales Basis of units or revenue); displays the maximum period raw sales that occurred in the corresponding cluster across the scenario window of time for the variant (used in step 1 of data cleansing)
  • Period Data Fields: non-editable integer or currency (depending on Sales Basis of units or revenue); displays the raw sales per period that occurred in the corresponding cluster for the variant
  • Target Data Cleansing – Post-Deseasonalization: this section displays deseasonalized sales data for all variants included in the segment, cluster set and cluster Parameters selections and shows those data points removed by steps 3 and 4 of the data cleansing process prior to deseasonalization (see Data Cleansing section below); the conditional formatting for data points removed by steps 1 also shows in this view to provide enhanced context when viewing the additional edits

 

  • Segment: non-editable alpha-numeric; displays the segment name
  • Cluster Set: non-editable alpha-numeric; displays the name of the cluster set assigned to the segment
  • Cluster: non-editable alpha-numeric; displays the name of the cluster within the indicated cluster set
  • Variant: non-editable alpha-numeric; displays the variant code
  • Variant Description: non-editable alpha-numeric; displays the variant description
  • Total: non-editable integer or currency (depending on Sales Basis of units or revenue); displays the remaining (after data cleansing step 1) deseasonalized sales that occurred in the corresponding cluster across the scenario window of time for the variant
  • Average: non-editable integer or currency (depending on Sales Basis of units or revenue); displays the average period deseasonalized sales that occurred in the corresponding cluster across the scenario window of time for the variant (used in step 3 of data cleansing)
  • Period Data Fields: non-editable integer or currency (depending on Sales Basis of units or revenue); displays the deseasonalized sales per period that occurred in the corresponding cluster for the variant
  • Display Currency: user editable dropdown; sets the currency to display for all revenue based fields in the view; defaults to the Home Currency for the active Assortment Plan
  • Execute: refreshes the Target Creation dialog
