Image View

Use Image view to get a visual summary of your assortment plan, either by cluster or by period. Here, you can get a sense for how your assortment transforms over time. You can also make quick visual edits on the fly to see if the removal or certain items helps or hinders your visual goal.

INotes:

  • Image view is not available in Collection mode
  • The filter parameters in this view are linked with those in the Main Planning view; when parameters change and are executed there, this view will update and refresh accordingly; Note that it is possible to change the filter criteria within Image view to, for example, focus on a smaller window of time
  • Due to the large data size associated with image files, selecting too much data to show in the view can cause the system to become sluggush and take a long time to refresh; the system policy Image View Maximum number of SQL Data Rows, accessible via Maintain > Policies > Module: Assortment, can be used to limit the amount of data that can be brought back in Image View and help mitigate such issues; the view will raise an error the the data selection exceeds the value set in this policy

Toolbar Actions

  • View Options:
  • Quick Text: sets text display options for image cells within the view
  • Show Code: toggles the display of the placeholder code at the bottom of each cell; on by default
  • Attributes: select an Attribute Family defined in the Attribute Management module to display in the upper-right corner of each cell
  • AUC, Ticket Price, ROS $, ROS U, Ttl Sls $, Ttl Sls U, Ttl Buy $, Ttl Buy U: selects the metric to display in the upper-right corner of each cell
  • Receipt Status: displays the Receipt Status, Clean or Dirty, in the upper-right corner of each cell as an indicator of if the placeholder requires recalculation of its receipt plan
  • Replen Status: displays the Replen Status, Clean or Dirty, in the upper-right corner of each cell as an indicator of if the placeholder has been updated with the most recent Replenishment forecast data; valid for placeholders of Replen status only
  • Status: displays the placeholder Status, Open, Created, Assigned, Committed, or Replen, in the upper-right corner of each cell
  • Note: only one quick text option (attribute or metric), plus the placeholder code, can be selected to show at once
  • Placeholder Tracking: When enabled, the placeholder selected in the Main Planning view (and any supporting views you have open) automatically changes to the placeholder of the image you select in Image view. This makes it more convenient to edit placeholders while working in Image view, because the placeholder values shown in the other views are always in sync with the image you select. Click this option to toggle enabling/disabling placeholder tracking.
  • Inactive: sets the desired visual behavior when temporarily making items inactive in Image view

  • Parameters: shows/hides the parameters pane at the top of the view
  • Execute: refreshes Image view, based on the selected Product, Site Hierarchy, Cluster Sets, and Window of Time

Parameters

  • Product: user editable dropdown; selects a product filter for the current view
  •   this button launches the Filter Conditions dialog, allowing the user to dynamically edit the criteria for the selected filter prior to executing it; the option also exists to create a new filter from the adjusted criteria on the fly
  •   this button launches the Maintain Filter dialog, allowing the user to create or import a new filter, or update, delete, or export an existing filter
  • Site Hierarchy: user editable tree multi-select; selects the levels of the current Plan’s Site Hierarchy to include in the current view
  • Cluster Set: user editable multi-select dropdown; selects the cluster sets to include in the current view
  • While cluster sets are not explicitly part of the Site Hierarchy, they are directly related, in that cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy; because of this, the Cluster Set dropdown will populate with the cluster sets contained beneath the selected Site Hierarchy levels
  • Window of Time: user editable tree multi-select; selects the window of time to display in the current view
  • Note: any placeholder/collection whose life cycle intersects with any part of the selected Window of Time will be displayed in the view; this is to allow the user to work with placeholders/collections holistically, as displaying subdivisions of their life cycle might alter the decisions made by the user if they have forgotten or are unaware of the full length of time a placeholder/collection will be for sale
  • Warning: the selected timeframe must be a contiguous range of periods; if it is not, the system will still use the window of time defined by the first and last selected periods
  • Display Currency: Sets the currency displayed in Image view.

  • Grid Config: Sets the pivot fields for the Image view grid. Select which fields to use as rows and which to use as columns.

Working with Image View

  • Question Mark Cells: placeholders that don’t currently have images assigned, either through an attached variant, or images that have been assigned to the placeholders themselves, show as question marks in Image view

  • Making Cells Inactive: for the sake of quick visual assessments, users can make placeholders temporarily inactive in the view by double-clicking any cell; if that placeholder is present in more than one cell, either by cluster or period, all cells related to the placeholder will be made inactive automatically
  • Note: making placeholders temporarily inactive in this manner has no effect on the Active status of placeholders in the Main Planning view
  • Editing Images: to change the image of any placeholder in the view, simply right click and click Edit. This will launch the Repository Explorer (detailed earlier in this section), in which you can select a new image from the repository, delete images, or add new ones
  • Note: images assigned in this manner will be applied to all instances of the placeholder, whether by cluster or period, in the view
