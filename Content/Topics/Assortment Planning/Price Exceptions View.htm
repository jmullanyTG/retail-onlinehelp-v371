Price Exceptions View

This view allows users to manage price exceptions by placeholder for all of the currencies used in the clusters to which that placeholder is assorted. This view reads information from the Currency Exchange Rates, Price Translation Management, and Price Translation Hierarchy Management views located in Plan > Views > Currency to populate the fields shown but provides users with the flexibility to override any of them to set local ticket prices in whatever way is most meaningful to them

 

Note: The system policy Line Visualization Allow Price Exceptions, accessible via Maintain > Policies > Module: Assortment, sets whether or not this view will be viewable/editable. If set to False, the view will still appear but will be disabled; note that to affect a change in this policy on an already open plan, the plan must be reopened through the Manage Assortment Plans dialog

 

  • Reset Default Factors: restores default translation factors for all non-actualized periods for the product level of the selected placeholder (as defined in the Price Translation Management, and Price Translation Hierarchy Management views)
  • Note: restoring default factors will cause the Local AUT and Home AUT Equivalent fields to recalculate accordingly
  • Apply: Saves all changes made since the last Apply and initiates recalculation of blended AUR values for the selected placeholder
  • Cancel: cancels all changes made since the last Apply and reverts them to previously saved values
  • Period: indicates the period being changed; this aligns with the period type for the current assortment plan
  • Home AUT: displays the selected placeholder AUT in default currency for the active assortment plan (as defined in the plan settings – Manage > Default Currency)
  • Note: this field shows a weighted average AUT for all clusters for the period in question, thus, if differences exist by cluster and/or from period to period, values shown may not align with those shown in the main planning view or the AUT view
  • Home AUT (ex VAT): displays the selected placeholder AUT excluding VAT in the default currency for the active assortment plan (as defined in the plan settings – Manage > Default Currency); hidden by default
  • Calculation: Home AUT / (1 + VAT %)
  • Local AUT: displays/sets the ticket price in local currency (as indicated in the column section header – EUR in the shot above) for the selected placeholder for the period
  • Calculation: Local AUT ex VAT + (Local AUT ex VAT * VAT %)
  • Local AUT (ex VAT): displays/sets the ticket price excluding VAT in local currency (as indicated in the column section header – EUR in the shot above) for the selected placeholder for the period
  • Calculation: Home AUT ex VAT Equivalent * Exchange Rate
  • Home AUT (ex VAT) Equivalent: displays/sets the ticket price as a local equivalent price excluding VAT in the home currency after a translation factor has been applied for the selected placeholder for the period
  • Calculation: Home AUT ex VAT * Translation Factor
  • Translation Factor: displays/sets the translation factor to apply to determine the final ticket price in home and local currencies (as indicated in the column section header – EUR in the shot above) for the selected placeholder for the period
  • Calculation: Home AUT ex VAT Equivalent / Home AUT ex VAT
  • Conditional Formatting: orange shaded cells in the Price Exceptions view indicate a change in Local AUT from the prior period for the currency indicated by the column section header
