Parameters

  • Product Hierarchy: user editable dropdown; sets the product hierarchy level(s) a Target Creation scenario must contain to be shown in the Target Creation Scenarios section
  • Selectable levels are based on the intersection of the product hierarchy assigned to the current assortment plan and the product hierarchy permissions of the active user
  • Site Hierarchy: user editable dropdown; sets the site hierarchy level(s) a Target Creation scenario must contain to be shown in the Target Creation Scenarios section
  • Selectable levels are based on the intersection of the site hierarchy assigned to the current assortment plan and the site hierarchy permissions of the active user
  • Window of Time: user editable dropdown; sets the window of time a scenario must intersect with to be shown in the Target Creation Scenarios section
  • Selectable periods are based on those configured in the system down to the Period set for the current assortment plan
  • Display Currency: user editable dropdown; sets the currency to display for all revenue based fields in the view; defaults to the Home Currency for the active Assortment Plan
  • Execute: Returns a list of scenarios in the Target Creation Scenarios section based on the criteria specified in the Product Hierarchy, Window of Time, and Site Hierarchy dropdowns and the Privacy/Sharing Settings for the scenarios returned
  • New: Launches a blank Target Creation window to create a new scenario

 

  • For details, see the Target Creation section below
  • Copy button: Launches the Target Creation window, populated with the Target Data Selection parameters, Target Segment Definitions, and Segment Cluster Details from the selected scenario in the Target Creation Scenarios section

 

  • All Target Data Selection and Target Segment Definitions are editable in copied scenarios
  • For details, see the Target Creation section below
  • Helpful Hint: in order to unlock target segment definitions and cluster details for editing, copied scenarios must first be saved. To do this without changing the scenarioâ€™s core data structure, simply change the scenario name to a unique name then click Save

 

  • Edit button: Launches the Target Creation window, populated with all data from the selected scenario in the Target Creation Scenarios section

  • All parameters are editable in scenarios of status Created, Calculated, or Rationalized.
  • For scenarios of status Smart Started, Target Segment Definitions and Segment Cluster Details can be edited, but doing so will require recalculation, re-Rationalization, and re-Smart Starting for changes to take effect in the active assortment plan
  • For details, see the Target Creation section below
  • Delete button: Deletes the selected Target Creation scenario
  • Warning: User will be prompted to confirm the action before deletion is completed:

  • Compare button: Launches the Scenario Comparison window, in which users can compare two Target Creation scenarios

 

  • Standard JustEnough Pivot Grid functionality, including ability to graph selected data elements and Export or Open analysis in a variety of popular formats, including Microsoft Excel and Adobe PDF
  • Only prerequisite for scenario comparison is that scenarios be in Calculated, Rationalized, or Smart Started status
  • Comparisons of scenarios representing disparate parts of the product/site hierarchies and/or windows of time are possible - they are just of limited usefulness
  • For an explanation of metrics, see the Target Creation section below
  • Smart Start button: Initiates the Smart Start process for the selected scenario, generating the number of placeholder items required, and seeding with beginning sales plan data elements
  • For details, see the Smart Start section later in this document
  • Fields button: Launches the customization dialog

 

  • Standard JE functionality to add/remove columns from the current section
  • Search capability for all fields available for the section
  • Drag & drop functionality to and from the fields area
  • Layouts for this section can be saved and restored through the right-click menu
  • Apply button: Commits changes to Scenario Name, Scenario Description, Privacy, and Sharing Settings (the only fields editable in this view) to the database
  • Cancel button: Undoes any changes made to Scenario Name, Scenario Description, Privacy, and Sharing Settings prior to Apply
