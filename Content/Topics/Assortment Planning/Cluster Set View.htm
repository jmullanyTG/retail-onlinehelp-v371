Cluster Set View

Cluster Set view contains all the essential metrics for building a sales plan and allows users to make Cluster Set or Cluster level changes to the single placeholder/collection selected in the Main Planning view

 

  • Site Count (Active): non-editable decimal (10.1); displays the average total site count and the average active site count (in parenthesis) of the cluster set/cluster between Phase In and Phase Out

 

  • Typically, this metric will be a static whole number that equates to the sum of the sites in the cluster set or cluster, however, when multiple cluster set snapshots are present during a placeholder/collection’s life, it is possible for site count by cluster to vary over time, thus necessitating the need for an average count
  • AUC (Average Unit Cost): user editable currency ($10.01); sets the average unit cost price for the cluster set/cluster.

    • Displays the weighted average of AUC values for all periods, both actualized and future, beneath

    • Note: Once a placeholder has begun to actualize (current date is greater than Phase In date), AUC will not be directly editable
  • AUT (Average Unit Ticket): user editable currency ($10.01); the average unit ticket price for the cluster set/cluster

    • Displays the weighted average of AUT values for all periods, both actualized and future, beneath.

    • AUT is assumed to be VAT inclusive and is treated as such for all dependent calculations

    • Because of the potential for multiple currencies, VAT (value added tax) rates, and promotional participation to vary by site within a single cluster, changes to AUT require an Apply to properly calculate the effects on the rest of the plan for a given placeholder/collection. Details for this calculation are available in a supplemental Excel model.

    • Period level changes to AUT can be made in the AUT right pane view. See AUT View for more information.

      • Notes:

        • Once a placeholder has begun to actualize (current date is greater than Phase In date), AUT is no longer directly editable

        • AUR (average unit retail), being an outcome of the calculation mentioned above, is not directly editable

  • Gross Margin %: non-editable percentage (10.01%); gross margin percentage for the cluster set/cluster
  • Calculation: (AUR ex VAT – AUC) / AUR ex VAT
  • Phase In: user editable period picker; selects the phase in date for the cluster set/cluster
  • Selections limit to periods prior to Phase Out, if set
  • Note: defaults to the earliest non-actualized period within the Window of Time in the data selection parameters
  • Mkdn Date (Markdown Date): user editable period picker; selects the markdown date for the cluster set/cluster
  • Selections limit to periods from Phase In through Phase Out
  • Mkdn date tells the system which period to stop respecting the various constraints utilized in the Receipt Plan supporting view, including Min Pres
  • When viewing the by-period plan for the cluster set/cluster in Sales Plan or Receipt Plan view, Min Pres will drop to zero in the period specified by Mkdn Date
  • Note: Mkdn date can be pushed by changing Phase In or Phase Out
  • Phase Out: user editable period picker; selects the phase out date for the cluster set/cluster
  • Selections limit to periods after Phase In, if set
  • Note: defaults to the last period within the Window of Time in the data selection parameters
  • Note: for placeholders/collections of Replen Status, Phase Out dates are derived from the maximum Phase Out date set in the Forecasting module for all relevant inventories
  • Curve Mode: user editable dropdown; sets the mode of the selected Seasonality Curve
  • Static: the selected curve is applied exactly as it exists at the time of application and the cluster set/cluster’s curve will not change if changes are made to the curve, whether manually in the Curves module or through dynamic recalculation. Think of this as making a copy of the applied curve
  • Dynamic: changes made to the curve will be pushed down to the cluster set/cluster via automated system processes, and the dependent sales plans will recalculate when those changes are pushed
  • Seasonality Curve: user editable dropdown; sets the seasonality curve applied to the cluster set/cluster
  • Used to break down Ttl Sls $ and Ttl Sls U by period in Sales Plan View; see Line Visualization Supporting Views – Sales Plan for more information
  • Note: curves must be generated with the Enhanced Filter type to be eligible for use within Assortment Planning and selectable here
  • click the in-cell ellipsis ( ) to launch the Curve Selection dialog

  • Tags: user editable multi-select dropdown; displays/sets the tags used to filter the curves returned for selection; making it easier for the user to find the desired one
  • Note: the maximum number of curves returned based on the selected tags is set by the system policy Max Selectable Lookup Curves, located in Maintain > Policies > Module: Curves; this is done to prevent slowdown in the user interface
  • Helpful Hint: assigning meaningful tags to curves upon generation will help avoid situations where the desired curve may be truncated from the list returned for selection
  • Curve Name: non-editable alpha numeric; displays the name of the corresponding curve
  • Curve Generation Type: non-editable alpha numeric; displays the method used to generate the corresponding curve, Manual or Specified Conditions Using Enhanced Filtering
  • Bucket Type: non-editable alpha numeric; displays the bucket type for the curve, usually weekly or monthly
  • Bucket Start Date: non-editable date (10/3/2017); displays the start date of the first bucket within the curve Start Date and End Date of the corresponding curve
  • Note: technically, this can be before Start Date if Start Date falls in the middle or at the end of the first bucket
  • Start Date: non-editable date (10/3/2017); displays the start date of the corresponding curve
  • End Date: non-editable date (10/3/2017); displays the end date of the corresponding curve
  • Number of Periods: non-editable integer; displays the number of time periods (buckets) encompassed by the curve
  • Existing: non-editable checkbox; indicates that the corresponding curve is already assigned to one or more placeholders in the current view; provided for ease of assignment of curves that have already been used for other existing placeholders
  • Tags: non-editable checkbox; indicates that the corresponding curve has one or more of the selected Tags attached to it
  • ROS $ (Rate of Sale Revenue): user editable currency ($10.01); sets the rate of sale (average sales per site per period) in revenue for the cluster set/cluster
  • Calculation: (Ttl Sls $ / No. Periods / Site Count)
  • ROS U (Rate of Sale Units): user editable integer; sets the rate of sale (average sales per site per period) in units for the cluster set/cluster
  • Calculation: (Ttl Sls U / No. Periods / Site Count)
  • Ttl Sls $ (Total Sales Revenue): user editable currency ($10.01); sets the total sales revenue for the cluster set/cluster from Phase In through Phase Out
  • Calculation: (ROS $ * No. Periods * Site Count)
  • Ttl Sls U (Total Sales Units): user editable integer; sets the total sales units for the cluster set/cluster from Phase In through Phase Out
  • Calculation: (ROS U * No. Periods * Site Count)
  • AUR (Average Unit Retail): non-editable currency ($10.01); displays the average unit retail price for the cluster set/cluster
  • Because of the potential for multiple currencies, VAT (value added tax) rates, and promotional participation to vary by site within a single cluster, AUR is not directly editable and is only changed as a result of changes to AUT, Phase In, Phase Out, and promotion applications. Details for this calculation are available in a supplemental Excel model
  • Calculation: (Ttl Sls $ / Ttl Sls U)
  • AUR ex VAT: Non-editable currency ($1.01); average retail (selling) price excluding VAT per unit for the placeholder item
  • Calculation: AUR / (1 + VAT %)
  • Min Pres (Minimum Presentation): user editable integer; sets the desired minimum presentation stock per site for the cluster set/cluster
  • Min Pres represents additional non-sales units to be purchased for the cluster set/cluster to maintain adequate stock for presentation within each store
  • Calculation: SUM (cluster site count * cluster min pres – for all clusters within a set) / cluster set Site Count (yields a weighted average) or (cluster site count * cluster min pres) for single clusters
  • Ttl Buy $ (Total Buy Cost Value): user editable currency ($10.01); sets the total buy in revenue for the cluster set/cluster
  • Changes to this metric directly affect the cluster set/cluster sales plan
  • Calculation: (Ttl Sls U + (Min Pres * Site Count) + Additional Buy) * AUC
  • Ttl Buy U (Total Buy Units): user editable integer; sets the total buy in revenue for the cluster set/cluster
  • Changes to this metric directly affect the cluster set/cluster sales plan
  • Calculation: (Ttl Sls U + (Min Pres * Site Count) + Additional Buy)
  • Additional Buy (Additional Buy Units): user editable integer; sets the quantity of additional non-sales and non-min pres units to buy for the cluster set/cluster
  • The primary function of this field is to show discrepancies between the sum of Ttl Sls U and (Min Pres * Site Count) and Ttl Buy U that can occur because of the ability to directly edit S/T %
  • To remove unwanted excess units and rebalance a cluster set/cluster’s sales plan, simply zero this field out and Apply
  • Calculation: (Ttl Buy U – Ttl Sls U – (Min Pres * Site Count))
  • S/T % (Sell-Through Percentage): user editable percentage (10.1%); sets the target sell-through percentage for the cluster set/cluster
  • Changes to this metric alter Ttl Sls U in relation to Ttl Buy U to achieve the entered S/T %
  • Calculation: (Ttl Sls U / Ttl Buy U)
  • Receipt ($): non-editable currency ($10.01); sum of Receipt Cost from Receipt Plan view for the placeholder/collection at cluster level; included as a comparison of the actual receipt plan to the estimated Ttl Buy $ in Cluster Set View
  • Receipt (U): non-editable integer; sum of Receipt Units from Receipt Plan view for the placeholder/collection at cluster level; included as a comparison of the actual receipt plan to the estimated Ttl Buy U in Cluster Set View
  • Units Per Location: non-editable integer; displays the average units per location for a cluster set/cluster based on Ttl Buy U and Site Count
  • Calculation: (Ttl Buy U / Site Count)
