Attributes View

The Attributes supporting view allows users to view and manage product attributes for placeholders/collections; see Attribute Management Standard Documentation for more information on Attribute Families, Values, and how their eligibility is set for items within the product hierarchy

 

  • Family Name: non-editable alpha numeric; displays the name of the attribute family
  • Family Description: non-editable alpha numeric; displays the description for the attribute family
  • Single Values: user editable dropdown; displays/sets the attribute value of the adjacent single-select attribute family for the selected placeholder/collection
  • when multiple placeholders/collections are selected that have different attribute values assigned, this field will show “(Multi)” to indicate this
  • Multiple Values: user editable multi-select; displays/sets the attribute values of the adjacent multi-select attribute family for the selected placeholder/collection
  • when multiple placeholders/collections are selected that have different attribute values assigned, this field will show the combination of all values present
  • Apply: saves all changes made to attribute values for the selected placeholder/collection since the last Apply
  • Cancel: cancels all changes made to attribute values for the selected placeholder/collection since the last Apply and reverts them to previously saved values
  • Data Population: the view will populate with all attribute families that any of the selected placeholders/collections is eligible for; attribute values available for selection will include all values that ANY of the selected items is eligible for
  • Attribute family and value eligibility can be set via Maintain > Attribute Management
  • Applying Changes Across Multiple Items: whether multiple lines are selected in the Main Planning view, changes are being applied in Collection mode, or both, the system will automatically resolve any conflicts that exist between the Applied values and the attribute value eligibility of the items the change is being applied to
  • Single-select attribute value changes will be applied to all placeholders/collections that are eligible for them; ineligible placeholders/collections will remain unchanged
  • Multi-select attribute value changes will be applied additively to placeholders/collections that are eligible for them; if only some of the applied values are eligible, those will be applied; ineligible placeholders/collections will remain unchanged
  • Dual-directional Attribute Flow: depending on where attributes are assigned – to placeholders in Assortment Planning or to variants in Variant Manager – attributes are transferred when a variant is assigned to a placeholder
  • Assigning a variant – any product attributes attached to the variant are transferred to the placeholder; any attributes that were already assigned to the placeholder but that don’t conflict with the variant attributes (i.e. single-select families that the variant already had assigned), they will remain on the placeholder and also be transferred to the variant
  • Note: a system policy exists that governs the inheritance of multi-value attribute families when variants are assigned; it can be found in Maintain > Policies > Module: Assortment > Multi Value Attribute Inheritance
  • When set to FALSE, the default setting, multi-value families will be applied additively to existing values when variants are assigned
  • Example: I have a placeholder with the multi-value family Embellishments and the values Flocked and Sequinned currently set; I assign a variant with the Embellishments values Flocked and Embroidered to that placeholder; the resulting values for the Embellishments family are: Flocked, Sequinned, and Embroidered
  • When set to TRUE, multi value families will overwrite the existing values when variants are assigned
  • Example: using the same case as the example above, the resulting values for Embellishments would be: Flocked and  Embroidered only
  • Attribute changes in Assortment Planning – a system policy exists that governs whether changes are allowed to attribute families attached to variants; this policy can be found in Maintain > Policies > Module: Assortment > Update Variant Attributes In Assortment Planning
  • When set to FALSE, all attribute families attached to the assigned variant will appear grayed out and will be locked for editing

  • Attribute changes in Variant Manager – changes made to variant attributes in Variant Manager will be transferred to all placeholders that variant is currently assigned to across all existing assortment plans that have the selected Variant Hierarchy assigned
