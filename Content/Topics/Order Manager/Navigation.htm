Navigation

The Navigation pane on the left-hand side of Order Manager serves as a launch pad for viewing and working with existing order documents. The convenient folder system also helps users organize their orders as they move through the various stages of the ordering process.

  • Filtering the Document List: by clicking the filter button in the top right of the Navigation header area, users can apply inventory or variant type filters to the order document list, effectively allowing them to search for orders containing specific site/item combinations or variants

  • Filter Entity Type: user editable dropdown; selects an entity type for the filter to be applied; either inventory (the combination of site and item master) or variant
  • Filter: user editable dropdown; selects a filter of products with which to filter the document list
  • : this button launches the Maintain Filter dialog, allowing the user to create or import a new filter, or update, delete, or export an existing filter
  • Select only some items from the current filter: launches the Item Selection dialog

 

  • The dialog will list all variants returned by the criteria in the selected filter; select the desired variants to include in the document by clicking the corresponding checkboxes in the Selected column
  • Items can also be selected using the buttons at the top left
  • : selects all rows currently highlighted
  • : de-selects all rows currently highlighted
  • : selects ALL rows
  • : de-selects ALL rows
  • Filter the list by typing in the filter row for any column

 

  • Typing in the filter row produces a “Starts With” filter effect by default
  • Typing % at the start of the value you want to filter for produces a “Contains” type filter instead (as shown above)
  • Choose Column(s): launches the Choose Fields Dialog, in which users can select additional columns to include in the Item Selection view to provide additional context for determining which products are desired

  • Cancel: closes the Item Selection dialog without selecting any items
  • OK: applies the filter with the selected items to the document list
  • Cancel: closes the filter selection dialog and does not filter the document list
  • OK:  applies the filter to the document list
  • Removing filters from the Document List: clicking this button in the upper right of the Navigation header area removes any filter that has been applied to the document list; only enabled when a filter is applied
  • Search Bar: user editable alpha-numeric; just below the Navigation header, enter text in this bar to apply a “Begins With” type filter on the orders in the Navigation pane
  • Workspace: contains all order documents that are not currently in one of the other document states
  • Public: contains all orders with the security setting of Public for which the user has sufficient permissions to view
  • Shared: contains all orders with the security setting of Shared for which the current user was included in the approved user list
  • Private: contains all orders with the security setting of Private that were created by the current user
  • Forwarded: in Order Manager, order documents can be forwarded to other users for review/approval; this folder contains all orders that have been forwarded either by or to the current user
  • Inbox: contains all orders forwarded from others to the current user
  • Outbox: contains all orders forwarded by the current user to others
  • Rejected: orders forwarded to others for approval can be rejected, indicating that changes are required; this folder contains all orders previously forwarded by the current user that were rejected
  • Pre-Authorized: given sufficient permissions based on the active workflow for an order document, users have the ability to pre-authorize, locking the order document from further edits; this folder contains all orders pre-authorized by the current user
  • Note: Pre-Authorization is provided in Order Manager more for the reservation of stock for orders that will ship to sites; pre-authorization in the context of commitment orders does not reserve any inventory because the inventory has not been ordered. If pre-authorization is not desired within a user workflow, the rights to pre-authorization can simply be removed at each node of the relevant workflow for all users in Workflow Manager
  • Authorized: given sufficient permissions based on the active workflow for an order document, users have the ability authorize, confirming that no further changes are required to the order and that is it ready for upload to external systems; this folder contains all order documents authorized by the current user
  • Uploading: based on the configuration and schedule of order uploads established during software implementation, authorized orders are uploaded to the external systems that handle purchase order management; this folder contains all order documents authorized by the current user that are currently being uploaded
  • Upload Failed: this folder contains all order documents authorized by the current user that experienced a failure during the upload process
  • Uploaded: this folder contains all order documents authorized by the current user that were successfully uploaded to external systems
  • Upload Verified: this folder contains all order documents authorized by the current user for which JustEnough has received confirmation of upload from external systems
  • Notes:
  • The system can be configured to transition the order document in one of two ways after an upload is verified:
  • Transition back to Workspace
  • Transition to the last folder the order was in prior to being Authorized
  • Additionally, the system can be configured to route the order document in one of two ways after an upload is verified:
  • Route to the user who created it
  • Route to the user who authorized it
  • Order Document Right-Click Actions: depending on the folder an order document resides in, a few actions are available by selecting and then right-clicking the order
  • Open: opens the order document in a new tab in the order viewing area at the right
  • Delete: launches a dialog asking the user for confirmation to delete the currently selected order

 

  • Yes: permanently deletes the selected order document
  • No: closes the delete confirmation dialog without deleting the selected order document
  • Navigation Pane Positioning & Viewing Options
  • Auto-Hide: clicking in the upper right of the Navigation pane toggles the pane to Auto-Hide mode, causing the pane to retract to a tab on the left side of the screen

 

To restore the pane to its normal position, simply hover over or click the tab then click the pin again to dock the pane

  • Manual Docking: the Navigation pane alternatively docked on the right side of the screen (or back on the left, if it was already on the right) by left-clicking and holding and then dragging the Navigation header area; the pane will temporarily become a floating window which can then be dragged to their the right or left side of the screen using the docking arrows that appear
