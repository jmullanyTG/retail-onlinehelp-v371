Add Assortment Plan Order Dialog Box

  • Source: non-editable alpha-numeric; indicates the type of order being created; always sets to Assortment Plan Commitment when clicking the Commitment button
  • Workflow: user-editable dropdown; sets the active workflow for the order documents created upon clicking OK
  • Note: available workflows are based on user permissions set in the Workflow module
  • Important: although an inactive flag exists within Workflow Manager to designate workflows that are no longer in play, this feature has not yet been integrated with Order Manager as of 3.6.0, thus, you will still see workflows flagged as inactive as selections in this dropdown
  • Name: user editable alpha-numeric; sets the name for the order or group of orders created
  • Note: the system also appends the values from the Assortment Plan, Product, and Owner fields, as well as the receipt date, to the end of the name specified for ease of identification
  • Description: user editable alpha-numeric; sets an optional description for the orders created
  • Owner: non-editable alpha-numeric; displays the UserID of the user who will initially own the created orders; defaults to the current user
  • Security: user editable dropdown; defines which users will have access to view/change the created orders
  • Public: all users with sufficient permissions can view/edit
  • Private: only the owner of the created order(s) can view/edit
  • Shared: only specified users have access to view/edit
  • Change: launches the Maintain Users and Roles dialog; only active when Security is set to Shared

 

  • Filter Field: type in this field to filter the list of Roles and Users below; filters using a “Contains” type filter by default
  • Orientation Button ( ): changes the orientation of the Roles and Users selection area from a standard multi-select to a tree style multi-select as show below:

 

  • Cancel: closes with Maintain Users and Roles dialog without setting/changing the list of authorized users
  • OK: sets/changes the list of authorized users to those selected in the Maintain Users and Roles dialog
  • Assortment Plan: user editable dropdown; selects the assortment plan from which to create commitment orders
  • Product: user editable dropdown; selects a product filter to create commitment orders for
  •   this button launches the Filter Conditions dialog, allowing the user to dynamically edit the criteria for the selected filter prior to executing it; the option also exists to create a new filter from the adjusted criteria on the fly
  •   this button launches the Maintain Filter dialog, allowing the user to create or import a new filter, or update, delete, or export an existing filter
  • Site Hierarchy: user editable tree multi-select; selects the levels of the Assortment Plan’s Site Hierarchy to create commitment orders for
  • Cluster Set: user editable multi-select dropdown; selects the cluster sets to create commitment orders for
  • While cluster sets are not explicitly part of the Site Hierarchy, they are directly related, in that cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy; because of this, the Cluster Set dropdown will populate with the cluster sets contained beneath the selected Site Hierarchy levels
  • Window of Time: user editable tree multi-select; selects the window of time to create commitment orders for
  • Note: only receipts planned within the specific window of time selected will be captured in the created orders
  • Warning: the selected timeframe must be a contiguous range of periods; if it is not, the system will still use the window of time defined by the first and last selected periods
  • Cancel: Closes the Add Assortment Plan Order dialog without creating any orders
  • OK: creates one order document per receipt date for all planned receipts captured by the specified criteria
  • Error: if no receipts are captured by the criteria you specify, you will receive the following error message:

This may occur because the receipts you are trying to capture do not have the Commit flag checked in Receipt Plan view, because the placeholders in question do not have size curves assigned, or because you have already created commitment orders for them. If you wish to re-create a commitment order, you must delete the already existing commitment order within Order Manager.
