Practical Application

The integration of Assortment Planning with Order Manager and Workflow creates a great deal of flexibility in fleshing out order details and communicating those both within your organization and externally to ERP/POM systems and suppliers. Here are a few tips to help you get the most out of the system:

  • Workflow and Order Document Approvals – a common practice in the retail/wholesale industry is to have approvals at multiple stages in the ordering process, potentially submitting order information to suppliers several times throughout, with increased detail in each submission
  • Example: I have an order for Men’s Tops with a 6 month lead time from order submission to delivery in the DC. The supplier for all items on the order only requires initial unit commitments up front, with sizing and prepacking details being provided 4 months and 2 months prior to delivery, respectively. Internally, my organization requires review by the planning manager before submission of order details to suppliers

When defining the workflow that will govern this order, the three main nodes to be created are Initial Commitment, Break By Size, and Prepack. For each node, you have the ability to determine what rights a given user has, thus, you would set it up so that your regular planner users only have rights to do the required analysis step (Break By Size or Prepack in this example) and to Forward the document to their manager for review. In the same node, you would then assign rights to the manager to Authorize if the order is ready to upload and to Reject to send it back to the person who forwarded it if changes are required (see the JustEnough Workflow Manager Standard Documentation for more information)

 

As each node is completed, the cycle of analysis, forward, authorize, and upload starts over until all nodes in the workflow have been completed. By setting Completion Days Out for each node (in the example above, this would equate to roughly 180 days for Initial Commitment, 120 days for Break By Size, and 60 days for Prepack) you can track your progress and upcoming submission due dates for the order and ensure you provide the added order details in enough time to meet the requirements of your supplier. These due dates can be reviewed and actioned against in the Assortment Upcoming Commitments Dashboard within Assortment Planning (See Assortment Planning Dashboard - Assortment Upcoming Commitments Dashboard above for details).

  • Expedited Workflow: another common use case in the industry is an opportunity, or at-once, buy, where you must work on a condensed time scale that does not allow for the standard approval process normally used in your organization
  • Example: I have a rush order for Men’s Tops that I need to receive in 3 months from order submission to delivery in the DC. I have spoken with the product supplier and they have agreed to an accelerated timeline, but asked that I provide all order details, including sizing and prepack information, up front

Notice that the three main nodes, Initial Commitment, Break By Size, and Prepack, still remain. Because of the condensed timeline, however, a single individual will be responsible for both Analysis and Authorization of order documents and only one Authorization will occur after all order details have been defined.

 

It is possible to set rights so that a manager is still required to Authorize the order or extend Authorize rights to regular planners at your discretion. You can also manage access to the accelerated workflow itself by user so, for example, only managers are allowed to use condensed timeline workflows (see the JustEnough Workflow Manager Standard Documentation for more information)

 

Because all three nodes within the workflow must be completed prior to Authorization and upload to the supplier, the workflow should be configured to allow transition between each node without requiring Authorization or Upload Verification for the first two. Only in the Prepack Node should rights to Authorize be made available, as Authorization will initiate upload of the order.

  • Rainbow Packs – prepacks containing more than one variant, whether these be simply different colors of the same item, or different items entirely that are somehow related, can be achieved easily enough through the integration of Variant Manager and the use of an Order Manager field called Grouping Identifier
  • In Variant Manager > Prepack tab, create a new prepack configuration containing the variants you want to prepack together and Save
  • In Grid view for the order document, enter identical values in the Grouping Identifier field for the order lines containing the variants you want to prepack together and Save
  • In the Prepack tab of the Details view, you will see the Grouping Identifier you entered for the desired order lines listed in the Prepack Groups area – select it, make sure the Calculate field for prepack configuration you created in Variant Manager is checked, and Calculate; the system will use the rainbow pack you created as it would any normal prepack
