Working with Order Documents

The majority of work within Order Manager is done in the order document viewing area to the right of the Navigation pane. Initially, on launching Order Manager, this area will appear blank unless you’ve created an order using the Create Commitments function from Receipt Plan view and opted to open the order, or clicked a link to a specific order in the Assortment Upcoming Commitments Dashboard. Assuming the area is blank, simply navigate to the desired order document in the Navigation pane and double-click it or right-click and select Open

  • Viewing Order Documents: order documents the user has chosen to view appear as tabs with the order name just below the top toolbar

  • Document Options: by right-clicking a document tab, you gain access to the following options:
  • Close: closes the order document you right-clicked
  • Close All But This: closes all open order documents except the one you right-clicked; only available when multiple documents are open
  • Float: converts the document tab to a window that can be sized and positioned on the screen (or other screens, in multiple monitor configurations)
  • Note: floating can also be achieve by simply clicking and dragging the document header without right-clicking and selecting Float
  • After floating, click the document header and drag the document to the desired location
  • Right click the document header and click Dock as Tabbed Document to return the document to its tabbed position
  • You can also hover over the directional buttons of the positioning reticle that appears while dragging to see possible docked locations for the floated window

 

  • New Vertical Tab Group: configures all documents currently being viewed vertically side by side; after clicking this option, right click the document header again and select Move to Previous Tab Group to return to the last screen configuration

  • New Horizontal Tab Group: configures all documents currently being viewed horizontally stacked; after clicking this option, right click the document header again and select Move to Previous Tab Group to return to the last screen configuration

 

  • Order Header
  • Edit: launches the Update Order Header dialog, in which users can change the Workflow (given sufficient permissions), Name, Description, and Security settings for an order document

  • Delete: launches a dialog asking the user for confirmation to delete the active order document

 

  • Yes: permanently deletes the order document
  • No: closes the delete confirmation dialog without deleting the order document
  • Workflow
  • Narrative: launches the Narrative dialog, which shows the history of document status changes for the order document

 

  • Order: non-editable integer; indicates the order in which the status changes in the narrative occurred
  • Status Date: non-editable date (mm ddd yyyy); displays the date the order document moved to the status indicated
  • Status Description: non-editable alpha numeric; displays the description of the status the document moved to on the specified Status Date
  • Status Owner: non-editable alpha numeric; displays the user ID of the user owning the document after the status change occurred
  • Note: non-editable alpha-numeric; when a note is attached to the selected entry in the narrative, displays the note text
  • Modified User: non-editable alpha-numeric; when a note is attached to the selected entry in the narrative, displays the UserID of the user who added the note
  • Redacted: user editable checkbox; when a note is attached to the selected entry in the narrative, allows the user who added the note to redact it from viewing for other users; non-editable when viewed by users other than the one who added the note
  • Add Note: launches the Add Order Note dialog, allowing users to add standalone note entries to the order document narrative

  • Note: user editable alpha-numeric; sets the text for the note to include in the narrative entry
  • Cancel: closes the dialog without adding a note to the order document narrative
  • OK: adds a new note entry to the order document narrative

 

  • Pre-Authorize: launches the Pre-Authorize Order dialog, given doing so is allowed by the workflow permissions assigned to the order document; otherwise disabled or hidden

 

  • Note: user editable alpha-numeric; sets the text for an optional note to include in the narrative entry
  • Cancel: closes the dialog without Pre-Authorizing the order document
  • OK: updates the document status to Pre-Authorized, locking it for editing, and adds a corresponding entry to the order document narrative

 

  • Authorize: launches the Authorize Order dialog, given doing so is allowed by the workflow permissions assigned to the order document; otherwise disabled or hidden
  • Same as Pre-Authorize dialog above, except clicking OK updates the document status to Authorized
  • Note: Authorized orders are uploaded to external systems per the upload schedule defined during software implementation
  • Forward: launches the Forward Order dialog, given doing so is allowed by the workflow permissions assigned to the order document; otherwise disabled or hidden

 

  • Forward To:  user editable dropdown; selects the UserID of the person to whom the order document will be forwarded
  • Note: user editable alpha-numeric; sets the text for an optional note to include in the narrative entry
  • Cancel: closes the dialog without forwarding the order document
  • OK: updates the document status to Forwarded, and adds a corresponding entry to the order document narrative
  • Retract: launches the Retract Order dialog, given doing so is allowed by the workflow permissions assigned to the order document; otherwise disabled or hidden

 

  • Note: user editable alpha-numeric; sets the text for an optional note to include in the narrative entry
  • Cancel: closes the dialog without retracting the order document
  • OK: updates the document status to what it was prior to being forwarded, and adds a corresponding entry to the order document narrative
  • Reject: launches the Reject Order dialog, given doing so is allowed by the workflow permissions assigned to the order document; otherwise disabled or hidden

 

  • Note: user editable alpha-numeric; sets the text for an optional note to include in the narrative entry
  • Cancel: closes the dialog without rejecting the order document
  • OK: updates the document status to what it was prior to being forwarded, and adds a corresponding entry to the order document narrative
  • Status
  • Edit: launches the Commitment Status dialog

  • Current Status: non-editable dropdown; displays the current workflow status of the order document
  • Status Options: user editable dropdown; sets the workflow status to transition the order document to from the available options as defined in the workflow assigned to the order document
  • Cancel: closes the Commitment Status dialog without changing the workflow status of the order document
  • OK: transitions the order document to the selected workflow status
  • Grid: shows/hides the Grid view (order lines view) in the order document viewing area; defaults to show
  • Note: Grid view is locked to show unless the Details view is also showing, in which case it can be hidden

  • Details: shows/hides the Details view in the order document viewing area; defaults to hide

The Details view contains the Break By Size and Prepack views for enriching order lines with sizing and prepack details prior to being uploaded to external systems

  • Note: the visibility of these views and their necessity in the ordering process is also defined by the workflow assigned to the order document
  • Refresh: refreshes the order document Grid view
  • Name: non-editable alpha numeric; displays the order document name
  • Description: non-editable textbox icon; when clicked, displays a text box with the order document description
  • indicates there is a description present for the order document
  • indicates no description is present for the order document
  • Workflow: non-editable alpha numeric; displays the workflow currently assigned to the order document
  • Status: non-editable alpha numeric; displays the current workflow status for the order document
  • Source: non-editable alpha numeric; displays the order document type; will always be Assortment Plan Commitment for order documents created from assortment plans
  • Owner: non-editable alpha numeric; displays the UserID of the user who created the order document
  • Security: non-editable alpha numeric; displays the security setting for the order document
  • Save: saves manual changes made to the order document in the Grid view since the last time it was saved; disabled if no changes have been made
  • Undo: undoes any manual changes made to the order document in Grid view since the last time it was saved; disabled if no changes have been made
  • Columns
  • Choose: launches the Choose Fields Dialog, in which users can select additional columns to include in the order document view to provide more information on the items in a given order

  • Cancel: closes the Item Selection dialog without selecting any items
  • OK: refreshes the order document view to include/exclude columns as specified in the Choose Fields dialog
  • Calculated: launches the Create Calculated Column dialog

 

  • Column Name: user editable alpha-numeric; sets a name for the new calculated column
  • Column Type: user editable dropdown; selects the data type for the new calculated column
  • Integer: use for data that will consist only of whole numbers
  • Decimal: use for data that requires decimal precision
  • DateTime: use for data that specifies, dates, times, or both
  • String: use for data that will be formatted and treated as text only
  • Boolean: use for binary (true/false) data
  • Format String: user editable alpha-numeric; specifies the way data in the new calculated field is to be formatted
  • Example: entering #,###.00 would yield numeric values with commas every three places to the left of the decimal and round to a static two decimal places to the right of the decimal; “1000.9” would be formatted “1,000.90” and “10.891” would be formatted “10.89”; a format string of 0,000.00 would instead yield “0010.89”
  • Example: entering mm/dd/yyyy would yield date values with two digits for month and day and four digits for year, with month, day, and year separated by slashes (11/19/2017 for example)
  • Cancel: closes the Create Calculated Column dialog without creating a new column
  • OK: launches the Expression Editor dialog

  • The Expression Editor allows users to write custom expressions with which to populate new calculated columns; for more information on using the expression editor, visit the following URL: https://documentation.devexpress.com/#WindowsForms/CustomDocument6212
  • Order Line
  • Copy: launches the Copy dialog, allowing users to copy the selected order line(s) to another order document

  • Copy To: user editable dropdown; selects the order document the copied order line(s) will be copied into; defaults to <Create New Ad Hoc Order>, which creates a new order document with the copied line(s)
  • Cancel: closes the Copy dialog without copying the selected order line(s)
  • OK: copies the selected order lines to the specified order document
  • Move: launches the Move dialog, allowing users to move the selected order line(s) from the current order document to another order document

  • Move To: user editable dropdown; selects the order document to move the selected order line(s) to; defaults to <Create New Ad Hoc Order>, which creates a new order document with the selected line(s)
  • Cancel: closes the Move dialog without moving the selected order line(s)
  • OK: moves the selected order lines to the specified order document
  • Delete: deletes the selected order line(s)
  • Warning: the system does not ask for confirmation prior to deleting the order lines, however, this action can be reversed with the Undo button
  • Merge: merges two or more order lines that share the same variant into a single line
  • Unmerge: separates previously merged order lines into their original counterparts

  • Pivot: changes the Grid view for the order document to a Pivot Grid, allowing the user to configure the view for further analysis by dragging and dropping fields into Filter, Column, Row, and Data areas

 

  • Order Document Fields
  • Variant Code: non-editable alpha-numeric; displays the variant code for the order line
  • Variant Description: non-editable alpha-numeric; displays the variant description for the order line
  • Color Code: non-editable alpha-numeric; displays the color code for the variant
  • Color Description: non-editable alpha-numeric; displays the color description for the variant
  • Cluster Name: non-editable alpha-numeric; displays the cluster name the order line was generated for
  • Note: only applicable for commitment orders generated from Receipt Plan view in Assortment Planning
  • Cluster Description: non-editable alpha-numeric; displays the cluster description the order line was generated for
  • Note: only applicable for commitment orders generated from Receipt Plan view in Assortment Planning
  • User Order Date: user-editable calendar control; displays/sets the order date for the line; defaults to the date the order was generated
  • User Ship Date: user-editable calendar control; provided to allow the user to manually enter shipping dates communicated from suppliers if systemic transfer of shipping dates is not possible; defaults to the User Order Date
  • User Delivery Date: user-editable calendar control; displays/sets the target receipt date for the order line; defaults to the Sunday of the receipt period defined in Assortment Planning
  • User Order Quantity: user-editable integer; displays/sets the total order quantity in units for the order line; defaults to the receipt units quantity for the corresponding receipt period in Assortment Planning
  • Note: orders generated from Receipt Plan view will have order lines by Cluster, in which case the order quantities will be specific to the corresponding receipt units quantity for the cluster specified by the order line; orders generated from within Order Manager are consolidated by variant by default but can be unmerged into their cluster/order line counterparts
  • User Cost Price: user editable currency ($5.00); displays/sets the cost price per unit for the order line; defaults to the receipt AUC for the corresponding receipt quantity and period in Assortment Planning
  • Note: orders generated from within Order Manager that represent multiple cluster level orders with differing AUC values will display a weighted average in this field
  • User Order Cost Value: non-editable currency ($5,000,00); displays the total cost for the order line, calculated as User Order Quantity * User Cost Price
  • Effective Order Date: non-editable calendar control; displays the date the order is confirmed received by external systems, or the User Order Date if no confirmation has been received
  • Effective Ship Date: non-editable calendar control; displays the most current ship date information sent by external systems
  • Effective Delivery Date: non-editable calendar control; displays the most current delivery date information sent by external systems
  • Effective Order Quantity: non-editable integer; displays the most current order quantity information sent by external systems
  • Effective Cost Price: non-editable currency ($5.00); displays the most current order cost per unit information sent by external systems
  • Effective Order Cost Value: non-editable currency ($5,000.00); displays the most current total cost for the order line, calculated as Effective Order Quantity * Effective Cost Price
  • Grouping Identifier: user editable alpha-numeric; allows the user to enter text to create ad hoc pre-pack groups for variants contained in order lines, thus allowing for prepacks that span more than one variant to be created; used only in the Prepack tab of the Details view
